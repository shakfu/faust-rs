# DX7 E.Piano — faust-rs Rust backend example

Plays the classic DX7 **E.PIANO 1** (ROM1A) patch — note **C5** — through code
generated by the faust-rs **Rust backend** (`-lang rust`), and writes the
result to a WAV file. No dependencies; plain `cargo run`.

```sh
cargo run --release
# → dx7-piano-c5.wav  (4 s, stereo 16-bit PCM, 44.1 kHz)
open dx7-piano-c5.wav   # listen (macOS)
```

## How it fits together

| File | Role |
|---|---|
| `dsp/dx7_alg5.dsp` | `dx.algorithm(5)` from the Faust `dx7.lib` (DX7 algorithm 5 = the E.Piano topology). Copied from `javascriptmusic/examples/dx7/dsp/`. |
| `src/dx7.rs` | **Generated** by `faust-rs -lang rust -cn Dx7Piano` — regenerate with `./regenerate.sh`. |
| `src/main.rs` | Host program: `Meta`/`UI` traits, a path-matching UI visitor that programs the E.PIANO 1 patch (values from `javascriptmusic/examples/dx7/dx7-sequence.js`), note rendering, WAV writer. |

The host controls the synth the same way a Faust C++ Rust architecture would:
it calls the generated `build_user_interface` with a UI visitor that tracks the
open/close-box group stack, resolves each full path to a `ParamIndex`, then
applies the values through `set_param`
(e.g. `DX7/Operator 2/Amp Env Generator/Rates/R1`). The `gate` button is
driven per block: on for 1.5 s, then released, with 2.5 s of release tail.

## Regenerating the DSP code

```sh
./regenerate.sh
# or explicitly:
../../../target/release/faust-rs -lang rust -cn Dx7Piano \
    -I /opt/homebrew/share/faust dsp/dx7_alg5.dsp -o src/dx7.rs
```

(Build the compiler first: `cargo build --release -p compiler` at the repo
root. `FAUSTLIBS` can point at another Faust standard-library install.)
