## Mechanize the lockstep extension in the Lean specification

Commit date: 2026-07-12
Source commit: this commit

Extended
[`vector-mode-scheduling-formal-spec.lean`](../vector-mode-scheduling-formal-spec.lean)
with the lockstep bundling layer announced in the port plan. The file still
compiles with bundled Std only, no `sorry`, no axioms.

Added:

- **Structural isomorphism modulo leaves.** `Shape` is the constructor skeleton
  of an `Expr` with leaf payloads erased (literal values, input channels, state
  resources; projection indices and clock annotations stay structural).
  `IsoModuloLeaves a b := a.shape = b.shape`, so the equivalence laws are
  inherited from `Eq`; `isoB` is the decidable checker (the reference for the
  planned Rust shape-hash detector), with `#guard` positives and negatives.
- **`iso_decorations_agree` (proved).** Isomorphic lanes receive the same value
  type, rate, vectorability, and clock — the fact that lets one bundle share a
  single transport element type and placement. Effects are deliberately not
  claimed equal; pairwise commutation handles them.
- **`LoopKind.lockstep (width)`** as a fourth kind (serial in time, parallel
  across lanes).
- **`LockstepObligations`** — the finite bundle gate: ≥ 2 lanes, no duplicates,
  lanes in the plan, and pairwise `Independent` (new `Reaches` reachability
  over plan edges), commuting `loopEffects`, same epoch rank.
- **`LockstepSafe := FissionSafe`** with the lockstep order substituted,
  making explicit that the extension adds no new semantic axiom.

## Add lockstep vectorization to the vector plans, backed by a measured benchmark

Commit date: 2026-07-12
Source commit: this commit

Explored "vertical" vectorization: k structurally identical, mutually
independent recursive loops (e.g. 4 biquads on 4 outputs) executed in lockstep,
one SIMD lane per instance — the only recursion acceleration compatible with
the bit-exact contract, since each lane runs exactly its scalar IEEE op
sequence (unlike parallel-form IIR restructurings, which reassociate).

New benchmark [`bench_lockstep_biquad.c`](../../tests/bench/bench_lockstep_biquad.c)
(4 independent RBJ peak biquads, DF2T, distinct per-lane coefficients; Apple
M1, clang 14, `-O3`, no fast-math, chunk 512, best of 3):

- V1 four separate scalar loops (current lowering): 4.15 ns/lane-sample;
- V2 plain fusion, 4 scalar chains: **3.69x, bit-exact** — clang SLP-vectorizes
  the fused body on its own, no interleaving/intrinsics needed (C/C++ backends
  only need the plan-level fusion decision);
- V3 lane-innermost C loop + chunk transposes: 3.34x bit-exact (the
  backend-neutral FIR lowering for interpreter/Cranelift/wasm);
- V4k explicit NEON on interleaved buffers: 4.53x ceiling; its 1e-5 drift vs
  scalar demonstrates the FMA-contraction-policy requirement;
- transpose overhead 12-22% ⇒ external `compute` ABI stays planar.
