## cleanup(recursion): finalize `DEBRUIJNREC` rename across propagate and shared recursion helpers

Commit date: 2026-03-28
Source commit: `f4f765f952fcfec4d22003b8cb6c0811dc6bebc1`

Completed the de Bruijn binder rename so the live Rust codebase now uses
`DEBRUIJNREC` consistently as the recursive-group binder tag instead of keeping
the earlier mixed `DEBRUIJN` / `DEBRUIJNREC` state.

### Scope

- Kept `crates/propagate/src/lib.rs` on the renamed binder tag and aligned the
  local comments/Rustdoc with the emitted runtime shape:
  - binder constructor docs now say `DEBRUIJNREC(body)`
  - body matcher docs now refer to `DEBRUIJNREC`
  - capture-avoidance comment now names the inner `DEBRUIJNREC` binder
- Updated `crates/propagate/tests/core_api.rs` so recursion-shape assertions
  check `DEBRUIJNREC` directly instead of the pre-rename `DEBRUIJN` tag
- Removed the temporary compatibility layer from `crates/tlib/src/recursion.rs`
  and `crates/tlib/src/lib.rs`:
  - `DEBRUIJNREC_TAG` is now the only exported binder tag constant
  - `match_de_bruijn_rec(...)` now matches only `DEBRUIJNREC`
  - no legacy alias/export for `DEBRUIJN_TAG` remains in live code
- Dropped the transitional legacy-binder regression from
  `crates/tlib/tests/recursive_trees.rs` so tests reflect the new canonical
  contract rather than preserving the old one
- Updated current documentation under `docs/` to describe `DEBRUIJNREC` as the
  canonical binder tag without mentioning a legacy compatibility path:
  - `docs/flatnode-rec-to-signals-en.md`
