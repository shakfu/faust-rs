`delay-rs-simplification-experiment-2026-06-21-en.md`, plus the per-strategy
file split. The whole arc is behavior- and performance-preserving: emitted FIR
is byte-identical, so the generated C/WASM is identical. `cargo test -p transform`

## Legibility experiment: delay.rs simplification analysis (FR + EN)

Commit date: 2026-06-21

A design/analysis doc (no code change) exploring whether `signal_fir/delay.rs`
(1671 lines) can be made far easier for a human to read while keeping identical
robustness and performance. Written as twin files in `porting/`:

- `delay-rs-simplification-experiment-2026-06-21-en.md` (canonical)
- `delay-rs-simplification-experiment-2026-06-21-fr.md`

Contents:

- **Context** — where `delay.rs` sits in the compilation chain (stage 3 of
  `transform`), and the three phases it participates in with `DelayManager`
  state carried between them: PREPARE (`analyze_signals` + `scan_signals` +
  `ensure_delay_line`), LOWER (`emit_*_for_line`), SAMPLE-END
  (`emit_sample_end_updates`), plus the cross-cutting recursion-merge read of
  `rec_output_analysis` from `recursion.rs`.
- **Six concrete reading frictions** (F1–F6), each anchored to `file:line`:
  the two near-duplicate tree walks (F1), the strategy concept scattered over
  five sites (F2), impossible `DelayRuntimeState` branches the types don't
  forbid (F3), the three-way split in `ensure_delay_line` (F4), the two
  hand-assembled borrow bundles (F5), and `FirBuilder` boilerplate burying the
  arithmetic (F6).
- **Three independent, orthogonal restructurings:** A — strategy as a closed
  object (vertical, one file per strategy behind a `DelayKind` trait); B — a
  `plan → emit` pipeline with an explicit immutable `DelayPlan` value
  (horizontal, collapses the two walks into one); C — a thin compile-time
  arithmetic layer so index formulas read like the doc-comments (depth).
  Comparison table + recommended order **C → B → A**.
- **Safe/testable migration** anchored on a byte-identical golden-FIR diff plus
  the existing 71 `signal_fir` tests (~30 delay-specific) and the impulse
  oracle (cpp 92/93); per-proposal recipe with differential testing during each
  switch; explicit argument for why performance is preserved (same FIR ⇒ same
  C/WASM; static enum dispatch, not `dyn`).
- **Worked before/after proof of concept:** `if_wrapping_read_index` shrinks
  from ~42 lines of builder ceremony to 3 lines that visibly match
  `buf[(idx + size − N) wrapped]`, emitting identical FIR.

Follow-up in the same session: turned the seeding request into a reusable,
hardened prompt template (explicit deliverable/format, FIR-identity invariant,
mandatory `file:line` grounding, orthogonal-axes requirement, ≥1 real
before/after); executing it surfaced the worked-example gap, now filled in both
docs.

Validation: documentation only — no code changed, no tests affected. Companion
doc links verified to resolve; `file:line` references checked against current
`delay.rs`.
