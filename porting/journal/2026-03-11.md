### `codegen` / `interp-ffi` / `cranelift-ffi` — Realign crate README files with current FFI artifacts and backend pipeline

Commit date: 2026-03-11

Source commit: pending

#### Context

Several crate README files had drifted from the current workspace structure:

- `interp-ffi` still documented itself as producing final `faust`
  `cdylib`/`staticlib` artifacts even though it now builds as an `rlib`
  consumed by `faust-ffi`
- `cranelift-ffi` still described the older crate outputs and an outdated
  internal module layout
- `codegen` still showed an inverted `fir`/`transform` pipeline order and a
  stale CLI example using `--lang fir` to generate `.fbc`

This made the crate-local documentation misleading for contributors trying to
understand the current backend and FFI packaging model.

#### Changes

- [README.md](/Users/letz/Developpements/RUST/faust-rs/crates/interp-ffi/README.md)
  - corrected crate outputs to `rlib`
  - corrected the Rust library target name to `faust_interp`
  - documented that final `cdylib` / `staticlib` distribution artifacts come
    from `crates/faust-ffi`
  - clarified that `build.rs` exists but does not currently generate headers

- [README.md](/Users/letz/Developpements/RUST/faust-rs/crates/cranelift-ffi/README.md)
  - corrected crate outputs to the current `rlib` packaging model
  - documented the `faust-ffi` aggregation role for final distribution
  - updated current-status notes to mention the native runtime descriptor and
    JIT-backed compute path already present in the implementation
  - replaced the outdated `src/ui.rs` mention with the current `runtime.rs`
    and `clif.rs` internal structure

- [README.md](/Users/letz/Developpements/RUST/faust-rs/crates/codegen/README.md)
  - fixed the pipeline diagram to the current
    `parse -> boxes -> eval -> propagate -> signals -> transform -> fir -> codegen`
    ordering
  - updated the interpreter backend section to reflect the current
    `compute_block` / `compute_dsp_block` split
  - corrected the CLI example so `.fbc` generation uses `--lang interp`
    instead of `--lang fir`

- [JOURNAL.md](/Users/letz/Developpements/RUST/faust-rs/JOURNAL.md)
  - added the `2026-03-11` daily journal entry to the top-level index

- [README.md](/Users/letz/Developpements/RUST/faust-rs/porting/journal/README.md)
  - added the `2026-03-11` day entry to the journal day index
