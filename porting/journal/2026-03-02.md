### `tlib` / `box-ffi` / `compiler` — Correct port of C++ `tree2str`

Commit date: 2026-03-02

#### Context

The C++ `tree2str` is semantically strict: it only returns a string when the
tree node is a `Symbol`, and raises a compile-time error otherwise:

```cpp
LIBFAUST_API const char* tree2str(Tree t) {
    Sym s;
    if (!isSym(t->node(), &s)) {
        ERROR("ERROR : the parameter must be a symbol known at compile time : ", t);
    }
    return name(s);
}
```

The previous Rust implementation provided `tree_to_string`, a general-purpose
converter that silently rendered any `NodeKind` (Int, Float, Tag, Nil, Cons, …)
as a string. `Ctree2str` called it unconditionally, so passing a non-symbol tree
would return garbage text instead of signalling an error — incorrect parity.

Additionally, `tree_to_string` was the only caller of itself in the two
`compiler` call sites that had already matched on the kind, so the function was
redundant there too.

#### Changes

**`crates/tlib/src/lib.rs`**
- Removed `tree_to_string` entirely.
- Added `tree_to_str(arena, id) -> Option<&str>`: returns `Some(name)` only
  when the node is `NodeKind::Symbol`, `None` otherwise. Follows the same
  `Option`-returning pattern as `tree_to_int` / `tree_to_double`.

**`crates/box-ffi/src/lib.rs`**
- `Ctree2str`: replaced `tree_to_string` with `tree_to_str`. On `None`,
  prints the C++-equivalent error message using `{:?}` on `arena.kind(id)`,
  and returns `null` (matching C++ error behaviour).
- Import updated: `tree_to_string` → `tree_to_str`.

**`crates/compiler/src/lib.rs`**
- `render_human_box_expr`: inlined the two `tree_to_string` call sites by
  binding the payload directly in the pattern match:
  - `NodeKind::StringLiteral(s) => format!("\"{}\"", s)`
  - `NodeKind::Symbol(s) => s.to_string()`
- Removed the now-unused `tree_to_string` import.

#### C++ parity

| Aspect | C++ `tree2str` | Rust `tree_to_str` |
|---|---|---|
| Non-symbol input | `ERROR(…)` + abort | `None` + `eprintln!` + `null` return |
| Symbol input | `name(s)` (interned `const char*`) | `Some(&str)` (borrowed from `Arc<str>`) |
| Type signature | `const char* tree2str(Tree)` | `fn tree_to_str(&TreeArena, TreeId) -> Option<&str>` |
