# 2026-07-23

## Document domain-local `ma.SR` and the current library clamp

Commit date: 2026-07-23
Source commit: this commit

Expanded the root README clock-domain overview to state that `ma.SR` is
automatically adapted to the effective local rate: `SR * H` inside `US(H)` and
`SR / H` inside `DS(H)`. Algorithms deriving coefficients from `ma.SR` therefore
do not need a manual rate correction inside these blocks.

Documented the current `platform.lib` limitation: its
`min(192000, max(1, fSamplingFreq))` definition keeps the adapted expression
clamped in generated code. As a result, large upsampling factors cannot expose
their full local rate—for example, `US(8)` at 48 kHz observes 192 kHz instead
of 384 kHz—until the library definition is relaxed or redesigned.

## Publish `libfaust-rs` for C/C++ and retain the Wasm module name

Commit date: 2026-07-23
Source commit: this commit

Renamed the public C/C++ distribution to `libfaust-rs`. Rust library target
names cannot contain hyphens, so `faust-ffi` builds an internal `faust_rs`
target and `cargo run -p xtask -- build-libfaust` publishes the stable native
artifacts: `libfaust-rs.a` and the platform dynamic equivalent
(`libfaust-rs.dylib`, `libfaust-rs.so`, or `faust-rs.dll`). The export checker
now validates the published dynamic library.

The separate WebAssembly compiler module intentionally retains its established
`libfaust-rs.wasm` filename. Its Cargo-internal source artifact remains
`faust_wasm_ffi.wasm`, which the `xtask` packaging step renames and validates.
