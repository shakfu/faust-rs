## Plan augmented-state recursion for local FAD consumers

Commit date: 2026-04-17
Source commit: pending

Added `porting/rec-augmented-primal-tangent-state-plan-2026-04-17-en.md`, a
new plan for replacing the current "expand-after-Rec" handling of `fad(exp, x)`
with a recursion model that can carry primal and tangent lanes directly when a
recursive branch consumes AD outputs locally.

The plan is motivated by the `auto_pan5.dsp` family:

- `fad(loss, prev)` itself is fine,
- the failure comes from `fad(loss, prev) : !, _` inside the feedback function,
- which requires the tangent to be available *inside* the recursive branch,
  not only after the enclosing `Rec` is built.

The proposed direction is:

- keep the current deferred expansion mode for existing recursive FAD corpus
  cases that do not consume AD outputs locally,
- add an augmented-state `Rec` mode when local AD consumers require the real
  `[primal, tangent]` outputs during recursive wiring.
