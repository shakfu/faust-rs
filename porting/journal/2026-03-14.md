### `transform` ‚Äî `de.delay(n, d)` with `ma.SR`-based amount: structural min fallback

Commit date: 2026-03-14

Source commit: TBD

#### Context

`ef.echo1s` (and any pattern using `de.delay(n, d, x) = x @ min(n, max(0, d))` where
`d` involves `ma.SR`) failed in faust-rs with `[FRS-SFIR-0004]`.

`ma.SR` is defined as `min(192000.0, max(1.0, fconstant(int fSamplingFreq, <math.h>)))`.
`fSamplingFreq` has interval `empty()` in the type system (C++: `interval()` = NaN,NaN ‚Äî
no static range is known).  The interval algebra's `min` and `max` functions propagate
`empty()` when either operand is empty, so the entire `hslider * ma.SR / 1000.0` chain
evaluates to `empty()`.  The delay amount `SIGMIN(int(65536), SIGMAX(int(0), ...))` then
has interval `empty()`, and `check_delay_interval` rejects it.

C++ compiles the same DSP without error.  Its type inference for `SIGDELAY` only checks
causality bounds (guarded by `gCausality`, off by default) and does NOT size the delay
line during type inference ‚Äî sizing happens in the code-generation phase.

The fix: when both the literal-int and interval-based paths fail, fall back to a
structural inspection of the delay-amount node.  `de.delay(n, d, x)` always wraps the
amount in `min(n, max(0, d))`, so `SIGMIN(SigInt(n), _)` or `SIGMIN(_, SigInt(n))`
with `n >= 0` provides a conservative upper bound even when interval analysis yields
`empty()`.  This produces the correct delay-line size `next_pow2(n+1)`.

#### Changes

`crates/transform/src/signal_fir/module.rs`:

- New private method `min_const_upper_bound`: if `sig` is `SIGMIN(SigInt(n), _)` or
  `SIGMIN(_, SigInt(n))` with `n >= 0`, returns `n`.
- `delay_size_for_amount` updated: after the interval-based path returns `None`, calls
  `min_const_upper_bound` as a final structural fallback.
- Docs for both methods explain the root cause and the `de.delay` pattern.

`tests/corpus/rep_69_variable_delay_sr_millisec.dsp` (new):

- `import("stdfaust.lib"); process = de.delay(65536, int(hslider("ms",0,0,1000,1)*ma.SR/1000.0));`
- Exercises the `min_const_upper_bound` fallback; generates delay line size 131072
  (mask `131071`) ‚Äî identical to the C++ reference output for `ef.echo1s`.

#### Validation

- 33 unit tests pass: `cargo test -p transform`
- Full workspace tests pass: `cargo test --workspace`
- `golden-check` passes for all 69 corpus entries.
- `echo.dsp` from `examples/delayEcho/` compiles successfully end-to-end.

### `transform` ‚Äî Accept audio-rate delay amounts with bounded intervals + Rustdoc

Commit date: 2026-03-14

Source commit: e44838f

#### Context

`process = @(int(_+10));` compiled in C++ but was rejected in faust-rs with
`[FRS-SFIR-0004] SIGDELAY requires a constant integer amount or a UI parameter
with a bounded interval`.

The expression `int(_+10)` has TINPUT interval `[-1, 1]` propagated through
`+10`, giving interval `[9, 11]` ‚Äî bounded, positive, and a perfectly valid
delay bound.  The rejection came from a blanket `Variability::Samp` guard in
`variable_delay_max_bound` that was added as a workaround when TINPUT still
carried the broken placeholder interval `[f64::MIN, f64::MAX]`.  With TINPUT
now correctly typed as `interval(-1, 1)` (C++ `global.cpp:671`), the guard was
obsolete and overly strict.

#### Changes

`crates/transform/src/signal_fir/module.rs`:

- Removed the `Variability::Samp` guard from `variable_delay_max_bound`.
  The existing `check_delay_interval` checks (`is_bounded()` and `hi() >= 0`)
  are sufficient ‚Äî they reject truly unbounded or empty intervals regardless
  of variability.
- Removed the now-unused `Variability` import.
- Doc comment updated to state the actual rule without historical justification.
- `delay_size_for_amount` doc: "slider / bounded UI control" ‚Üí "bounded
  interval" (variability-agnostic).
- Comprehensive Rustdoc pass over the entire file:
  - `DelayLineInfo` and `RecArrayInfo`: moved doc before `#[derive]` (Rustdoc
    requirement), added field-level docs.
  - `SignalToFirLower`: all 34 fields now documented; struct-level doc rewritten
    as a concise paragraph.
  - `build_module`: added `# Parameters` section.
  - 22 previously undocumented private methods now have doc comments:
    `scan_delay_lines`, `scan_delay_child`, `lower_fixed_delay`,
    `recursion_feedback_info`, `active_recursion_info`, `ensure_delay_line_decl`,
    `ensure_iota_state`, `current_iota_index`, `delayed_iota_index`,
    `masked_delay_index`, `bump_iota`, `register_clear_table`,
    `register_clear_recursion_array`, `fresh_loop_var`, `constant_delay_amount`,
    `pow2limit_for_delay`, `ensure_button_zone`, `control_spec`, `control_range`,
    `emit_ui_metadata_for_target`, `emit_control_ui_metadata`,
    `ensure_slider_zone`, `ensure_bargraph_zone`, `ensure_soundfile_zone`,
    `emit_ui_program`, `emit_ui_node`, `ensure_recursion_array`.
  - Existing docs for `new`, `prepare_delay_lines`, `lower_signal` tightened.

`crates/transform/src/signal_fir/mod.rs`:

- `variable_delay_is_rejected_explicitly` renamed to
  `variable_delay_with_audio_input_amount_uses_tinput_interval`: now asserts
  that `input(1)` as a delay amount is **accepted** (interval `[-1, 1]`, hi=1,
  produces a 2-sample delay line).
- New test `variable_delay_with_strictly_negative_hi_is_rejected`: slider shifted
  by -200 gives interval `[-200, -100]` (hi=-100 < 0) ‚Äî must be rejected.

`tests/corpus/rep_68_variable_delay_audio_rate.dsp` (new):

- `process = @(int(_+10));` ‚Äî audio-rate amount with interval `[9, 11]`.

#### Validation

- 33 unit tests pass: `cargo test -p transform`
- Full workspace tests pass: `cargo test --workspace`
- `golden-check` passes for all 68 corpus entries including `rep_68`.

### `sigtype` ‚Äî Full type inference parity audit vs `sigtyperules.cpp` (15 fixes)

Commit date: 2026-03-14

Source commit: 6baa561

#### Context

Following the sigtype crate implementation (2026-03-13), a systematic
line-by-line comparison of `TypeAnnotator::infer_inner()` against the C++
`inferSigType()` in `compiler/signals/sigtyperules.cpp` revealed 15 rules
that diverged.  All were corrected.

#### Changes

`crates/sigtype/src/rules.rs`:

**Input**
- `Interval::new_default()` ‚Üí `Interval::new(-1.0, 1.0, -24)`.
  C++: `TINPUT = makeSimpleType(kReal, kSamp, kExec, kVect, kNum, interval(-1, 1))`.

**Button / Checkbox**
- `Nature::Int` ‚Üí `Nature::Real`
- `Vectorability::Scal` ‚Üí `Vectorability::Vect`
- `Boolean::Bool` ‚Üí `Boolean::Num`
  C++: `castInterval(TGUI, gAlgebra.Button(interval(0)))` where `TGUI` is
  `{kReal, kBlock, kExec, kVect, kNum, interval()}`.

**VSlider / HSlider / NumEntry**
- `Computability::Init` ‚Üí `Computability::Exec`.
  C++: `castInterval(TGUI, ...)` inherits `kExec` from `TGUI`.

**Delay1 / Delay**
- Added `promote_interval(reunion(tx.interval, singleton(0)))`.
  C++: `castInterval(sampCast(t), reunion(t->getInterval(), interval(0)))`.
  The first sample of any delay is 0; the interval must include it.

**Select2**
- Removed forced `samp_cast`; all-Konst inputs now stay Konst.
- Condition contributes to variability/computability join.
  C++: `makeSimpleType(st1|st2 nature, st1|st2|stsel variability/comp/vec, ...)`.

**Seq**
- `union_types(T(x), T(y))` ‚Üí `{ self.infer(x)?; self.infer(y) }`.
  C++: `T(x,env); return T(y,env)` ‚Äî type is the second signal's type only.

**PermVar / ZeroPad**
- Added `promote_interval(reunion(tx.interval, singleton(0)))`.
  C++: same `reunion` call as Delay1.

**OnDemand / Upsampling / Downsampling**
- Was `infer_array(subs)` ‚Üí `TupletType`.
- Now `make_simple(Real, Samp, Exec, Scal, Num, interval(-1,1))`.
  C++: `makeSimpleType(kReal,kSamp,kExec,kScal,kNum,interval(-1,1))`.
  (Must NOT be `kKonst`, otherwise constant propagation removes the node.)

**Lowest / Highest**
- `int_cast(tx)` ‚Üí `make_simple(Real, Konst, Comp, Vect, Num, singleton(tx.interval.lo/hi))`.
  C++: `makeSimpleType(kReal,kKonst,kComp,kVect,kNum, interval(i1.lo/hi()))`.

**AssertBounds**
- Fixed decode order (was `_, lo=max, hi=cur`).
- Return type: `cur`'s type with interval clamped to
  `[max(cur.lo, min.lo), min(cur.hi, max.hi)]`.

**Bargraph (HBargraph / VBargraph)**
- `samp_cast(tx).promote_interval(bargraph_range)` ‚Üí
  `tx.promote_variability(Variability::Block)`.
  C++: `T(s1,env)->promoteVariability(kBlock)` ‚Äî lo/hi not used in return type.

**FConst**
- `interval::new_default()` ‚Üí `interval::empty()`.
  C++: `interval()` (NaN, NaN ‚Äî no static range).

**FVar**
- Separated from FConst path: `Variability::Block`, `Computability::Exec`.
  C++: `makeSimpleType(tree2int(type),kBlock,kExec,kVect,kNum,interval())`.

**Proj**
- Added group-level variability/computability promotions and forced `Scal`.
  C++: `inferProjType(..., kScal)`.

**Soundfile / SoundfileLength / SoundfileRate / SoundfileBuffer**
- All had wrong fields or were `make_maximal()`.
  Now match exact C++ types (Int/Block/Init for length and rate, dynamic
  variability from part argument for buffer).

#### Validation

- 32 unit tests pass: `cargo test -p sigtype` (27 ‚Üí 32 after new tests)
- Full workspace tests pass: `cargo test --workspace`
