### `eval` × `normalize` — parity plan: `boxPropagateSig` + `simplify` call sites

Commit date: 2026-03-14

Source: `compiler/evaluate/eval.cpp`

#### Context

Now that `normalize::simplify` is available, the next milestone is wiring it
into the eval crate so that the Rust evaluator achieves the same
constant-folding and parameter-reduction behaviour as the C++ evaluator.  The
C++ `eval.cpp` calls `boxPropagateSig` + `simplify` at six distinct sites; the
plan covers all six without deferral.

#### Deliverables

`porting/eval-simplify-parity-plan-2026-03-14-en.md` (new, ~850 lines):

- **§2 — Six C++ call sites** analysed with exact line numbers and code
  excerpts:
  - CS-1 (`isBoxSeq`): numeric folding of `(2,3):+` → `5`.
  - CS-2 (`isBoxRoute`): route parameter normalisation to literal `boxInt(n)`.
  - CS-3 (`isBoxNumeric`): test whether a box is a compile-time numeric constant.
  - CS-4 (`eval2double`): convert a 0→1 box to `f64` (slider bounds).
  - CS-5 (`eval2int`): convert a 0→1 box to `i32` (table sizes, channel index).
  - CS-6 (`boxSimplification`): memoised box-level constant simplifier, called
    on every UI widget parameter.

- **§3 — Current Rust approximation**: `eval_box_to_scalar_signal` already
  calls `propagate_typed` but not `simplify`; `map_children` handles `Seq`
  and `Route` without numeric folding.

- **§5 — Eight implementation steps**:
  1. Expose `simplify_const(arena, sig)` in `normalize`'s public API
     (empty type map — sufficient for constant folding at eval time).
  2. Add `normalize` dep to `eval/Cargo.toml`.
  3. `propagate_box_and_simplify`: private helper combining `propagate_typed` +
     `simplify_const` for 0→1 boxes.
  4. `is_box_numeric`, `eval_box_to_f64`, `eval_box_to_i32` (CS-3/4/5).
  5. `box_simplification` + `inside_box_simplification` with memoisation (CS-6).
  6a. Call `box_simplification` on every UI widget parameter (CS-4/5/6).
  6b. `eval_box_to_int_node` + `normalize_route_spec` + `BoxMatch::Route` arm
      in `eval_value` (CS-2 — see below).
  7. `eval_seq_with_folding` + `BoxMatch::Seq` arm (CS-1).
  8. `simplify_const` call in `eval_box_to_scalar_signal` (CS-3 quick win).

- **§6 — CS-2 Route** (not deferred): the `propagate` crate's
  `usize_from_int_node` and `flatten_route_ints_into` require literal `Int`
  nodes; arithmetic expressions (`1+1`) cause `PropagateError` at propagation
  time.  The fix belongs in `eval` (reduce to literals before leaving eval),
  not in `propagate`.

### `normalize` — Step 4: `simplify.rs` — memoized rewrite engine

Commit date: 2026-03-14

Source: `compiler/normalize/simplify.cpp`

#### Changes

`crates/normalize/src/simplify.rs` (new, ~620 lines):

- `simplify(arena, types, sig)`: public entry point.  Allocates a per-pass
  `HashMap<SigId, Option<SigId>>` cache and delegates to `sig_map`.
  C++: `Tree simplify(Tree sig)` with `gGlobal->SIMPLIFIED` global key.

- `sig_map(arena, cache, types, sig)`: memoized depth-first traversal.
  - Cache hit: `None` → return `sig`; `Some(r)` → return `r`.
  - `Rec(body)`: insert `None` sentinel before descending (break cycles),
    return `Rec(new_body)`.
  - General case: clone `(kind, children)` from arena, map each child,
    rebuild via `arena.intern(kind, new_children)`, apply `simplification`,
    cache.

- `simplification` + `match_simplification`: local algebraic rewrite rules:
  - Math primitive constant folding (sin, cos, exp, log, sqrt, abs, floor,
    ceil, rint, round, acos, asin, atan, atan2, fmod, remainder, pow, min, max).
  - BinOp constant folding (Int op Int, Float op Float, mixed).
  - Negation rewrites: `-n*(x-y) → n*(y-x)`, `-1*(x-y) → y-x`.
  - Nested mul folding: `n*(m*x) → (n*m)*x`.
  - Special Sub: `0-x → -1*x`.
  - Neutral/absorbing elements for Add/Sub/Mul/Div/And/Or/Xor/shifts.
  - Self-ops: `x AND x = x`, `x >= x = 1`, `x < x = 0`, etc.
  - AND/OR with boolean: `1 AND bool = bool`.
  - Default BinOp: `normalize_add_term`.
  - Delay1 / Delay: `normalize_delay1_term` / `normalize_delay_term`.
  - IntCast / FloatCast: constant folding.
  - BitCast: pass through.
  - Select2: selector constant folding + identical-branch collapse.
  - Enable / Control: 0/1 simplification.
  - Lowest / Highest: extract interval bound from types map when available.

- `fold_binop`: all 17 BinOp variants; integer wrapping; no div-by-zero.
- `is_left/right_neutral/absorbing`, `is_sig_bool`, `numeric_val`.
- 18 unit tests.

`crates/normalize/src/lib.rs`: declared `pub(crate) mod simplify`.

**Design**: cache is local `HashMap` per call (no global state).
Raw `arena.node(id)` + `arena.intern(kind, children)` used for general node
rebuilding without exhaustive match.

### `normalize` — Step 3: `normalize.rs` — add-term + delay-term normalization

Commit date: 2026-03-14

Source: `compiler/normalize/normalize.cpp`

#### Changes

`crates/normalize/src/normalize.rs` (new, ~270 lines):

- `normalize_add_term(arena, types, t)`: constructs an `Aterm`, then runs the
  factor-extract loop — while `D = greatest_divisor` has `complexity > 0`,
  factorizes the aterm by D.  Returns `aterm.normalized_tree()`.
  C++: `normalizeAddTerm(Tree t)`.

- `normalize_delay1_term(arena, types, s)`: delegates to `normalize_delay_term`
  with `d = sigInt(1)`.  C++: `normalizeDelay1Term(Tree s)`.

- `normalize_delay_term(arena, types, s, d)`: if `s` is a `Clocked` signal,
  strips the clock and dispatches to `clock_normalize_delay_term`.  Otherwise
  returns `sigDelay(s, d)` unchanged.
  C++: `normalizeDelayTerm(Tree s, Tree d)`.

- `clock_normalize_delay_term(arena, types, clock, s, d)` (pub(crate)): applies
  the five rewrite rules inside a clock scope:
  - `s @ 0` → `clk :: s` (or `sigDelay(clk :: s, 0)` for `Proj` nodes)
  - `0 @ d` → `clk :: 0`
  - `(k * s) @ d` with `order(k) < 2` → `k * clock_normalize((clk :: s) @ d)`
  - `(s / k) @ d` with `order(k) < 2` → `clock_normalize((clk :: s) @ d) / k`
  - `(s @ n) @ m` with `order(n) < 2` → `clock_normalize(clk :: s, d + n)`
  - fallthrough → `sigDelay(clk :: s, d)`

- `unclock(arena, s)` (private): iteratively strips `Clocked` wrappers.

- Declared `pub(crate) mod normalize`.

  `normalized_tree` to combine signed partial sums.

### `sigtype` + `transform` — Fix `FConst`/`FVar` interval; add `min_const_upper_bound` fallback

Commit date: 2026-03-14

Source commits: 1a0d914, TBD

#### Context

`ef.echo1s` (and any pattern using `de.delay(n, d, x) = x @ min(n, max(0, d))` where
`d` involves `ma.SR`) failed in faust-rs with `[FRS-SFIR-0004]`.

`ma.SR = min(192000.0, max(1.0, fconstant(int fSamplingFreq, <math.h>)))`.

The root cause was a wrong parity fix applied to `FConst` in the 2026-03-13 sigtype
audit (see entry below).  The audit changed `FConst`'s interval from `Interval::new_default()`
to `interval::empty()`, claiming "C++: `interval()` = NaN, NaN".  But **C++ `interval()`
is not empty**: it uses default member initializers:

```cpp
// compiler/interval/interval_def.hh
double fLo{std::numeric_limits<double>::lowest()};  // = f64::MIN
double fHi{std::numeric_limits<double>::max()};     // = f64::MAX
int    fLSB{-24};
```

`interval()` in C++ = `[f64::MIN, f64::MAX]` = Rust `Interval::new_default()`.
With the wrong `empty()` interval, `min`/`max` propagated empty through the entire
`hslider * fSamplingFreq / 1000` chain, giving `empty()` for the delay amount — which
`check_delay_interval` correctly rejected.

With the corrected `new_default()` interval:
- `fSamplingFreq ∈ [MIN, MAX]`
- `max(1.0, fSamplingFreq) ∈ [1.0, MAX]`
- `min(192000, ...) ∈ [1.0, 192000]` — interval algebra correctly clamps at 192000
- `hslider[0,1000] * [0.001, 192] ∈ [0, 192000]`
- `int_cast → [0, 192000]`, `- 1 → [-1, 191999]`, `max(0, ...) → [0, 191999]`
- `min(65536, ...) → [0, 65536]` — `check_delay_interval` returns 65536 ✓
- Delay line size: `next_pow2(65537)` = 131072 — matches C++ output

Note on `gCausality`: `compile_scal.cpp` calls `typeAnnotation(L2, true)`, so causality
IS active in C++ — but the causality check only fires for `SIGDELAY` inside `sigtyperules`,
which uses the same (correct) type information.  The interval `[0, 65536]` passes the
causality check `i.isValid() && lo >= 0 && hi < INT_MAX`.

#### Changes

`crates/sigtype/src/rules.rs` (this commit):

- `infer_foreign_const_type` and `infer_foreign_var_type`: `interval::empty()` →
  `Interval::new_default()`.  Docstring now cites the C++ member initializers.

`crates/transform/src/signal_fir/module.rs` (commit 1a0d914, retained):

- `min_const_upper_bound`: structural fallback for `SIGMIN(SigInt(n), _)`.  With
  correct FConst typing this fallback is not exercised for `ma.SR` patterns, but it
  remains as defence-in-depth for any other expression that yields empty/unbounded
  intervals as a delay amount.
- `delay_size_for_amount` doc updated to reflect that the structural path is a last resort.

`tests/corpus/rep_69_variable_delay_sr_millisec.dsp` (commit 1a0d914, retained):

- `import("stdfaust.lib"); process = de.delay(65536, int(hslider("ms",0,0,1000,1)*ma.SR/1000.0));`
- Now exercises the **interval path** (bound from `[0, 65536]`), not the structural fallback.
  Golden output: delay line 131072, mask `131071`.

### `normalize` — Signal simplification/normalization porting plan

Commit date: 2026-03-14

#### Context

The C++ `compiler/normalize/` directory implements the algebraic simplification of
Faust signals before code generation: constant folding, additive/multiplicative
factorization, delay normalization, and the full `simplifyToNormalForm` pipeline.
The Rust `crates/normalize/` crate existed only as a scaffold.

A detailed porting plan was written to drive the implementation.

#### Plan contents

`porting/normalize-simplify-port-plan-2026-03-14-en.md`

Five-step port, in dependency order:

| Step | Module | Source | Role |
|---|---|---|---|
| 1 | `mterm.rs` | `mterm.hh/.cpp` (14 KB) | Multiplicative term `k·x^n·y^m/…` |
| 2 | `aterm.rs` | `aterm.hh/.cpp` (8 KB) | Additive term (sum of mterms) |
| 3 | `normalize.rs` | `normalize.hh/.cpp` (5 KB) | Add-term + delay normalization |
| 4 | `simplify.rs` | `simplify.hh/.cpp` (13 KB) | Rewrite engine + memoised `sig_map` |
| 5 | `normalform.rs` | `normalform.hh/.cpp` (6 KB) | Pipeline coordinator (Phase 1 scope) |

**Key design decisions:**

- `BTreeMap<SigId, …>` replaces `std::map<Tree, …>` (deterministic order).
- C++ property cache (`setProperty(SIMPLIFIED, …)`) → `HashMap<SigId, SigId>` local
  to each pass — no global mutable state.
- `gGlobal->…` options → `NormalFormOpts` struct passed by reference.
- `getSigOrder()` → `sig_order()` in `normalize/src/order.rs` if not already in `sigtype`.

**Step 5 scope — Phase 1 only (rest deferred):**

Phase 1 of `normalform.rs` covers only the three sub-steps currently performed in
`crates/transform/signal_prepare`: de Bruijn → symbolic, type annotation, signal
promotion. The full C++ pipeline (UI promotion, FTZ wrapping, auto-diff, double
promotion pass, causality check) is deferred to Phase 2.

#### Validation plan

```bash
cargo test -p normalize     # 17 unit tests (mterm, aterm, delay rules, simplify rules)
cargo test -p transform     # regression on existing signal_prepare tests
cargo test --workspace      # full suite
```

### `sigtype` — Full type inference parity audit vs `sigtyperules.cpp` (15 fixes)

Commit date: 2026-03-14

Source commit: 6baa561

#### Context

Following the sigtype crate implementation (2026-03-13), a systematic
line-by-line comparison of `TypeAnnotator::infer_inner()` against the C++
`inferSigType()` in `compiler/signals/sigtyperules.cpp` revealed 15 rules
that diverged.  All were corrected.

#### Changes

`crates/sigtype/src/rules.rs`:

**Input**
- `Interval::new_default()` → `Interval::new(-1.0, 1.0, -24)`.
  C++: `TINPUT = makeSimpleType(kReal, kSamp, kExec, kVect, kNum, interval(-1, 1))`.

**Button / Checkbox**
- `Nature::Int` → `Nature::Real`
- `Vectorability::Scal` → `Vectorability::Vect`
- `Boolean::Bool` → `Boolean::Num`
  C++: `castInterval(TGUI, gAlgebra.Button(interval(0)))` where `TGUI` is
  `{kReal, kBlock, kExec, kVect, kNum, interval()}`.

**VSlider / HSlider / NumEntry**
- `Computability::Init` → `Computability::Exec`.
  C++: `castInterval(TGUI, ...)` inherits `kExec` from `TGUI`.

**Delay1 / Delay**
- Added `promote_interval(reunion(tx.interval, singleton(0)))`.
  C++: `castInterval(sampCast(t), reunion(t->getInterval(), interval(0)))`.
  The first sample of any delay is 0; the interval must include it.

**Select2**
- Removed forced `samp_cast`; all-Konst inputs now stay Konst.
- Condition contributes to variability/computability join.
  C++: `makeSimpleType(st1|st2 nature, st1|st2|stsel variability/comp/vec, ...)`.

**Seq**
- `union_types(T(x), T(y))` → `{ self.infer(x)?; self.infer(y) }`.
  C++: `T(x,env); return T(y,env)` — type is the second signal's type only.

**PermVar / ZeroPad**
- Added `promote_interval(reunion(tx.interval, singleton(0)))`.
  C++: same `reunion` call as Delay1.

**OnDemand / Upsampling / Downsampling**
- Was `infer_array(subs)` → `TupletType`.
- Now `make_simple(Real, Samp, Exec, Scal, Num, interval(-1,1))`.
  C++: `makeSimpleType(kReal,kSamp,kExec,kScal,kNum,interval(-1,1))`.
  (Must NOT be `kKonst`, otherwise constant propagation removes the node.)

**Lowest / Highest**
- `int_cast(tx)` → `make_simple(Real, Konst, Comp, Vect, Num, singleton(tx.interval.lo/hi))`.
  C++: `makeSimpleType(kReal,kKonst,kComp,kVect,kNum, interval(i1.lo/hi()))`.

**AssertBounds**
- Fixed decode order (was `_, lo=max, hi=cur`).
- Return type: `cur`'s type with interval clamped to
  `[max(cur.lo, min.lo), min(cur.hi, max.hi)]`.

**Bargraph (HBargraph / VBargraph)**
- `samp_cast(tx).promote_interval(bargraph_range)` →
  `tx.promote_variability(Variability::Block)`.
  C++: `T(s1,env)->promoteVariability(kBlock)` — lo/hi not used in return type.

**FConst**
- `interval::new_default()` → `interval::empty()`.
  C++: `interval()` (NaN, NaN — no static range).

**FVar**
- Separated from FConst path: `Variability::Block`, `Computability::Exec`.
  C++: `makeSimpleType(tree2int(type),kBlock,kExec,kVect,kNum,interval())`.

**Proj**
- Added group-level variability/computability promotions and forced `Scal`.
  C++: `inferProjType(..., kScal)`.

**Soundfile / SoundfileLength / SoundfileRate / SoundfileBuffer**
- All had wrong fields or were `make_maximal()`.
  Now match exact C++ types (Int/Block/Init for length and rate, dynamic
  variability from part argument for buffer).

#### Validation

- 32 unit tests pass: `cargo test -p sigtype` (27 → 32 after new tests)
- Full workspace tests pass: `cargo test --workspace`
