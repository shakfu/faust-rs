## Fix Windows directory-info path assertion

Commit date: 2026-06-17
Source commit: this commit

Fixed a Windows CI failure in the compiler CLI test for C++-compatible
directory-info precedence. The test previously hardcoded the Unix-rendered
`/opt/faust/lib` string, while `FaustInstallPaths::from_parts(...)` builds a
native `PathBuf` and therefore renders `/opt/faust\lib` on Windows.

The assertion now derives the expected output from `paths.lib_dir` through the
same path rendering helper used by the implementation. This keeps the test
focused on precedence behavior without baking in Unix separators.

`AGENTS.md` now also documents that code constructing, normalizing, displaying,
or comparing filesystem paths must be checked for cross-platform behavior, with
`Path`/`PathBuf` and component-aware comparisons preferred over ad hoc string
handling.

Validation:

- `cargo test -p compiler --bin faust-rs render_directory_info_uses_cpp_precedence`
- `cargo test -p compiler --bin faust-rs`
- `cargo fmt --all -- --check`
