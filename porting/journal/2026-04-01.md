## fix(tests): make Cranelift runtime smoke corpus explicit about stdlib fixtures

Commit date: 2026-04-01
Source commit: working tree after `2597c7d`

Adjusted the Cranelift interpreter-vs-JIT runtime smoke corpus so it no longer
assumes `stdfaust.lib` is globally installed on CI runners. The smoke test now
passes an explicit Faust libraries root to both the interpreter compile path
and the Cranelift FFI file-constructor path when available, and it excludes the
two stdlib-dependent corpus cases from the default smoke list when no library
root is configured.

  - pass `-I <root>` argv entries into

  - keep the self-contained smoke fixtures always enabled
  - include `rep_61_fmin_sr.dsp` and `rep_64_dynamic_rem.dsp` only when a Faust

## fix(tests): make `zita_pipeline` explicit about optional Faust library roots

Commit date: 2026-04-01
Source commit: working tree after `1fc7ccb`

Adjusted the `zita_pipeline` integration tests so they no longer assume that
`stdfaust.lib` is installed in the runner's default search path. The tests now
mirror the repository's existing optional-external-corpus pattern: use
`FAUST_RS_FAUSTLIBRARIES_ROOT` (or the local default checkout path) when
available, otherwise skip only the specific `stdfaust.lib`-missing case instead
of failing the whole `compiler` test target in CI.

### Scope

- updated
  [crates/compiler/tests/zita_pipeline.rs](/Users/letz/Developpements/RUST/faust-rs/crates/compiler/tests/zita_pipeline.rs)
  to:
  - add a local `faustlibraries_root()` helper aligned with the other optional
    stdlib-dependent tests
  - compile `zita_min.dsp` through `compile_file_to_signals(..., &[root])` when
    an explicit Faust libraries checkout is available
  - skip only `CompilerError::Import(SourceReaderError::UnresolvedImport {
    name: "stdfaust.lib", .. })` when no library root is configured
  - keep all other parse/eval/propagate/lowering failures fatal

### Validation

- `cargo test -p compiler --test zita_pipeline`
