## fix(tests): normalize path separators in Windows CI path assertions

Commit date: 2026-04-01

Two compiler unit tests failed on the Windows CI runner:

- `compiler_compile_wasm_artifact_supports_memory_source_import_dirs`
- `compiler_compile_file_to_wasm_emits_file_provenance_fields`

Both asserted that `out.dsp_json.contains(&root.display().to_string())`. On
Windows, `Path::display()` emits backslash separators (`C:\Users\...\tmp\...`),
but the Faust C++ backend serializes all paths in JSON using forward slashes
(`C:/Users/.../tmp/...`). The `contains` check therefore always failed.

### Scope

- updated
  [crates/compiler/src/lib.rs](/Users/letz/Developpements/RUST/faust-rs/crates/compiler/src/lib.rs):
  replaced both `root.display().to_string()` path assertions with a
  forward-slashâ€“normalized equivalent (`replace('\\', "/")`).

### Validation

- `cargo test -p compiler --lib compiler_compile_wasm_artifact_supports_memory_source_import_dirs compiler_compile_file_to_wasm_emits_file_provenance_fields`
