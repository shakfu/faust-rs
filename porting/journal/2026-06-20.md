## Formalize signal-to-FIR as a typed-rewriting calculus

Commit date: 2026-06-20
Source commit: this commit

Added `porting/signal-to-fir-rewriting-calculus-2026-06-20-en.md`, an abstract
companion to the implementation analysis that frames every pipeline phase as a
typed rewrite `phase : {t | Pre(t)} → {t | Post(t)}` between refinement sorts
over two term algebras (`𝕋_sig`, `𝕋_fir`).

The document defines the tower of invariant predicates, the lowering precondition
`L_prep` as their conjunction, a taxonomy of the six phase kinds (refresh /
narrowing rewrite / decoration / membership decision / translation / intra-FIR
rewrite), and — for every phase — the complete rule set, grounded in the
implementation (`normalform.rs`, `simplify.rs`, `rec_merge.rs`, `delay.rs`, and
the lowering modules): de Bruijn reification, unary-projection canonicalization,
the full promotion cast inventory (with `P` as its normal form), the algebraic
simplification rules, recursion-group merging, one-sample-delay canonicalization,
the typing/reduced-type/delay/placement attribute rules, the `verify` acceptance
predicate, the full lowering translation (delay layers, recursion carriers,
BlockReverseAD), and intra-FIR CSE.

The verification section derives the analysis's W-findings (W2–W5, W8, W13) as
type/coverage/soundness obligations and surfaces one new gap: `verify` enforces a
predicate strictly weaker than the documented lowering precondition `L_prep` — it
checks neither `P` (promotion invariant) nor `D1` (one-sample-delay form).

A bibliography section maps each borrowed concept (refinement types, Hoare logic,
term rewriting, denotational/stream semantics, the Faust signal algebra,
progress/preservation, abstract interpretation, de Bruijn/nominal binding,
attribute grammars, catamorphisms, hash-consing, CSE/value numbering, compiler
correctness, IEEE-754 semantics, reverse-mode AD) to the section that uses it,
with references.

Also added a cross-link from the analysis document to this formal companion.

## Remove dead delay/option state in the FIR fast-lane

Commit date: 2026-06-20
Source commit: this commit

Acted on findings W6/I12 from
`porting/signal-to-fir-transform-analysis-2026-06-20-en.md` by removing verified
dead state from `crates/transform`:

- `DelayManager::rec_group_max_delay` was write-only (populated by
  `try_record_rec_delay`, never read; recursion sizing uses
  `rec_output_analysis`). Removed the map and folded the pattern check into the
  existing `is_recursion_delay_chain` predicate, which is exactly equivalent.
- The per-signal `delay_analysis` map and its accessor had no readers. Removed
  the map, `record_delay_analysis`, and its call site in `analyze_node`, keeping
  the consumed `rec_output_analysis`.
- `SignalFirOptions::strict_mode` was never read; every construction site set it
  to the default. Removed the field and updated the in-workspace construction
  sites in `transform`, `box-ffi`, and `compiler`.
- Dropped now-stale `#[allow(dead_code)]` on the used `get_delay_line` and
  `rec_output_analysis` accessors.

No behavior change; net -64 lines.

## Analyze signal-to-FIR transformation pipeline

Commit date: 2026-06-20
Source commit: this commit

Added `porting/signal-to-fir-transform-analysis-2026-06-20-en.md`, a deep
end-to-end analysis of the signal-to-FIR transformation in `crates/transform`
(`signal_prepare` + `signal_fir`).

The document walks every step of the three stages — the `plan_signals` contract
gate, the `signal_prepare` staging sequence (clone, `de_bruijn_to_sym`, unary
projection canonicalization, the five interleaved type passes, two promotions,
two simplifications, recursion-group merge, one-sample-delay canonicalization,
reduced-type derivation, postcondition verification), and the `build_module`
lowering stage (sharing/variability pre-pass, delay analysis/scan/alloc,
reverse-time classification, `lower_signal` dispatch, BRA scheduling, per-bucket
CSE, lifecycle assembly) — describing inputs, outputs, invariants, justification,
and sequencing for each.

It then records 15 weaknesses (W1–W15) and 12 improvements (I1–I12), with the
non-obvious findings verified against the source: five full type annotations per
preparation, two overlapping delay-analysis passes, the write-only
`rec_group_max_delay` map and unused per-signal `delay_analysis` map, the unread
`strict_mode` option, the fixed `MAX_BRA_TAPE_BLOCK_SIZE` overflow cliff, the
verifier-vs-lowerer node-family boundary mismatch, and the `SignalToFirLower`
god-struct.
