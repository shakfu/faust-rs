## Propagate parser precision variants to `--double`

Commit date: 2026-07-15
Source commit: this commit

Fixed a precision-selection regression in the Rust parser. The compiler's
`--double`/`Float64` setting was reaching FIR/code generation but not the
parser context that selects `singleprecision` versus `doubleprecision`
definitions. As a result, `ma.MIN` could use the single-precision variant in
`MIN_test`, overflow after multiplication by `1e307`, and produce `inf`.

Parser entry points now accept the C++-compatible `float_size` mode, import
expansion preserves it for every loaded source, and the evaluator propagates
the session precision through dynamically loaded libraries. Added a compiler
regression test for double-precision variant selection.

## Bound recursiveness analysis to one expansion per signal

Commit date: 2026-07-15
Source commit: this commit

Fixed a scalar compilation blow-up exposed by the `jprev_demo_test` entry in
`faustlibraries/tests/demos_tests.dsp`. A macOS process sample showed the
compiler spending nearly all of its time in
`signal_fir::vector_analysis::compute_recursiveness` after reaching roughly
2 GB of resident state.

The Rust analysis memoized recursiveness by `(signal, complete recursive
environment)`. On a heavily shared recursive DAG, this revisited the same
signal once per binder combination. The C++ reference instead stores the
`RECURSIVNESS` property directly on each signal and returns that first-visit
value independently of later environments. The Rust memo is now keyed by
signal identity only, preserving that C++ behavior and making the traversal
linear in the reachable signal DAG.

Added a structural regression with 18 shared recursive layers. It checks that
the 91 reachable signals are expanded exactly once; the former key admitted an
exponential `2^18` family of environments. This is a private implementation
alignment with no public API or ABI impact.

  `faust-rs -pn jprev_demo_test tests/demos_tests.dsp`, which now emits the

`delayed recursive carrier -> recursive writer loop` with a certified fused
serial group. The strategy-independent plan retains all logical loop
identities, while routing rematerializes each internal transport as a scalar
stack value and assembly emits the member iteration bodies in topological order
inside one physical `for i0` loop. Unsupported multi-carrier, longer-chain, or
clock-crossing shapes still fail closed through the existing vector-plan
fallback.

Extended the vector-plan DTO and JSON schema with fused-group records. The
independent Rust L2 checker reconstructs delayed dependencies, recursive
carrier/writer identity, clock compatibility, loop ownership, and exact
internal-transport coverage from verified decorations. The assembled-FIR
checker separately requires copy-in before the physical loop, delayed read and
state write inside it, copy-out afterwards, and scalar store/load routing for

internal compiler DTO with no C/C++ ABI impact. Assembly keeps that checked
plan record authoritative and validates its projection directly against
`VectorFirAssembly::top_level_statement`, instead of duplicating the same
membership in a second independently mutable assembled-group DTO. This avoids
plan/assembly identity drift while preserving the plan's required observable
`pre -> one serial body -> post` shape.

## Guard unsafe vector transports for recursive delay reads

Commit date: 2026-07-15
Source commit: this commit

Fixed a vector-mode runtime regression exposed by
`ba.pulse_countup_loop(4, 1) + 0.001` under `-vec -lv 1 -ss 3`: the checked
vector path could transport a delayed recursive carrier across vector loops,
precomputing chunk reads before the loop that writes the next recursive state.
The generated C++ then read stale or uninitialized short-delay storage and
could emit `-inf`.

The vector assembly now orders non-clocked short-delay copy-in phases before
all chunk loop bodies, and copy-out phases after them. More importantly, the
checked vector selector rejects cross-loop transports whose producer depends on
a delayed recursive carrier; those cases use the scalar fallback path until the
vector certificate can prove a fused serial loop for that pattern. Fallback
module construction now uses `ComputeMode::Scalar` rather than re-entering the
transitional vector module.

Added a `vector_mode` regression test for `pulse_countup_loop` across both
loop variants and all four scheduling strategies. Validated with:

## Isolate scheduled temporaries between sibling clock regions

Commit date: 2026-07-15
Source commit: this commit

Fixed a scalar scheduling regression introduced by `e79799b1` (`Schedule
symbolic recursion like C++`). Shared sample-rate expressions are materialized
as stack `iTemp*`/`fTemp*` variables at their Hsched position, but the lowering
cache was still global. Two sibling `ondemand` blocks could therefore reuse a
temporary declared only in the first block, and FIR verification rejected the
second block's undeclared load.

The lowering cache now follows the compute-region stack. Ancestor values remain
visible in descendants, child values disappear when their guarded region
closes, and values emitted through an ancestor redirection remain in that
ancestor. A separate set preserves the global first-emission scheduling trace
without widening lexical value visibility.

History was checked on the exact `faustide-od` fixture: `e79799b1^` compiles it,
while `e79799b1` reproduces the undeclared `iTemp0`/`iTemp1` diagnostics. A
self-contained two-domain phase fixture now covers all four `-ss` strategies.
`bubble.dsp`, `gamme.dsp`, and `ondemand.dsp` all compile under `-ss 0..3`.
Formatting, warning-free workspace clippy, workspace/all-target tests, and all
versioned golden snapshots pass.

Known separate gap: the parent revision already shared a hash-consed startup
delay line between the two sibling domains, unlike C++ which allocates one line
per clock context. Correcting that runtime-parity issue requires changing the
delay plan and manager keys from signal-only identity to contextual
signal/domain identity; it was not conflated with this lexical-cache fix.
