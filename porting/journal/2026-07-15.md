# 2026-07-15 - Guarded-region cache isolation

Branch: `ondemand-vec-fad-synthesis`.

## Guard unsafe vector transports for recursive delay reads

Commit date: 2026-07-15
Source commit: this commit

Fixed a vector-mode runtime regression exposed by
`ba.pulse_countup_loop(4, 1) + 0.001` under `-vec -lv 1 -ss 3`: the checked
vector path could transport a delayed recursive carrier across vector loops,
precomputing chunk reads before the loop that writes the next recursive state.
The generated C++ then read stale or uninitialized short-delay storage and
could emit `-inf`.

The vector assembly now orders non-clocked short-delay copy-in phases before
all chunk loop bodies, and copy-out phases after them. More importantly, the
checked vector selector rejects cross-loop transports whose producer depends on
a delayed recursive carrier; those cases use the scalar fallback path until the
vector certificate can prove a fused serial loop for that pattern. Fallback
module construction now uses `ComputeMode::Scalar` rather than re-entering the
transitional vector module.

Added a `vector_mode` regression test for `pulse_countup_loop` across both
loop variants and all four scheduling strategies. Validated with:

- `cargo fmt --all`
- `cargo test -p transform signal_fir::loop_graph::tests -- --nocapture`
- `cargo test -p compiler --test vector_mode -- --nocapture`
- faustlibraries target
  `tests/output-rs-cpp/pulse_countup_loop_test__vec1_ss3.out` using the local
  `faust-rs` binary, which now reports no differences within tolerance.

## Isolate scheduled temporaries between sibling clock regions

Commit date: 2026-07-15
Source commit: this commit

Fixed a scalar scheduling regression introduced by `e79799b1` (`Schedule
symbolic recursion like C++`). Shared sample-rate expressions are materialized
as stack `iTemp*`/`fTemp*` variables at their Hsched position, but the lowering
cache was still global. Two sibling `ondemand` blocks could therefore reuse a
temporary declared only in the first block, and FIR verification rejected the
second block's undeclared load.

The lowering cache now follows the compute-region stack. Ancestor values remain
visible in descendants, child values disappear when their guarded region
closes, and values emitted through an ancestor redirection remain in that
ancestor. A separate set preserves the global first-emission scheduling trace
without widening lexical value visibility.

History was checked on the exact `faustide-od` fixture: `e79799b1^` compiles it,
while `e79799b1` reproduces the undeclared `iTemp0`/`iTemp1` diagnostics. A
self-contained two-domain phase fixture now covers all four `-ss` strategies.
`bubble.dsp`, `gamme.dsp`, and `ondemand.dsp` all compile under `-ss 0..3`.
Formatting, warning-free workspace clippy, workspace/all-target tests, and all
versioned golden snapshots pass.

Known separate gap: the parent revision already shared a hash-consed startup
delay line between the two sibling domains, unlike C++ which allocates one line
per clock context. Correcting that runtime-parity issue requires changing the
delay plan and manager keys from signal-only identity to contextual
signal/domain identity; it was not conflated with this lexical-cache fix.
