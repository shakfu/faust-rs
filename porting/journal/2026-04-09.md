## refactor(cranelift-ffi): make Cranelift export headers self-contained like interp

Commit date: 2026-04-09

Source commit: pending

### Problem

The Cranelift FFI export headers did not match the structure used by the
interpreter backend:

- `cranelift-dsp-c.h` still read like a scaffold placeholder and depended on
  `faust/gui/CInterface.h` for core callback glue declarations
- `cranelift-dsp.h` included `cranelift-dsp-c.h` instead of embedding the C API
  declarations it wraps

That made the Cranelift headers less self-contained than
`interpreter-dsp-c.h` / `interpreter-dsp.h`, and the C++ wrapper kept an extra
header dependency that the interpreter wrapper no longer needs.

### Refactor

Updated:

- `crates/cranelift-ffi/include/cranelift-dsp-c.h`
- `crates/cranelift-ffi/include/cranelift-dsp.h`
- `JOURNAL.md`
- `porting/journal/README.md`

Reworked the Cranelift headers to follow the interpreter layout:

- `cranelift-dsp-c.h` now defines `FAUSTFLOAT`, `UIGlue`, `MetaGlue`, and the
  opaque C API handle types directly, with the exported functions grouped by
  section (`bitcode`, source constructors, cache, instance lifecycle, UI/meta,
  compute)
- `cranelift-dsp.h` is now self-contained and embeds the C API declarations it
  needs instead of including `cranelift-dsp-c.h`
- kept the Cranelift-specific surface unchanged, notably:
  `opt_level` on source constructors, signal/box constructor placeholders, and
  Cranelift foreign-function registration
- aligned comments, include guards, and section structure with the interpreter
  headers for easier side-by-side maintenance
- fixed the C++ helper that converts `getAllCCraneliftDSPFactories()` so it also
  frees the returned `char**` container after copying entries
- followed up with a four-header documentation pass across
  `interpreter-dsp-c.h`, `interpreter-dsp.h`, `cranelift-dsp-c.h`, and
  `cranelift-dsp.h` to align constructor notes, MT cache helpers, and stale
  wrapper comments while keeping `FromSignals` / `FromBoxes` out of the
  self-contained Cranelift export headers for now
