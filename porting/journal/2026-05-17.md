# 2026-05-17

## Share RAD local rule classification

Commit date: 2026-05-17
Source commit: this commit (`Share RAD local rule classification`)

Added `signals::ad_rules` as a backend-neutral classification table for local
reverse-AD math rules. The table covers unary math nodes, binary math nodes,
and primitive `SIGBINOP` arithmetic versus discrete zero-gradient operators.

`propagate::reverse_ad` now uses the shared table to collect active math
children and dispatch symbolic local adjoint formulas. The symbolic `pow`
base derivative was aligned with the FIR/BRA lowering to use
`y_bar * y * pow(x, y - 1)`, avoiding the fragile `x^y / x` form at `x = 0`.

`transform::signal_fir::module` now uses the same shared table to dispatch
`BlockReverseAD` local math adjoints while keeping tape loads, FIR math helper
registration, adjoint carries, and loop scheduling local to the FIR lowerer.
The former per-node math arms in both RAD dispatchers were removed so the
shared classification is the single dispatch path for those local rules.
The second cleanup slice moved the pure local formulas themselves behind a
small `RadFormulaBuilder` trait, so Signal IR and FIR lowering now share the
operator algebra while still supplying their own value representation and tape
loading policy.
`ad_rules.rs` now documents that this shared algebra is the narrow exception to
`signals` being only a representation crate, and unit tests check formula
shapes directly for `pow`, unary rules, `min`/`max`, and `BinOp::Div`.
Follow-up documentation tightened the local-rule boundaries, the `primal`
argument contract, the natural `select_nonzero` semantics, and the FIR adapter's
responsibility split from tape loading.

Validation:

- `cargo test -p propagate reverse_ad --test core_api`
- `cargo test -p transform`
- `cargo clippy -p signals -p propagate -p transform --all-targets -- -D warnings`

## Plan RAD local rule factorization

Commit date: 2026-05-17
Source commit: this commit (`Plan RAD local rule factorization`)

Documented a limited factorization plan for reverse-mode AD local math rules.
The plan keeps the existing Signal-IR symbolic RAD pass and FIR
`BlockReverseAD` lowering separate, while sharing backend-neutral rule
classification through `signals`.

The plan explicitly avoids moving tape allocation, sweep scheduling, FIR type
selection, or backend helper emission out of `transform`.

Validation:

- Documentation-only change; not run.
