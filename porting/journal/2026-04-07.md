# 2026-04-07

## fix(signal_fir): fall back to circular buffer when shift signal exceeds max_copy_delay

Commit date: 2026-04-07

### Problem

`par_fir_32.dsp` (32-tap FIR filter) produced wrong samples.  The filter uses a
single input `x` delayed by 1 through 31 samples.  Since `max_copy_delay = 16`,
delays 17–31 select the `CircularPow2` strategy.  The pre-scan
(`prepare_delay_lines`) therefore allocates a single `CircularPow2` buffer of
size 64 for the carried signal `x`.

When processing `x@1` (rewritten to `Delay1(x)` by `signal_prepare`),
`lower_shift_delay1` retrieved the already-allocated buffer via
`ensure_delay_line_decl(x, 1)`.  It received the `CircularPow2` line but
proceeded to apply Shift-strategy operations on it:

```
buf[0] = x;               // write at index 0 — wrong for circular buffer
buf[63] = buf[62]; ...    // shift loop — trashes circular slots
output = buf[1];          // reads wrong slot
```

All other accesses to that buffer (for delays 1–16 via `lower_fixed_delay` and
17–31 via the circular path) used `fIOTA`-based indexing, so the shift clobber
made every delay read incorrect.

### Fix

Added a strategy guard at the top of `lower_shift_delay1`:

```rust
if !matches!(line.strategy, DelayStrategy::Shift) {
    let init = self.zero_value_for_signal(node)?;
    return self.lower_delay_state(node, value, init);
}
```

When the pre-scan allocated a `CircularPow2` (or `IfWrapping`) buffer for the
carried signal, `lower_shift_delay1` now delegates to `lower_delay_state`, which
uses `fIOTA`-based read/write consistent with all other accesses on that buffer.

### Why this is the right boundary

`lower_shift_delay1` is only called when `max_copy_delay >= 1` and the signal is
not a recursion projection.  That check prevents using Shift for signals that
need circular geometry from a higher-level perspective, but it does not account
for the case where the *same* signal has a larger delay elsewhere that already
forced a circular allocation.  The new strategy guard closes that gap.

### Tests

All workspace tests continue to pass (`cargo test --workspace`).
