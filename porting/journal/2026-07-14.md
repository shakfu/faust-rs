# 2026-07-14 - Verified vector FIR routing

Branch: `ondemand-vec-fad-synthesis`.

## Add the P5.1 routed-FIR boundary

Commit date: 2026-07-14
Source commit: this commit

Implemented the additive P5.1 routing boundary in
`signal_fir::vector_route`. `VectorRouteSession` accepts only an opaque
`VerifiedVectorPlan`, applies the selected common `-ss` schedule per epoch,
materializes loop regions, and replaces the former global value-cache model
with separate control, owned-loop, and loop-local inline scopes. All typed
transport arrays are declared from the frozen plan before routing. Owned
cross-loop values can only emit and resolve the exact planned store/load pair;
no lookup can invent a buffer or stable name.

Added independent routed-FIR verification for definition coverage, legal region
visibility, FIR value types, exact transport identities and declarations,
producer/store and consumer/load linkage, and the canonical `i0 - vindex`
chunk index. Focused tests cover successful two-loop routing, all four `-ss`
strategies, control and inline scope rules, sibling-definition rejection,
missing loads, mutated load FIR, and substituted direct-use values. The public internal API is adapted from
C++ `DAGInstructionsCompiler`: plan construction owns allocation while routing
only resolves already-certified identities. It has no CLI or ABI effect.

The new tests exposed a P4.4 panic when an output root had already been visited
inline from an earlier root. Production construction now promotes that signal
to the shared root loop and revisits it to record both contexts; a real-signal
non-regression test covers the case. The route remains intentionally detached
from `build_module` until P6 supplies complete delay, recursion, clock, and AD
state transitions. Per-region CSE, bounded effect-event checking, actual
signal-expression routing, vectorization retention, and backend activation
remain later P5/P6 work.

Validation is fully green: `cargo fmt --all -- --check`;
`cargo clippy --workspace --all-targets -- -D warnings`;
`cargo test --workspace --all-targets`; both focused route/plan test modules;
and all 190 `cargo run -p xtask -- golden-check` snapshots unchanged.
