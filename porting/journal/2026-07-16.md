## Elide redundant pure C-family drops

Commit date: 2026-07-16
Source commit: this commit

The checked vector lowerer uses `Drop` statements as temporary FIR roots while
it materializes and verifies shared expression DAGs. The C and C++ emitters
were rendering every one as `(void)(...)`, so a vector APF build contained
dead constants, state reads, temporary loads, and pure arithmetic statements.

The shared C-family emitter now elides a `Drop` only when its FIR value is
structurally known to be side-effect-free. The proof is deliberately
conservative: foreign and math calls, tee writes, allocations, and unknown
nodes remain emitted. This removes the noisy `fSampleRate`, `6.283...`, and
`fVec*Temp` examples without changing FIR construction or the other backends.

Validated with the C-family unit suite and a conservative-purity regression
test, plus native C++ APF `-vec -lv 1` comparison against the 60,000-sample
reference oracle.

## Audit and correct scalar scheduling cost

Commit date: 2026-07-16
Source commit: this commit

Completed vector coverage recovery Phase 6. Release binaries built from the
parents of the authoritative `-ss` and vector-option commits established that
`reverb_designer` was already evaluation-bound (about 7.1 seconds of a 7.2
second compile) before either feature. The current release remains within 1.5%
of those historical totals, so the costly scalar frontend is not caused by
vectorization.

The audit did find and remove a genuine generic scalar scheduling overhead.
Scalar lowering had built vector occurrence/condition facts although it only
needs direct effects to orient conflicts. It now derives a compact direct-effect
map, and effect orientation groups nodes by resource instead of checking every
pair of stateful nodes. Unknown/impure foreign calls retain their complete
ordering barrier. On `reverb_designer`, scalar effect analysis and orientation
fell from approximately 140 ms and 157 ms to 2 ms and 8 ms respectively; the
whole signal-to-FIR phase is now about 136 ms.

The compiler's `-time` report now splits FIR planning, verified
preparation/normalization, hierarchical graph construction, scalar effects,
effect orientation, scheduling, vector certification, and lowering. Scalar
mode is structurally tested to retain `NotRequested` vector status and no
fallback detail. Direct scalar effects are compared against full vector facts,
and the foreign-barrier ordering test remains in place.

Validated with two 15,000-sample C++ impulse-oracle comparisons for
`reverb_designer.dsp` and `cubic_distortion.dsp` under scalar `-ss 0`, the
focused structural tests, formatting, clippy, workspace tests, and golden
snapshots. Full measurement method and remaining verified-preparation cost are
documented in `porting/scalar-compilation-history-and-cost-audit-2026-07-16-en.md`.

## Add checked vector-retention gates

Source commit: this commit

single/double precision, portable paths, and stable fallback reason codes. Its
complete 16-cell matrix (precision × `-lv` × `-ss`) is stored under

this guard, followed by an interpreter metamorphic test.

`vector-interp-opt-check` exercises delayed recursion (`APF`), structural
recursion (`capture`), `prefix` state, and UI lifecycle (`UITester`) under both
vector layouts and all four strategies. All 32 optimized/unoptimized checked
vector traces matched.

## Certify symbolic recursion carriers and state-mediated delay routing

Commit date: 2026-07-16
Source commit: this commit

Completed vector coverage recovery Phase 4. The decoration certificate now
identifies `SYMREC` and `SYMREF` as checked structural carriers, and the vector
plan preserves that fact explicitly. These tuple records keep authoritative
analysis facts but emit no standalone FIR value; separately planned scalar
projection bodies remain executable. Mutation tests reject a forged carrier
fact, and the independent plan/route checks continue to require definitions
for every non-structural signal. This avoids evaluating recursion bodies in a
carrier's unrelated loop and removes false cross-loop transports.

Variable delays whose certified amount interval can include zero now treat
their carrier dependency as state-mediated. The plan does not synthesize a
value transport that scalar lowering never performs. A crossing is accepted
only when both loops belong to one checked fused serial group; otherwise it
fails closed with an explicit `UnfusedImmediateDelayCrossing` diagnostic.

The 93-DSP corpus moved from 44 to 49 certified files. The additions are
`capture`, `echo`, `karplus`, `quadecho`, and `stereoecho`. The residual
waterfall is 26 event-bound fallbacks, 13 specific unfused-delay plan
fallbacks, three intentional pure-lowering fallbacks, one soundfile UI
fallback, and the existing SIGGEN error. `smoothdelay` now clears routing and
state verification and reaches the bounded-event rejection (4,967 events
versus the 4,096 limit).

All five new files passed the 60-case native C++ matrix: scalar `-ss 0..3` and
vector `-lv 0/1 x -ss 0..3`, each over 60,000 frames. The 120 focused vector
tests pass, including `BlockReverseAD` separation and colocated recursive
projection fixtures. `reverb_designer` profiling measured 60--61 seconds for
the scalar frontend, 5.83 seconds for the complete vector plan, and 67.0
seconds total in the reproducible timed vector run; no stage-level regression
from this slice was found.

The full workspace gate also exercised clocked recursion. Structural carriers
aggregate state effects but own no execution loop, so clock/state ownership is
verified on their executable projection bodies; the `clock-state` fixture is
certified under all four scheduling strategies.

Validated with:

- a complete `count_vector_corpus` sweep over all 93 DSPs;
- 60 native C++ impulse comparisons for the five newly certified files;
- `cargo fmt --all -- --check`;
- `cargo clippy --workspace --all-targets -- -D warnings`;
- `cargo test --workspace --all-targets`;
- `cargo run -p xtask -- golden-check`.

## Certify read-only vector tables and indirect recursive paths

Commit date: 2026-07-16
Source commit: this commit

Continued vector coverage recovery Phase 4 with checked read-only generated
tables. Finite waveform, literal, and pure computed generators are interpreted
at compile time and materialized as immutable typed declarations; `rdtable`
uses them with scalar-compatible bounds. Mutable table writes and unknown
foreign-function effects remain fail-closed. Generic routed-definition and CSE
identity handling was also corrected without consulting corpus filenames.

The 93-DSP counter moved from 27 to 44 certified paths. The 17 new files are
`BPF`, `math_simp`, `midi_tester`, `osc`, `phasor`, `pow`, `priority`,
`reverb_tester`, `select2`, `spat`, `switcher`, `table`, `vumeter`, `waveform2`,
`waveform3`, `waveform4`, and `waveform6`. The remaining waterfall is 31 event
fallbacks, 16 specific pure-lowering fallbacks, one soundfile UI fallback, and
the pre-existing SIGGEN error.

Native sample comparison found a generic three-loop delayed-recursion defect:
a delayed read could reach its recursive writer through an intermediate loop
that was outside the fused serial group. The producer now closes every
same-sample data path from delayed read to writer and records every internal
transport. The independent checker reconstructs data-only reachability and a
mutation test proves that removing the intermediate loop is rejected. Fused
groups are scheduling envelopes whose members inherit external dependencies;
the event certificate independently models their sample-major execution and
managed state transitions. A corpus-independent compiler fixture exercises the
indirect path under both loop variants and all four scheduling strategies.

Compile-cost profiling added a conservative event-count lower bound before
event materialization, reducing `spectral_level` from about 29.0 to 20.7
seconds. `reverb_designer` measured about 61.8 seconds scalar and 68.3 seconds
for a vector request. Stage timing attributes only about 0.3 seconds to the new
fused-group work; the complete pre-existing vector-plan stages cost about 5.8
seconds. Scheduling-envelope dependencies are precomputed once per epoch to
avoid a new repeated reconstruction cost. The dominant scalar frontend cost is
left visible for the planned Phase 6 historical audit.

Validated with:

- producer/checker tests for immutable generated tables and indirect fused
  paths, including forged-artifact mutation cases;
- scalar/vector interpreter bit-exactness over both loop variants and all four
  scheduling strategies;
- a complete `count_vector_corpus --json` sweep over all 93 DSPs;
- the 204-case native C++ impulse matrix for all 17 newly certified DSPs;
- `cargo fmt --all -- --check`;
- `cargo clippy --workspace --all-targets -- -D warnings`;
- `cargo test --workspace --all-targets`;
- `cargo run -p xtask -- golden-check`.
