## Factorize compiler CLI entry point

Commit date: 2026-06-01
Source commit: pending (recorded by the commit that includes this journal entry)

Completed the CLI extraction described in `porting/implementation_plan.md`.
`crates/compiler/src/main.rs` is now a small launcher that only creates the
64 MiB compiler thread and delegates execution to `cli::runner::run_main`.

New module layout under `crates/compiler/src/cli/`:

| File | Concern |
|---|---|
| `mod.rs` | CLI module wiring and local test module |
| `args.rs` | `clap` argument model, backend/error enums, legacy Faust-style argument normalization |
| `timer.rs` | compilation timing and timeout helper |
| `diagnostics.rs` | human/JSON diagnostic rendering and source-context helpers |
| `runner.rs` | CLI orchestration, output emission, backend dispatch, fixtures, architecture wrapping |
| `tests.rs` | former `main.rs` CLI tests moved with the implementation |

The extraction preserves the public CLI behavior: legacy aliases, `-lang`
selection, JSON/WASM companion output, diagnostic verbosity modes, FIR fixture
helpers, architecture wrapping, and the explicit large-stack launcher contract.
