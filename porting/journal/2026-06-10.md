## Validate slider/nentry init against [min, max] at eval time

Commit date: 2026-06-10
Source commit: this commit

The C++ compiler emits `ERROR : init = X outside of [min max] range in '...'`
when a slider or nentry init value lies outside its declared bounds. faust-rs
performed no such check, silently accepting invalid widgets.

Implementation updates:

- added `EVAL_SLIDER_INIT_OUT_OF_RANGE` (`FRS-EVAL-0006`) diagnostic code in
  `crates/errors/src/codes.rs`;
- added `EvalError::SliderInitOutOfRange { kind, label, init_bits, min_bits,
  max_bits }` variant in `crates/eval/src/error.rs`; float values are stored
  as `u64` bit-patterns so the enum remains `Eq`-derivable; `Display` and
  `IntoDiagnostic` decode them back to `f64` for the error message;
- added the range check in `eval_slider_like` in
  `crates/eval/src/ui_widgets.rs`: after all four numeric parameters are
  simplified to constants, their `f64` values are extracted via `match_box`
  and `init < min || init > max` triggers the new error, mirroring the C++
  `checkRange` call in `eval.cpp`.

Validation:

- `faust-rs t13.dsp` now reports the same class of error as `faust -lang cpp
  t13.dsp` for init values outside [min, max];
- `cargo test -p eval` — 72 tests pass;
- `cargo test -p compiler --test diagnostic_errors` — 19 tests pass.
