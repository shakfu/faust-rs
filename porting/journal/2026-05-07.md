# 2026-05-07

## Bridge typed IIR filters into RAD E1 transpose

Commit date: 2026-05-07

Source commit: 03b61a8

Added an internal `propagate::transpose_ad` bridge from typed scalar
`IirFilter` sections to the existing phase-E1 `DEBRUIJNREC` transpose
scaffold. The bridge lowers first-order/second-order `A/B/C/D` state-space
views into canonical recursive state-update groups, then reuses the current
cotangent-driven transpose path.

Direct third-order and higher IIR sections remain rejected with a structured
`UnsupportedIirOrder` diagnostic, preserving the biquad/first-order stability
constraint from the LTI filter intermediate-form plan.

Validation:

- `cargo test -p propagate transpose_ad::tests::iir_state_space_bridge`

## Add typed LTI filter extraction views

Commit date: 2026-05-07

Source commit: 39b8bd8

Added typed `FirFilter`, `IirFilter`, and private `StateSpace`/`A-B-C-D`
views in `crates/signals/src/filter_algebra.rs` as the first implementation
step for `porting/lti-filter-intermediate-form-plan-2026-05-06-en.md`.

The new API extracts validated typed views from C++-parity `sigFIR`/`sigIIR`
carriers and converts first-order/second-order scalar IIR sections to sparse
state-space rows. Direct order-3 and higher IIR conversion is rejected
explicitly to preserve the plan's numerical-stability constraint.

Validation:

- `cargo test -p signals filter_algebra`
