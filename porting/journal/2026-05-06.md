## docs(rad): analyze library IIR shapes for E1

Commit date: 2026-05-06
Source commit: this commit (`Analyze library IIR RAD E1 gaps`)

Mapped the next RAD E1 gap for standard Faust library IIR/biquad forms.

### Changes

- Added `porting/rad-lti-library-filter-analysis-2026-05-06-en.md`.
- Reproduced diagnostics for `fi.iir`, `fi.tf2`, and direct-form variants
  `tf21`, `tf22`, `tf22t`, and `tf21t`.
- Identified three separate blockers:
  - `fi.iir((1),(p,q))` uses `Delay(delay1(state), 1)` inside the recursive
    body, which is LTI but outside the current E1 extractor syntax.
  - `fi.tf2(1,0,0,p,q)` contains zero-multiplied feed-forward delay branches
    that RAD rejects before FIR simplification removes them.
  - direct-form variants that rewrite coefficients as `-a1`/`-a2` can break
    literal seed identity, producing zero requested gradients.
- Recommended a pre-RAD canonicalization from delayed single-lane IIR state to
  explicit companion state-space slots.

### Validation

- Local diagnostic runs with `cargo run -p compiler -- --dump-sig ...`.
- Local FIR shape checks with `cargo run -p compiler -- --dump-fir ... --signal-fir-lane fast`.
