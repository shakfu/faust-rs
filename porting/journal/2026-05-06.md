# 2026-05-06

## feat(rad): route LTI recursive drive adjoints

Commit date: 2026-05-06
Source commit: `16ccd3c`

Advanced public phase E1 from seed-independent recursion to the first active
recursive seed case: a seed in the direct drive term of an LTI recursive branch
now receives the reverse-time adjoint lane as its per-sample contribution.

### Changes

- The reverse sweep now records LTI recursive projection cotangents whenever
  the requested seeds occur inside the recursive group.
- After the ordinary sweep, recorded projection cotangents are grouped into
  `ReverseTimeRec` adjoint projections.
- Added drive-term adjoint routing for additive/subtractive/multiplicative/
  divisive feed-forward drive expressions that do not read the current
  recursive state.
- Added a regression proving an active recursive input seed returns a
  `Proj(_, ReverseTimeRec(...))` gradient lane.

### Remaining E1 Work

- Active feedback-coefficient gradients are still deferred because they require
  multiplying the reverse-time adjoint by the primal recursive state read as
  block-local data.
- Broader drive-term rule coverage can follow the normal feed-forward RAD rule
  table as needed.

### Validation

- `cargo test -p propagate active_drive_seed_inside_lti_recursion_returns_reverse_time_adjoint`
- `cargo test -p propagate propagate_reverse_ad_seed_independent_lti_recursive_body_succeeds`
- `cargo fmt --all`
- `cargo clippy -p propagate --all-targets -- -D warnings`

## feat(rad): accept seed-independent LTI recursive RAD

Commit date: 2026-05-06
Source commit: `d0c870b`

Made the first conservative public E1 behavior change: `rad(...)` now accepts
an LTI recursive primal when none of the requested seeds appears inside the
recursive group. In that case the recursive primal can be treated as a leaf and
all seed-gradient lanes are correctly zero.

### Changes

- Added seed-containment detection inside `ReverseADTransform`.
- Allowed seed-independent `recursive-linear-transpose` projections to pass
  active-subgraph collection as leaves.
- Recorded their cotangents in the recursive projection frontier for the later
  active-seed routing step.
- Updated the public propagation regression so
  `rad((2 : + ~ *(0.5)), x)` succeeds when `x` is unrelated to the recursion.
- Added a lower-level regression proving an active recursive input seed still
  reports the phase-E1 diagnostic.

### Remaining E1 work

- Use the recorded frontier to propagate reverse-time adjoints into active
  recursive inputs and LTI coefficients.
- Keep emitting per-sample seed-gradient contributions, with any block
  accumulation left explicit in DSP/host code.

### Validation

- `cargo test -p propagate propagate_reverse_ad_seed_independent_lti_recursive_body_succeeds`
- `cargo test -p propagate active_seed_inside_lti_recursion_still_reports_phase_e1`
- `cargo fmt --all`
- `cargo clippy -p propagate --all-targets -- -D warnings`

## feat(rad): build grouped recursive adjoint frontier

Commit date: 2026-05-06
Source commit: `445e63e`

Added the next phase-E1 bridge expected by the public RAD sweep: a frontier
helper that receives recursive projection cotangents, groups them by source
recursion, and returns ordered adjoint projections.

### Changes

- Added `build_lti_recursive_adjoint_projections`.
- The helper ignores non-eligible entries, preserving existing diagnostics for
  unsupported recursive families.
- Eligible projections that read the same LTI `DEBRUIJNREC` now share one
  `ReverseTimeRec` group.
- Added a regression proving frontier order is preserved and the shared
  reverse-time group carries the expected cotangents.

### Remaining E1 work

- Collect the projection-cotangent frontier from the actual reverse sweep.
- Merge returned reverse-time adjoint projections into per-sample gradients for
  active LTI coefficients/seeds.

### Validation

- `cargo test -p propagate lti_recursive_projection_frontier_shares_reverse_group`
- `cargo test -p propagate lti_recursive_adjoint_group_combines_slot_cotangents`
- `cargo fmt --all`
- `cargo clippy -p propagate --all-targets -- -D warnings`

## feat(rad): group LTI recursive adjoint cotangents

Commit date: 2026-05-06
Source commit: `f662f2a`

Continued phase E1 by adding the shared recursive-adjoint grouping primitive
needed before public `rad(...)` can lower recursive projections.

### Changes

- Added `build_lti_recursive_adjoint_group`, which accepts multiple
  `(slot, cotangent)` entries for one LTI `DEBRUIJNREC`.
- Duplicate slot cotangents are accumulated with signal addition.
- The single-projection bridge now delegates to the grouped helper.
- Added a regression for a two-lane recursive group with duplicate cotangents
  on one lane.

### Remaining E1 work

- Make the public RAD sweep discover all projection cotangents for a recursive
  group and call the grouped helper.
- Route reverse-time state adjoints into per-sample gradients for active LTI
  coefficients/seeds.

### Validation

- `cargo test -p propagate lti_recursive_adjoint_group_combines_slot_cotangents`
- `cargo test -p propagate lti_recursive_projection_builds_reverse_time_adjoint_projection`
- `cargo fmt --all`
- `cargo clippy -p propagate --all-targets -- -D warnings`
