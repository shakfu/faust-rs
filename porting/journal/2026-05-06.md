# 2026-05-06

## feat(rad): validate LTI recursive RAD runtime

Commit date: 2026-05-06
Source commit: `48d00a6`

Closed the first numeric runtime gap for phase E1 strict-LTI recursive RAD by
running `rad((2 : + ~ *(p)), p)` through the public compiler facade,
fast-lane FIR, and interpreter backend.

### Changes

- Added a compiler runtime regression for the closed-form recurrence
  `y[n] = 2 + p * y[n-1]`.
- Verified the gradient lane emits per-sample coefficient contributions
  `lambda[n] * y[n-1]`, with the adjoint recurrence evaluated backward over
  the current compute block.
- Fixed `ReverseTimeRec` construction to wrap the complete recursive group,
  preserving the `Proj(slot, group)` contract expected by signal preparation
  and FIR lowering.
- Classified reverse-time outputs structurally so products containing
  `Proj(_, ReverseTimeRec(...))` are scheduled in the reverse sample loop.
- Added block-local primal replay for `Delay1(primal)` by reading the already
  emitted forward primal output buffer in the reverse loop.
- Extended the interpreter backend with `LoadOutput`, corrected reverse
  simple-loop subtraction order, and kept multi-loop compute bodies in the
  I/O-capable DSP block.

### Remaining E1 Work

- Runtime coverage still targets the strict literal-coefficient LTI case; UI
  and input coefficients remain classified as E2 block-linear-time-varying.
- Wider LTI shapes such as multi-state and biquad-like recursions should get
  numeric fixtures next.

### Validation

- `cargo test -p codegen reverse_simple_for_loop`
- `cargo test -p codegen opcode`
- `cargo test -p transform reverse_time_rec`
- `cargo test -p propagate active_feedback_coefficient_seed_uses_primal_state_delay`
- `cargo test -p compiler fastlane_interp_lti_recursive_rad_feedback_coeff_matches_closed_form_contributions`
- `cargo fmt --all`
- `cargo clippy -p codegen -p transform -p propagate -p compiler --all-targets -- -D warnings`

## feat(rad): route LTI feedback coefficient adjoints

Commit date: 2026-05-06
Source commit: `50b50fa`

Completed the first strict-LTI feedback-coefficient routing path. For a
recursive branch such as `drive + p * y'`, a seed on `p` now receives the
per-sample contribution `reverse_adjoint[n] * delay1(y)[n]`.

### Changes

- Added a rewrite from current recursive-state references to causal primal
  state reads: `Proj(slot, DEBRUIJNREF(1))` becomes
  `delay1(Proj(slot, primal_group))` in coefficient-gradient terms.
- Extended LTI branch-adjoint routing for multiplication and division where one
  side reads recursive state and the other side contains an active seed.
- Added a regression proving an active strict-LTI feedback coefficient gradient
  contains both `ReverseTimeRec` and `Delay1` of the primal state.

### Remaining E1 Work

- Broader coefficient expression coverage can be added incrementally under the

## feat(rad): route LTI recursive drive adjoints

Commit date: 2026-05-06
Source commit: `16ccd3c`

Advanced public phase E1 from seed-independent recursion to the first active
recursive seed case: a seed in the direct drive term of an LTI recursive branch
now receives the reverse-time adjoint lane as its per-sample contribution.

### Changes

- The reverse sweep now records LTI recursive projection cotangents whenever
  the requested seeds occur inside the recursive group.
- After the ordinary sweep, recorded projection cotangents are grouped into
  `ReverseTimeRec` adjoint projections.
- Added drive-term adjoint routing for additive/subtractive/multiplicative/
  divisive feed-forward drive expressions that do not read the current
  recursive state.
- Added a regression proving an active recursive input seed returns a
  `Proj(_, ReverseTimeRec(...))` gradient lane.

### Remaining E1 Work

- Active feedback-coefficient gradients are still deferred because they require
  multiplying the reverse-time adjoint by the primal recursive state read as
  block-local data.
- Broader drive-term rule coverage can follow the normal feed-forward RAD rule
  table as needed.

Source commit: `d0c870b`

Made the first conservative public E1 behavior change: `rad(...)` now accepts
an LTI recursive primal when none of the requested seeds appears inside the
recursive group. In that case the recursive primal can be treated as a leaf and
all seed-gradient lanes are correctly zero.

### Changes

- Added seed-containment detection inside `ReverseADTransform`.
- Allowed seed-independent `recursive-linear-transpose` projections to pass
  active-subgraph collection as leaves.
- Recorded their cotangents in the recursive projection frontier for the later
  active-seed routing step.
- Updated the public propagation regression so
  `rad((2 : + ~ *(0.5)), x)` succeeds when `x` is unrelated to the recursion.
- Added a lower-level regression proving an active recursive input seed still
  reports the phase-E1 diagnostic.

### Remaining E1 work

  recursive inputs and LTI coefficients.
- Keep emitting per-sample seed-gradient contributions, with any block
