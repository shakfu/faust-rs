# 2026-04-06

## refactor(transform): extract Phase 1 placement analysis into `placement.rs`

Commit date: 2026-04-06

### Motivation

`signal_fir/module.rs` had grown to ~4 000 lines.  The Phase 1
variability-driven placement analysis — four tightly related items (`Bucket`,
`is_trivial_fir`, `analyze_signal_sharing`, `analyze_sig_rec`) — was scattered
near the top of the file among unrelated definitions.  Grouping them in a
dedicated module makes the conceptual boundary explicit and reduces the
cognitive load when reading either the analysis logic or the lowering engine.

### What moved

| Item | Kind | From | To |
|------|------|------|----|
| `Bucket` | enum | `module.rs` | `placement.rs` |
| `is_trivial_fir` | free fn | `module.rs` | `placement.rs` |
| `analyze_signal_sharing` | free fn | `module.rs` | `placement.rs` |
| `analyze_sig_rec` | free fn (private) | `module.rs` | `placement.rs` |

### What stayed in `module.rs`

The `impl SignalToFirLower` Phase 1 methods (`variability_of`,
`is_recursive_projection`, `materialize_in_bucket`) and the placement gate
inside `lower_sig` remain in `module.rs`.  They need mutable access to private
fields of `SignalToFirLower` (`store`, `constants_statements`,
`control_statements`, `const_counter`, `slow_counter`), which Rust's visibility
rules only allow within the defining module.

### Documentation

`placement.rs` carries a module-level doc comment that explains:

- the three execution tiers and their mapping to `Variability` variants,
- the role of each exported item,
- a pseudo-code sketch of the full placement gate so the split between the
  analysis data (in `placement.rs`) and the materialization methods (in
  `module.rs`) is self-evident.

`analyze_sig_rec` also received its first doc comment (previously undocumented).

### Unused import cleanup

`FirMatch` and `match_fir` were only used by `is_trivial_fir`; their import was
removed from `module.rs` after the move.

### Scope

- **New**: `crates/transform/src/signal_fir/placement.rs`
- **Modified**: `crates/transform/src/signal_fir/module.rs` — remove moved
  definitions, add `use super::placement::{Bucket, analyze_signal_sharing, is_trivial_fir}`,
  drop unused `FirMatch`/`match_fir` imports.
- **Modified**: `crates/transform/src/signal_fir/mod.rs` — declare `mod placement`.

### Validation
