# 2026-04-06

## docs(porting): design document for `DelayManager` refactor

Commit date: 2026-04-06

### Overview

Following the extraction of `delay.rs` (sizing/geometry layer), an architectural
analysis was conducted to determine whether the remaining stateful delay code in
`module.rs` could also be separated — and, if so, how to design that separation
to enable buffer-geometry extensibility.

The analysis established:
- Three `SignalToFirLower` fields are exclusively delay-owned and can move to a
  `DelayManager` struct (`delay_lines`, `rec_group_max_delay`,
  `scheduled_delay_writes`).
- Six other fields are shared with recursion/table/general lowering
  (`struct_declarations`, `clear_statements`, `clear_init_seen`,
  `sample_statements`, `uses_iota`, `scheduled_state_updates`) and cannot
  be exclusively owned by a delay component.
- `lower_fixed_delay` and `lower_delay_state` must stay in `module.rs` because
  they call `lower_signal()`, the recursive core of the lowering engine.

The proposed design introduces three new pieces in `delay.rs`:

1. **`DelayManager`** — owns the three delay-exclusive fields; scan methods
   (`scan_signals`, `try_record_rec_delay`, …) become its private methods,
   making them unit-testable in isolation from the FIR lowering engine.

2. **`DelayFirCtx<'a>`** — a borrow bundle assembled from disjoint fields of
   `SignalToFirLower` (Rust's field-level split borrows make this safe).
   Passed to `DelayManager` methods that need to emit FIR nodes.

3. **`DelayLineModel` trait** — abstracts the buffer geometry (size formula,
   write/read index expressions, counter bump) so that non-power-of-two
   models (`ModuloModel`, `LinearModel`, `SegmentedModel`) can be added
   without touching the lowering logic.

### Scope

- **New**: `porting/delay-manager-design-2026-04-06-en.md`

## refactor(transform): extract delay-line sizing analysis into `delay.rs`

Commit date: 2026-04-06

### Motivation

`signal_fir/module.rs` contained two conceptually distinct layers for delay
management:

1. **Sizing analysis** — pure functions that determine how large a circular
   buffer must be (`pow2limit_for_delay`, `delay_size_for_amount`, …).  These
   read only `arena` and `sig_types`; they have no FIR side-effects.
2. **FIR materialization** — stateful methods that allocate struct arrays,
   declare `fIOTA`, emit read/write index expressions, and schedule
   `instanceClear` loops.  These need mutable access to the full lowering
   engine state.

Extracting layer 1 into `delay.rs` isolates the pure circular-buffer geometry
from the FIR emission machinery, making both easier to read and test in
isolation.

### Why only the sizing layer could move

All FIR materialization methods (`prepare_delay_lines`, `scan_delay_lines`,
`lower_fixed_delay`, `ensure_delay_line_decl`, `ensure_iota_state`, …) are
`impl SignalToFirLower` methods that need mutable access to fields such as
`store`, `struct_declarations`, `clear_statements`, `sample_statements`, and
several dedup guards.  Rust's visibility rules require those methods to live
in the same module as `SignalToFirLower`'s definition, which is `module.rs`.
Making all those fields `pub(super)` purely for code organisation would weaken
encapsulation without real benefit.

### What moved

| Item | Kind | Description |
|------|------|-------------|
| `DelayLineInfo` | struct | Metadata for one allocated ring buffer (`name`, `size`) |
| `pow2limit_for_delay` | free fn | `next_power_of_two(delay + 1)` with overflow checks |
| `constant_delay_amount` | free fn | Extract a literal `SIGINT` delay amount |
| `variable_delay_max_bound` | free fn | Interval-analysis upper bound for variable amounts |
| `min_const_upper_bound` | free fn | Structural fallback: `SIGMIN(SigInt(n), _)` pattern |
| `delay_size_for_amount` | free fn | Unified resolver (tries all three strategies) |

### What stayed in `module.rs`

All stateful delay methods:
`prepare_delay_lines`, `scan_delay_lines`, `scan_delay_child`,
`try_record_rec_delay`, `lower_delay`, `lower_fixed_delay`,
`lower_delay_state`, `recursion_feedback_info`, `active_recursion_info`,
`ensure_state_slot`, `ensure_delay_line_decl`, `ensure_iota_state`,
`current_iota_index`, `delayed_iota_index`, `masked_delay_index`,
`bump_iota`, `register_clear_table`.

### Call-site updates

Four `self.method(…)` calls were replaced with free-function calls:

| Location | Before | After |
|----------|--------|-------|
| `scan_delay_lines` | `self.delay_size_for_amount(amount)` | `delay_size_for_amount(self.arena, self.sig_types, amount)` |
| `lower_delay` | `self.delay_size_for_amount(amount)` | `delay_size_for_amount(self.arena, self.sig_types, amount)` |
| `ensure_delay_line_decl` | `self.pow2limit_for_delay(delay)` | `pow2limit_for_delay(delay)` |
| `ensure_recursion_array_for_group` | `self.pow2limit_for_delay(total_delay)` | `pow2limit_for_delay(total_delay)` |

### Import cleanup

`check_delay_interval` was only used by `variable_delay_max_bound`; its import
was removed from `module.rs` and is now local to `delay.rs`.

### Documentation

`delay.rs` carries a module-level doc comment that explains:
- the power-of-two circular buffer model (`fIOTA`, masked index, ring buffer size),
- the recursion + delay merging optimisation,
- a two-column table mapping each function to its purpose,
- an explicit note on why FIR materialization stays in `module.rs`.

### Scope

- **New**: `crates/transform/src/signal_fir/delay.rs`
- **Modified**: `crates/transform/src/signal_fir/module.rs` — remove moved
  definitions, add `use super::delay::{DelayLineInfo, delay_size_for_amount, pow2limit_for_delay}`,
  drop `check_delay_interval` from `sigtype` import, update 4 call sites.
- **Modified**: `crates/transform/src/signal_fir/mod.rs` — declare `mod delay`.

### Validation

- `cargo build -p transform` — clean, no warnings.
- `cargo test -p transform` — 64 tests pass, 0 failures.

## refactor(transform): implement `DelayManager` — standalone delay-line component

Commit date: 2026-04-06

### Overview

Implemented the design described in
`porting/delay-manager-design-2026-04-06-en.md`.  The three delay-exclusive
fields previously scattered on `SignalToFirLower` are now owned by a new
`DelayManager` struct in `delay.rs`, with a `DelayFirCtx` borrow bundle
enabling FIR emission from within the manager and a `DelayLineModel` trait
abstracting buffer geometry.

### What moved to `delay.rs`

| Item | Kind | Notes |
|------|------|-------|
| `DelayManager` | struct | Owns `delay_lines`, `rec_group_max_delay`, `scheduled_delay_writes` |
| `DelayFirCtx<'a>` | struct | Borrow bundle for FIR-emitting manager methods |
| `DelayLineModel` | trait | Buffer geometry abstraction (buffer_size, write_index, read_index, bump) |
| `CircularPow2Model` | struct | Current power-of-two implementation |
| `scan_node` (private) | fn | Moved from `scan_delay_lines` in module.rs |
| `scan_child` (private) | fn | Moved from `scan_delay_child` in module.rs |
| `try_record_rec_delay` (private) | fn | Unchanged logic; now takes `arena` as parameter |

### Changes to `module.rs`

- **Replaced** three fields (`delay_lines`, `rec_group_max_delay`,
  `scheduled_delay_writes`) with one field: `delay: DelayManager`.
- **`prepare_delay_lines`**: simplified to call `self.delay.scan_signals(…)?`
  then iterate the returned `max_delays` map.
- **`ensure_delay_line_decl`**: thin delegate using explicit field-level split
  borrows to construct a `DelayFirCtx` (excluding `self.delay`) and call
  `self.delay.ensure_delay_line(carried, delay, &mut ctx)`.
- **`lower_fixed_delay`**: `scheduled_delay_writes.insert(value)` →
  `self.delay.schedule_delay_write(value)`.
- **`ensure_recursion_array_for_group`**: `self.rec_group_max_delay.get(…)` →
  `self.delay.rec_max_delay(var.as_u32(), index)`.
- **Removed**: `scan_delay_lines`, `scan_delay_child`, `try_record_rec_delay`,
  `register_clear_table` (all logic moved to `delay.rs` or superseded by
  `DelayFirCtx::register_delay_clear`).

### Rust split-borrow pattern

The key design challenge was allowing `self.delay` to be borrowed mutably
while `self.store`, `self.struct_declarations`, etc. are simultaneously
borrowed.  Rust's field-level split borrows solve this:

```rust
// In ensure_delay_line_decl:
let mut ctx = DelayFirCtx {
    store: &mut self.store,
    // ... other fields, NOT self.delay ...
    uses_iota: &mut self.uses_iota,
};
self.delay.ensure_delay_line(carried, delay, &mut ctx)
// ↑ self.delay borrowed mutably — disjoint from fields in ctx
```

This pattern works only when the context struct is constructed via an
explicit struct literal (field-level borrows are visible to the checker);
a helper method taking `&mut self` would re-unify all borrows.

### `uses_iota` stays on `SignalToFirLower`

`uses_iota` is shared between delay and recursion lowering paths
(`ensure_iota_state` is also called by `lower_proj` via recursion).
It is included in `DelayFirCtx` as `uses_iota: &mut bool` rather than
being exclusively owned by `DelayManager`.

- `cargo build -p transform` — clean, no warnings.
- `cargo test --workspace` — all tests pass, 0 failures.

## refactor(transform): extract Phase 1 placement analysis into `placement.rs`

Commit date: 2026-04-06

### Overview

Moved `Bucket`, `is_trivial_fir`, `analyze_signal_sharing`, and `analyze_sig_rec`
out of module.rs into a new signal_fir/placement.rs, with a module-level
doc that explains the three execution tiers, the role of each item, and
a placement-gate pseudo-code sketch bridging the analysis data (here) and
the materialization methods (still in module.rs).

### Why the impl methods stayed in `module.rs`

The `impl SignalToFirLower` Phase 1 methods (`variability_of`,
`is_recursive_projection`, `materialize_in_bucket`) need mutable access to
private fields of `SignalToFirLower`.  Rust's visibility rules require them
to live in the same module as the struct definition.

### Scope

- **New**: `crates/transform/src/signal_fir/placement.rs`
- **Modified**: `crates/transform/src/signal_fir/module.rs` — remove moved
  definitions, add `use super::placement::{Bucket, analyze_signal_sharing, is_trivial_fir}`,
  drop unused `FirMatch`/`match_fir` imports.
- **Modified**: `crates/transform/src/signal_fir/mod.rs` — declare `mod placement`.

### Validation

- `cargo build -p transform` — clean, no warnings.
- `cargo test -p transform` — 64 tests pass, 0 failures.
