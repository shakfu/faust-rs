## fix(ffi): add foreign-function helpers to C++ backend headers

Commit date: 2026-03-31
Source commit: working tree after `b74f0c8`

Completed the missing C++ FFI surface for backend foreign-function registry
helpers so the wrapper headers track the C APIs used by `ffunction(...)`
workflows. The interpreter header is self-contained, so it now redeclares the
underlying C entrypoints and exposes matching C++ convenience helpers. The
Cranelift header already included its C header, so the missing work there was
the C++ helper layer only.

  - add `registerCInterpreterForeignFunction`,

## fix(compiler): correct `--dump-cpp-from-fbc` CLI help text

Commit date: 2026-03-31
Source commit: working tree after `9430e15`

Corrected the `clap` help text for `--dump-cpp-from-fbc` so it matches the
actual backend behavior. The previous description claimed the flag emitted a
C++ wrapper that delegated to the `interpreter-dsp` runtime, but the
`generate_cpp_from_fbc` path emits self-contained native C++ with no
interpreter runtime dependency.

### Scope

- updated
  [crates/compiler/src/main.rs](/Users/letz/Developpements/RUST/faust-rs/crates/compiler/src/main.rs)
  to:
  - change the `--dump-cpp-from-fbc` help string to
    `Read interpreter .fbc text and emit self-contained native C++`
  - tighten the `--cpp-class-name` help text so it refers to generated C++
    class naming rather than an FBC "wrapper" surface

### Validation

- `cargo test -p compiler cli_parse_accepts_dump_cpp_from_fbc -- --nocapture`
- `cargo run -p compiler -- --help`
