## Clock domains P3 (slice 3) — per-domain IOTA for circular delay lines in blocks

Commit date: 2026-07-07
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

Third slice of roadmap P3: delay lines inside clocked blocks now advance
in **fire time** — the last big named rejection of the base port falls.

- **Name-parametrized circular cursor** (`circular_pow2.rs`): the cursor
  primitives (`cursor_load` / `cursor_current_index` /
  `cursor_delayed_index` / `cursor_advance`) take the field name;
  `GlobalCircularCursor` delegates with `fIOTA` (non-clocked programs
  bit-identical). `DelayLineInfo` gains `cursor: Option<String>`
  (`None` = global).
- **Cursor assignment pass** (`assign_clocked_delay_cursors`, replaces
  the slice-1 rejection): after delay planning, every `CircularPow2` line
  whose carrier's inferred domain is clocked gets the domain's
  `fIOTA_d<i>` field (C++ `declareRetrieveIotaName`, via the P2.3
  `DomainCounters` registry) through `DelayManager::set_line_cursor`.
  The guarded block appends `fIOTA_d<i> = fIOTA_d<i> + 1` to its own
  sample-end phase — the cursor advances once per fire, after all
  reads/writes of the tick. Inner `IfWrapping` lines keep a named
  rejection (per-line counters still bump at the top sample end).
- **Stricter recursion guard**: the old `uses_iota` delta check (blind
  when the outer program already used `fIOTA`) is replaced by a
  `global_cursor_reads` use counter bumped by
  `ensure_global_circular_cursor` — any circular recursion carrier
  reached from inside a block is now rejected deterministically until the
  per-domain cursor is wired into circular recursion storage.

Runtime validation (interpreter): with an always-true clock, an inner
`@(17)` (forced CircularPow2) matches the unclocked program sample for
sample; with a sparse clock (fires on even ticks) it matches a
fire-indexed reference model (the tap reads the value written 17 *fires*
ago, holds between fires). 19 tests in `ondemand_pipeline.rs`; full
workspace suite + 190 goldens green.

Remaining for full P3: per-domain cursors for circular *recursion*
carriers and inner IfWrapping lines, SR/UI-in-block policy, FIR-dump
structural tests vs plan §2.4, differential harness vs `8eebea429`.

## Clock domains P3 (slice 2) — integer ondemand, upsampling, downsampling

Commit date: 2026-07-07
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

Second slice of roadmap P3: all three wrapper kinds now lower and run.
`ensure_guarded_block` dispatches on a guard shape selected from the
wrapper kind and clock type (plan §3.8):

- **`BoolIf`** (boolean OD, unchanged): `if (clock != 0) { body }`.
- **`CountedLoop`** (integer OD and US): `for (lOd<i> = 0; lOd<i> < clock;
  lOd<i>++) { body }` via the generic FIR `SimpleForLoop`. The open block
  now carries an optional `LoopCtx { var, bound }` (stack parallel to
  `open_domains`), which the new `ZeroPad` generator reads:
  `ZeroPad(x, h)` inside a counted block lowers to
  `((lOd == bound - 1) ? x : 0)` — the US input is exposed on the **last**
  inner iteration only (C++ `generateZeroPad`).
- **`DsModulo`** (DS): `if (fDSCounter_d<i> == 0) { body }` with the
  every-tick post-code `counter = (counter + 1) % clock` — the P2.3
  `DomainCounters` registry is now wired into lowering (declare/retrieve
  by `ClockDomainId` + wrap-to-zero increment emitter).

Non-boolean real OD clocks and non-integer US/DS clocks get named
`FRS-SFIR-0007` rejections.

One semantic finding pinned as a test
(`domain_free_body_state_advances_at_outer_rate`): per the clock
calculus' least fixed point (plan §4.1), a recursion group whose
definitions touch **no domain-internal signal** stays at the audio rate —
`upsampling(1 : (+ ~ _))` advances its counter once per outer tick, only
its *value* is annotated into the domain. The emission honors this
faithfully through region redirection (the group's statements land in the
top region, outside the loop). Bodies that read their clocked inputs are
pulled into the domain and run per inner iteration, as the other runtime
tests verify: integer-OD accumulation ×3 per tick (OD inputs are *not*
padded), US running sum (zero-pad on last iteration + two inner
iterations), DS fire-every-2 hold pattern.

Runtime validation through the interpreter; 17 tests in
`ondemand_pipeline.rs`; full workspace suite + 190 goldens green.
Remaining for full P3: per-domain IOTA wiring into CircularPow2/IfWrapping
delay strategies inside blocks, SR/UI-in-block policy, FIR-dump structural
tests vs plan §2.4, differential harness vs `8eebea429`.

## Clock domains P3 (slice 1) — boolean ondemand guarded-block lowering, all backends

Commit date: 2026-07-07
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

First slice of roadmap P3 (synthesis §6, landing order §8 item 4): **boolean
`ondemand` compiles and runs end-to-end** — the first clocked programs to
execute through faust-rs.

- **New entry point** `compile_signals_to_fir_fastlane_clocked(...)`
  (threads the P0.2 `ClockDomainTable`); the compiler facade now routes all
  native lowering through it. For clocked programs it runs the P1 analyses
  as a pre-lowering gate (clk_env inference + hgraph partition + schedule;
  failures surface as the new `FRS-SFIR-0008` `ClockAnalysis` code), then
  activates the clocked lowering state. Programs without wrappers are
  strictly unchanged (190 goldens zero-diff).
- **Guarded-block emission** (`module/clocked.rs`, C++ `generateOD` /
  `CodeIFblock`): the wrapper lowers to `if (clock != 0) { body }` using the
  generic FIR `If`/`Block` statements (P2.1 vocabulary decision), built in a
  nested **child region frame** (`RegionTree` gained an open-child stack).
  Node generators per plan §3.8: `Seq(od, y)` → `CS(od); CS(y)`;
  `Clocked`/`TempVar` → passthrough (snapshot semantics fall out of
  evaluating inside the guard); `PermVar` → `fPerm<i>` struct field cleared
  to 0, assigned inside the block, loaded outside.
- **Emission model (adaptation)**: demand-driven lowering + **region
  redirection** instead of C++'s Hsched-driven walk — when lowering reaches
  a signal whose inferred domain is a strict ancestor of the open block's
  domain, appends are redirected to the ancestor's region, so external
  computations land *before* the guard statement. Recursive state inside the
  block works unchanged: carrier updates are emitted in the child region,
  so state advances exactly on fire.
- **Scope guards** (everything else stays a named `FRS-SFIR-0007`):
  `upsampling` / `downsampling` / integer `ondemand` (clock interval not
  provably ⊆ [0,1]); delay lines inside a domain whose strategy advances
  globally (`CircularPow2` via shared `fIOTA`, `IfWrapping` counters) —
  only shift-strategy delays and recursion carriers are legal inside a
  block until the per-domain IOTA (P2.3 counters) is wired into the
  strategies.
- **Backends**: no backend work was needed — the guarded block reuses the
  generic `If` statement already emitted by the shared C-family core,
  interp, wasm, and cranelift (the `if_wrapping` delay strategy precedent).
  Generated C++ matches the C++ reference shape (guarded `if`, `fRec`
  updated inside, `fPerm0` hold).
- **Runtime validation** (interpreter, input-driven boolean clock):
  hold/fire semantics (`y = 2x` at fire, hold otherwise) and
  accumulate-on-fire-only (`ondemand(+ ~ _)`) both match the sample-level
  reference recurrences (`crates/compiler/tests/ondemand_pipeline.rs`, now
  14 tests).

Remaining for full P3: counted-loop integer OD, US/DS blocks (per-domain
IOTA/DSCounter wiring), SR/UI-inside-block policy, FIR-dump structural
tests vs the plan §2.4 C++ captures, and the differential harness against
the `8eebea429` branch binary.

## Clock domains P2 — compute-region infrastructure in signal_fir

Commit date: 2026-07-07
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

Roadmap P2 (synthesis §6, landing order §8 item 3), in three slices:

- **P2.1 — design note** as the rustdoc of the new
  `signal_fir/module/region.rs`: the `Region` tree model (scalar
  `SampleLoop ⊃ GuardedBlock(OD/US/DS)`; vector
  `ChunkLoop ⊃ {LoopNode | Island ⊃ GuardedBlock}`), the single visibility
  rule ("a value computed in region R is reusable only in R and its
  descendants; cross-region reuse goes through named storage"), and the
  FIR-vocabulary decision: guarded blocks will reuse the existing generic
  `If`/`SimpleForLoop`/`Block` statements (vector doc §4 finding) — no new
  FIR node kinds.
- **P2.2 — diff-free refactor.** The flat `SamplePhases` accumulator of
  `SignalToFirLower` is replaced by a `RegionTree` whose current region owns
  the phased statement lists (`immediate`/`post_output`/`sample_end` live on
  as `RegionPhases`). All ~30 append sites across
  `arithmetic`/`core_lowering`/`bra`/`state`/`tables`/`ui_lowering`/`build`
  now go through `regions.current_phases_mut()` — the region API is the only
  compute-append surface (the P2 exit criterion). `build_module` flattens
  the current region per loop slice and `reset_sample_loop_state` opens the
  next sibling region (`SampleLoop` / `ReverseSampleLoop`), preserving the
  forward/reverse two-slice assembly exactly. Per-bucket CSE (`cse.rs`) and
  variability placement (`placement.rs`) are untouched: while the tree holds
  one region per loop, bucket and region coincide, so the visibility rule
  holds trivially; guarded child regions (P3) will narrow it.
  **Acceptance met: zero diff on all 190 goldens, full workspace suite
  green.**
- **P2.3 — domain storage classes.** New
  `signal_fir/delay/domain_counters.rs` (C++ `declareRetrieveIotaName` /
  `declareRetrieveDSName`, compile_scal.cpp): `DomainCounters` registry of
  per-clock-domain `fIOTA_d{id}` / `fDSCounter_d{id}` persistent struct
  fields, declared once and cleared to 0 (same idempotent pattern as the
  global `fIOTA`), **keyed by `ClockDomainId`**; increment and
  wrap-to-zero (`% modulo`, the DS firing guard) statement emitters for the
  P3 guarded-block post-code. Unit tests cover declare/retrieve idempotence,
  per-domain distinctness, clear-to-zero shape, and both increment shapes.
  The `PermVar` → cleared-field and `TempVar` → parent-region-local rules
  land with their P3 generators (they need the guarded blocks to exist).

## Clock domains P1 — clock-environment inference + hierarchical graph/schedule

Commit date: 2026-07-07
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

Roadmap P1 (synthesis §6, landing order §8 item 2): the two clock-domain
analyses, as new `transform` modules (the crate gains a dependency on
`propagate` for the P0.2 `ClockDomainTable`; no cycle).

- **P1.1 — `transform::clk_env`** (port of C++ `clkEnvInference.cpp`,
  638 lines, `8eebea429`). `C⟦sig⟧` for every reachable prepared signal,
  as a side map (no tree-property mutation). The C++ rule names are kept
  in comments for parity audits: `R_PROJ` (group hypothesis seeding, join
  of definition envs), `R_CLOCKED` (visibility check + lift), `R_CD`
  (clock computed outside, outputs exactly in the inner domain, literal-0
  exception, result = parent), `R_SEQ` (left dominates right, result
  left), table rules and `R_COMPOSITE` (join over children).
  `find_fixpoint` is the Jacobi-style Kleene iteration from `nil` with the
  C++ safety bound (1000); `is_ancestor_clk_env` / `max_clk_env` implement
  the chain-restricted order and join over `ClockDomainId` parent links —
  `maxClkEnv`'s partiality (the scoping rule) surfaces as the structured
  `Incomparable` error naming both domains and the offending signal, and
  every rule violation has its own named error variant. Inference runs on
  the prepared (SYMREC) forest; the `SIGCLOCKENV` token ids survive the
  staging-arena clone, so the propagation-owned table stays valid.
- **P1.2 — `transform::hgraph`** (port of C++ `DependenciesGraph.cpp` /
  `DependenciesUtils.cpp` / `DependenciesScheduling.hh`). Single DFS from
  the outputs dispatching each signal into the digraph of its inferred
  domain: `GraphKey::Top` + one subgraph per OD/US/DS wrapper
  (`needs_subgraph`); the clock stays outside as a block-precondition
  edge; strict-ancestor (`is_external`) dependencies become
  `wrapper → external` edges in the wrapper's own graph with the external
  placed in its own domain's graph. `get_signal_dependencies` ports the
  immediate/delayed split (`delay ≥ 1` places but does not order;
  `Seq(od, y)` depends only on `od`; `Prefix` state-reads its value).
  `audit_hgraph` (C++ `auditHgraph`) checks the partition property and
  backs a debug assertion in `build_hgraph`. `schedule` serializes each
  graph with one deterministic depth-first toposort on immediate edges
  (C++ `dfschedule`; pluggable `-ss` strategies deferred), reporting
  instantaneous in-graph cycles as causality errors. C++ `controls`
  hoisting is deliberately left to `signal_fir` placement until P3.

Tests: one unit test per inference rule plus fixpoint convergence /
least-solution and the incomparable diagnostic (`clk_env/tests.rs`, 9
tests); partition/schedule/external/cycle/delayed-edge coverage
(`hgraph/tests.rs`, 5 tests); and an end-to-end pipeline test on a real
`ondemand` program (compile → prepare → annotate → hgraph → schedule) in
`crates/compiler/tests/ondemand_pipeline.rs`. Full workspace suite + 190
goldens green.

## Clock domains P0 — guards & groundwork (indivisible change set)

Commit date: 2026-07-07
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

First implementation step of the 2026-07-07 synthesis
(`porting/ondemand-vec-fad-interleave-synthesis-2026-07-07-en.md` §6, flat
landing order §8 item 1). P0.1–P0.4 land together by design: no intermediate
state may compile silently-zero gradients.

- **P0.1 — `signal_prepare` survives clocked nodes.** `Clocked(env, y)` now
  has its own arm in `signal_prepare/verify.rs`: the clock-env child is
  validated as opaque (nil or `SIGCLOCKENV` token) and never visited as a
  signal; a token reached in signal position is a structural error. The same
  opacity rule was applied to the two other traversals that walked the env
  generically: `sigtype`'s `populate_reachable_types` (was the "type
  validation failed" front-door failure) and `normalize`'s promotion
  (`promote_signal` no longer promotes the env child). `signal_fir` now
  rejects the whole clocked family (`OnDemand`/`Upsampling`/`Downsampling`/
  `Clocked`/`Seq`/`TempVar`/`PermVar`/`ZeroPad` and the token) with the new
  dedicated `FRS-SFIR-0007` `ClockedNotLowered` code naming the construct —
  never a panic, never the generic `FRS-SFIR-0004`.
- **P0.2 — `ClockDomain` side table + instance uniqueness.** New
  `propagate::clock_domain` module: `ClockDomain { parent, kind, clock,
  wrapper_box, inputs }` in an append-only `ClockDomainTable`; the allocated
  `ClockDomainId` *is* the uniqueness token. `make_clock_env` no longer
  builds the C++ `(parent, nil, nil, box, inputs...)` cons tuple (whose nil
  `slotenv`/`path` made structurally identical instances collide under
  hash-consing — the C++ de Bruijn collision class, plan §3.4); it allocates
  a table entry and embeds only the opaque `SIGCLOCKENV(id)` leaf in the
  graph. New `SigMatch::ClockEnvToken` + `SigBuilder::clock_env_token` in
  `signals`. The table is exposed on `PropagateOutput` and
  `SignalCompileOutput` (`clock_domains`).
- **P0.3 — memoization cross-constraint recorded.** The propagate
  memoization plan (`cpp-propagate-eval-memoization-port-plan-2026-07-04`)
  now states `clock_env ∈ key` as a hard correctness requirement (not
  future-proofing) with a dedicated validation item (same box under two
  domains → distinct signals).
- **P0.4 — loud AD diagnostics at domain boundaries.** `forward_ad` no
  longer routes the clocked glue through the silent `zero_tangent`
  catch-all: an explicit arm records a `FadUnsupportedNode` boundary error
  (surfaced by `generate_fad_signals_multi` before any dual escapes) with
  the new `FRS-PROP-0004` diagnostic ("fad cannot yet differentiate across
  an ondemand/upsampling/downsampling boundary", with the Phase-A workaround
  in the help text). The RAD rejection now names the clocked construct
  (`ondemand`, `clocked wrapper`, `clock-domain boundary variable`, …)
  instead of kind `"other"`, with a tailored diagnostic note.

Exit criteria pinned in `crates/compiler/tests/ondemand_pipeline.rs` (9
tests): the three wrappers pass `signal_prepare` (incl. a recursive-state
body), `ondemand` alone reaches `signal_fir` and gets `FRS-SFIR-0007`,
two structurally identical `ondemand` instances get distinct domains,
`fad`-around and `fad`-inside-with-crossing-seed get `FRS-PROP-0004`, and
`rad`-around names the construct — the four cohabitation §4 rows, now with
no silently-zero-tangent path. The `signal_fir` lowering-coverage drift
guard gained a `ClockedRejection` category.

## Ondemand/vec/FAD/interleave consolidated synthesis (FR + EN)

Commit date: 2026-07-07
Source commit: this commit

Added `porting/ondemand-vec-fad-interleave-synthesis-2026-07-07-fr.md` and
its English twin `…-en.md` (kept in sync). The synthesis supersedes
`ondemand-vec-fad-implementation-roadmap-2026-06-10-en.md` as the tracking
surface; the three 2026-06-10 analysis documents remain the per-topic
references.

What it establishes:

- **State re-verified on `main-dev`**: nothing of roadmap P0–P9 has landed
  (clock env still traversed as a signal in
  `signal_prepare/verify.rs:257`; silent `zero_tangent` catch-all at
  `forward_ad.rs:1075`; `make_clock_env` still nil `slotenv`/`path`;
  RAD rejection still `kind: "other"`; no `clk_env` module, no
  `ComputeMode`). Four refactors since June moved the targets and lower
  the cost of P2 and P3.2/V5: `signal_prepare` split
  (mod/verify/rewrites), `signal_fir/delay/` per-strategy split, the W9
  `SignalToFirLower` decomposition, and the shared C-family emitter core.
- **Cross-plan constraint recorded**: the in-flight propagate memoization
  port must include the clock environment in its result-memo key
  (C++ does, `propagate.cpp:918-929`), or it would *create* the P0.3
  same-box-under-two-domains bug rather than inherit it.
- **New spectral track S1–S5** (from the `interleave` analysis in
  `RUST/INTERLEAVED.md`, formalized and corrected):
  `interleave(N, FX) = serialize_in : od(FX) : serialize_out` with
  `serialize_in = par(i, N, _@(N-1-i))` as pure sugar; the only missing
  brick is the zero-stuff output `↑₀`, implementable as **library sugar**
  `up0(H, y) = y * (H != 0)` for boolean periodic clocks (option A — zero
  new compiler primitive; option B `SigZeroVar` kept as fallback for
  native RAD transpose). Derived end-to-end latency is **N−1** (not the
  2N−1 of the initial analysis) with the phase-(N−1) clock and `@(j)`
  output delays — `interleave(N, id) = @(N−1)`, `N=2` recovers `mem`; to
  be locked by the S1 N=2 table. Overlap-add falls out of the same
  construction. FFT reuses `an.fftb`/`an.c_bit_reverse_shuffle`
  unchanged; the library's sliding FFT is the oracle. S1–S3 depend only
  on P3; differentiable STFT (S4) requires FAD Phase B (P5); `rad`-STFT
  at constant hop fits the YOLO LPTV transpose before the general P8
  tape.
- Updated flat landing order interleaving S1–S5 with P0–P9, consolidated
  risks (new: memoization×clock_env, FFT compile-time staging N=4→1024,
  unlocked phase convention), and the oracle-surface table.

Documentation only; no code behavior changed.
