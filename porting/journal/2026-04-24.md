## fix(eval): report recursive case divergence as eval stack overflow instead of aborting

Commit date: 2026-04-24

Source commit: pending

### Context

A recursive function fixture using ordinary Faust definition clauses,

`fact(1) = 1;`
`fact(n) = n * fact(n-1);`
`process = par(i, 3, fact(i));`

reaches `fact(0)` and therefore diverges during evaluation. The C++ reference
compiler reports `ERROR : stack overflow in eval`, but the Rust port still
recurred through case-application paths until the host thread aborted with a
native stack overflow.

### Fix

- Extended the evaluator's structural recursion guard to cover recursive
  application/case-dispatch paths in addition to `a2sb` lowering.
- Wrapped `apply_value_list_value(...)` and
  `apply_pattern_matcher_value(...)` in `enter_structural` /
  `leave_structural` so runaway recursive case evaluation fails with
  `EvalError::RecursionDepthExceeded` before the OS stack is exhausted.
- Lowered the default evaluator depth budget from the previous Rust-only 4096
  to 256 so library/test-thread execution also fails deterministically before
  the thread stack aborts.
- Updated the displayed evaluator error text to mirror the C++ wording:
  `stack overflow in eval (depth budget N)`.
- Added a targeted diagnostic regression test that compiles the exact
  factorial/par fixture on a 64 MiB worker thread and asserts the failure stays
  in eval stage with readable recursion guidance.

### Validation

- `cargo test -p compiler --test diagnostic_errors diverging_recursive_case_reports_eval_error_instead_of_aborting`
- `cargo test -p compiler --test signal_pipeline corpus_fad_lambda_recursive_seed_shares_recursion_across_primal_and_tangent`
- `cargo run -p compiler -- --dump-sig /tmp/fact_stack_overflow.dsp`
- `cargo fmt --all`

## fix(codegen): emit valid C/C++ infinity and NaN literals

Commit date: 2026-04-24

Source commit: pending

### Context

Comparing `faust` and `faust-rs` on local `faustlibraries/tests/` fixtures exposed a hard backend
generation bug on `MIN_test` from `maths_tests.dsp`.

`MIN_test = ma.MIN * 1e307;` overflows to positive infinity in single
precision. The Faust C++ reference compiler emitted `INFINITY`, while the Rust
backend generated invalid literals such as `inf.0f` in C and C++ because the
 shared float formatter appended `.0`/`f` blindly to non-finite values.

### Fix

- C++ backend:
  - `trim_float` now special-cases `NaN`/`Infinity` and emits `NAN`,
    `INFINITY`, or `-INFINITY`.
  - `Float32` emission now uses a dedicated helper so the `f` suffix is added
    only for finite values.
- C backend:
  - mirrored the same `NaN`/`Infinity` special casing and finite-only `f`
    suffix handling.
- Added targeted signal-fir-lane regression tests covering the overflowed
  constant source in both C and C++ backends.

### Validation

- `cargo test -p compiler --test signal_fir_lane fastlane_c_emits_valid_infinity_literal_for_overflowed_float_constant -- --exact`
