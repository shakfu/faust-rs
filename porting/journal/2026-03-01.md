### `eval` — Document and wire `pattern_matcher` into `apply_case_rules`

Commit date: 2026-03-01

#### Context

`crates/eval/src/pattern_matcher.rs` contained a full port of the C++ Graef tree-automaton
(`make_pattern_matcher` / `apply_pattern_matcher`) but was never compiled or called:
no `mod pattern_matcher;` declaration existed in `lib.rs`, and `apply_case_rules` used
an independent hand-written recursive `match_pattern` function instead.

#### Changes

**`crates/eval/src/pattern_matcher.rs`** — full Rustdoc added:
- Module-level doc: algorithm overview (incremental Graef), construction/matching phases,
  transition ordering invariant, C++ correspondence table.
- All public types documented: [`Path`], [`Rule`], [`TransKind`], [`Trans`], [`State`],
  [`Automaton`], [`make_pattern_matcher`], [`apply_pattern_matcher`].
- All private helpers documented: `is_cons`, `get_node_tag`, `is_box_pattern_op_binary/ternary`,
  `is_box_pattern_var`, `is_box_num`, `subtree`, `make_state`, `make_var_state`,
  `merge_state`, `merge_trans_var/cst/op`, `build_automaton_metadata`,
  `add_subst`, `apply_pattern_matcher_internal`.
- **Borrow bug fixed** in `apply_pattern_matcher`: `match_box(arena, assoc.id)` borrows
  `arena` as `&arena`; the subsequent `arena.intern_symbol()` requires `&mut arena`.
  Fix: collect the identifier name into a local `String` before calling `intern_symbol`.
- Nonlinearity check refactored to collect rule indices first, avoiding borrow aliasing
  between `automaton.states[s_idx].rules` and `env_out`.

**`crates/eval/src/lib.rs`**:
- Added `mod pattern_matcher;` declaration (module was present on disk but not compiled).
- `apply_case_rules` rewritten to use the automaton:
  1. Determine expected arity from the first rule (unchanged).
  2. Call `pattern_matcher::make_pattern_matcher(arena, rules_rev)` → `Automaton`.
  3. Allocate per-rule `Vec<Option<Environment>>` child scopes.
  4. Loop over consumed arguments: `pattern_matcher::apply_pattern_matcher(arena, &automaton, state, arg, &mut envs)`.
  5. In the final state, pick the first surviving rule, eval its RHS, apply rest args.
- Removed `match_pattern` (now dead — its role is fully covered by the automaton).

#### C++ parity

| Aspect | C++ (`evalCase` + `patternmatcher.cpp`) | Rust (after this commit) |
|---|---|---|
| Rule compilation | `make_pattern_matcher(rules)` once per call | Same — `pattern_matcher::make_pattern_matcher` |
| Matching loop | `for arg: apply_pattern_matcher(A, s, arg, C, E)` | Same — loop over `consumed` |
| Nonlinearity | Path-based subterm extraction + env check | Same — `subtree` + `env.lookup` |
| Variable binding | `pushValueDef` into env chain | `env.bind(sym_id, value)` into child scope |
| Rule priority | First rule in source order wins | Same — first `Some(env)` in `final_state.rules` |
| Pattern evaluation | Raw patterns (not eval'd before matching) | Same — automaton uses raw pattern trees |

### `tlib` — Unify `TagRegistry` + `SymbolInterner` into a single `SymbolTable` type

Commit date: 2026-03-01

#### Context

`TreeArena` contained two private structs — `TagRegistry` (for IR node-type tags) and
`SymbolInterner` (for user-defined Faust identifiers) — that were structurally identical:
same fields (`to_id: AHashMap<Arc<str>, u32>`, `to_str: Vec<Arc<str>>`), same `intern()`
and `name()` methods, with `SymbolInterner` having one extra `get()` (read-only lookup).
The duplication served no purpose; both are bidirectional `&str ↔ u32` interners.

#### Changes

**`crates/tlib/src/arena.rs`**
- Replaced `TagRegistry` and `SymbolInterner` with a single `SymbolTable` struct.
- `SymbolTable` exposes `intern()`, `get()`, and `name()` — the union of both former APIs.
- `TreeArena` fields unchanged: `tag_registry: SymbolTable`, `symbol_interner: SymbolTable`.
- Public `TreeArena` API unchanged (all `intern_tag`, `tag_name`, `intern_symbol`,
  `get_symbol`, `symbol_name` methods delegate identically).
- Net: −68 lines of duplicated code, +17 lines for the unified type.

### `eval` + `tlib` — SymId interning: `Box<str>` → `u32` for O(1) environment lookup

Commit date: 2026-03-01

#### Context

A performance opportunity identified in the previous session: environment bindings stored
`SymId = Box<str>`, making every symbol comparison O(len(name)). C++ achieves O(1) comparison
by using hash-consed `Tree*` pointer equality. The Rust equivalent is to intern symbol names as
dense `u32` IDs and compare integers.

#### Design decisions

**Two-method split** on `TreeArena` to avoid `&mut` borrow conflicts:

| Method | Borrow | Purpose |
|---|---|---|
| `intern_symbol(&mut self, name: &str) -> u32` | `&mut` | Bind path: assigns a new id if not seen before |
| `get_symbol(&self, name: &str) -> Option<u32>` | `&` | Lookup path: returns existing id or `None` — no side effects |
| `symbol_name(&self, sym: u32) -> Option<&str>` | `&` | Diagnostics: reverse lookup for human-readable output |

`get_symbol` returning `None` for an unknown name is a correct short-circuit: if the name was
never interned, it was never bound in any environment, so `env.lookup` would also return `None`.
Callers can skip the lookup and return `UndefinedSymbol` directly without touching `&mut self`.

This split is required because `eval_box`'s `Ident` arm has `name: &'a str` borrowed from
`arena` via `match_box(arena, expr)`. The `&self` method coexists with that borrow; `&mut` would
not.

#### Changes

**`crates/tlib/src/arena.rs`**
- Added `SymbolInterner` struct (parallel to existing `TagRegistry`):
  - `to_id: AHashMap<Arc<str>, u32>` + `to_str: Vec<Arc<str>>` for O(1) intern/reverse-lookup.
  - `fn intern(&mut self, name: &str) -> u32` — allocates one `Arc<str>` shared between both tables.
  - `fn get(&self, name: &str) -> Option<u32>` — lookup-only, `&self`.
  - `fn name(&self, id: u32) -> Option<&str>` — reverse lookup.
- Added `symbol_interner: SymbolInterner` field to `TreeArena` struct.
- Initialized in `with_capacities` alongside `tag_registry`.
- Added three public methods: `intern_symbol`, `get_symbol`, `symbol_name`.

**`crates/eval/src/lib.rs`**
- **`SymId`** type alias: `Box<str>` → `u32`. Doc updated with C++ parallel (hash-cons pointer
  equality), SIMD layout note (`Vec<(u32,u32)>` → 4 bindings per cache line), and lookup/intern
  split rationale.
- **`Environment::bind`**: `name: impl Into<SymId>` → `sym: SymId` (plain `u32`).
- **`Environment::lookup`**: `name: &str` → `sym: SymId`. Body: `*s == sym` (O(1) `u32` compare).
- **`Environment::lookup_local`**: `name: &str` → `sym: SymId`. Same O(1) compare.
- **`Environment::local_names`**: added `arena: &TreeArena` parameter; uses `arena.symbol_name(*sym)`
  to convert `u32` ids back to strings for diagnostics.
- **`Environment::visible_names`**: added `arena: &TreeArena` parameter; propagated to recursion.
- **`Environment::top_level_names`**: added `arena: &TreeArena` parameter; propagated to recursion.
- **`eval_process_with_stats`**: changed `env.lookup("process")` to
  `arena.get_symbol("process").and_then(|sym| env.lookup(sym))`.
- **`eval_box` Ident arm**: `env.lookup(name)` → `arena.get_symbol(name).and_then(|sym| env.lookup(sym))`;
  `env.local_names()` → `env.local_names(arena)` (and visible/top_level).
- **`eval_box` Abstr arm**: `ident_name(arena, arg)` returns `String`; added
  `let sym = arena.intern_symbol(&name)` before `scoped.bind(sym, arg)`.
- **`bind_definitions`**: added `let sym = arena.intern_symbol(&name)` before `lookup_local(sym)`
  and `bind(sym, bound)`.
- **`apply_list` Abstr arm**: added `let sym = arena.intern_symbol(&param_name)` before
  `scoped.bind(sym, arg)`.
- **`eval_iter_body`**: added `let sym = arena.intern_symbol(var_name)` before `scoped.bind(sym, ival)`.
- **`apply_case_rules` bindings copy loop**: `name.clone()` → `*sym` (u32 is Copy).
- **`match_pattern`**: changed `arena: &TreeArena` → `arena: &mut TreeArena` to allow
  `intern_symbol` for pattern variable names. Fixed structural matching borrow: nodes' child
  slices collected into `Vec<(TreeId, TreeId)>` before recursive `&mut` calls (releases
  `arena` borrow before recursion).
- **Module doc performance table**: updated `(Box<str>, u32)` (~24 bytes) → `(u32, u32)` (8 bytes);
  lookup column updated `O(d × n × len)` → `O(d × n)`.
- **`Environment` struct doc**: updated memory per binding note.

#### Performance summary

| Aspect | Before (Box<str>) | After (u32 SymId) | Improvement |
|---|---|---|---|
| Per-comparison cost | O(len) — up to 30+ byte compares | O(1) — one `u32 == u32` | ~10-30× |
| Memory per binding | ~24 bytes (`Box<str>` + TreeId + padding) | 8 bytes (`u32` + `u32`) | 3× |
| Cache lines per scope (32 bindings) | ~24 × 32 = 768 bytes (~12 lines) | 8 × 32 = 256 bytes (4 lines) | 3× |
| Layout | Non-uniform (heap alloc per name) | `Vec<(u32,u32)>` — SIMD-scannable | |
| C++ parallel | n/a | Hash-cons pointer equality — same O(1) | parity |

#### Borrow-checker notes

The key design constraint: in `eval_box`'s `match match_box(arena, expr)` arm for `Ident`, the
variable `name: &'a str` borrows from `arena` for `'a`. All operations in that arm must use
`&self` borrows only. `get_symbol(&self)` satisfies this; `intern_symbol(&mut self)` would not.

In `Abstr`, `Abstr(arg, body)` arms: `arg, body: TreeId` are `Copy`, so `BoxMatch` is fully
consumed after destructuring and the arena borrow from `match_box` ends. `intern_symbol(&mut)`
is then safe.

In `match_pattern`: structural matching collected children into `Vec<(TreeId, TreeId)>` *inside*
a block to release the `pn: &TreeNode` / `vn: &TreeNode` borrows before making recursive
`&mut TreeArena` calls.

### `eval` — Parity analysis and gap-fill: `Environment` vs C++ `environment.cpp`

Commit date: 2026-03-01

#### Context

A parity analysis was conducted between the Rust `crates/eval/src/lib.rs` `Environment`
implementation and its C++ counterparts (`compiler/evaluate/environment.hh/.cpp`).

The C++ environment is a **persistent linked list of `ENV_LAYER` tree nodes** stored in the
global hash-cons pool. Each definition is wrapped in a **closure object** —
`closure(expr, genv, visited, lenv)` — that captures the environment at definition time.
Key C++ primitives: `pushNewLayer`, `pushMultiClosureDefs`, `addLayerDef`,
`searchIdDef`, `pushEnvBarrier`, `copyEnvReplaceDefs`, `updateClosures`.

The Rust environment is an **imperative `Vec<(SymId, TreeId)>`** with an optional parent
pointer. Definitions are stored as bare `TreeId`s without closure capture, because the Rust
evaluator is a single-pass explicit-parameter function (`eval_box`) that always threads the
correct lexical scope by constructing child scopes (`push_scope()`) before evaluating any
sub-expression. This is semantically equivalent to closure capture for Faust's purely
lexically-scoped language.

#### Divergences identified

| Feature | C++ (`environment.cpp`) | Rust before this commit |
|---|---|---|
| Closure wrapping | `closure(expr, genv, visited, lenv)` | Bare `TreeId` — not needed (eager eval) |
| Barrier mechanism | `pushEnvBarrier` / `searchIdDef` stops at sentinel | Local `Environment::empty()` for patterns — equivalent |
| `copyEnvReplaceDefs` | Present (letrec env rewiring + `updateClosures`) | Not needed — flat `push_scope` is equivalent |
| **Redefinition check** | `addLayerDef` throws `faustexception` on conflict | **Missing** — silently shadowed |
| Profiling stats | `gGlobal->gStats.fEnvLayersPushed/fEnvLookups/fEnvLookupTotalDepth` | **Missing** |

The closure model, barrier mechanism, and `copyEnvReplaceDefs` are intentional Rust divergences
with equivalent semantics. The redefinition check and profiling stats were genuine gaps.
