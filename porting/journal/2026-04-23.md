## test(compiler): validate recursive forward-mode AD numerically

Commit date: 2026-04-23

Source commit: pending

### Context

`fad_bug1.dsp` raised a concern that the generated C++ for a seeded
self-recursive system was wrong because the AD lowering materialized a second
recursive state next to the exposed primal state.

The produced code does duplicate the primal recurrence inside the AD row, but
that alone is not evidence of a semantic bug. For a seeded recursion such as:

```text
y[n] = p * y[n-1] + 2
```

forward-mode AD legitimately needs both:

- the process-visible primal recurrence `y[n]`,
- and an AD-local primal shadow feeding the tangent recurrence
  `dy/dp[n] = y[n-1] + p * dy/dp[n-1]`.

What was missing was hard evidence across more difficult recursive families:
single-state feedback, nested recursions, multi-output recursive groups, and
genuine mutual recursion.

### Implementation

Added Rustdoc and regression coverage:

- `crates/propagate/src/forward_ad.rs`
  - documented the current recursive codegen model explicitly:
    generated AD code may duplicate primal recursion state while remaining
    semantically correct.
- `tests/corpus/`
  - added:
    - `fad_recursive_parametric_self.dsp`
    - `fad_recursive_nested_parametric.dsp`
    - `fad_recursive_multi_output_parametric.dsp`
    - `fad_recursive_mutual_crossed_parametric.dsp`
- `crates/compiler/tests/signal_pipeline.rs`
  - added structural acceptance tests for the new recursive corpus fixtures.
- `crates/compiler/tests/fad_recursive_runtime.rs`
  - added runtime checks through the interpreter fast lane:
    - closed-form recurrence validation for the `fad_bug1`-style self-recursive
      case,
    - central finite-difference validation for nested recursion,
    - central finite-difference validation for multi-output recursion,
    - central finite-difference validation for mutual recursion.

### Verification

- `cargo fmt --all`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo test --workspace --all-targets`

Outcome:

- the `fad_bug1` pattern is numerically correct under the current lowering;
- the current concern is redundancy/optimization, not a proven semantic error;
- the new recursive fixtures now guard against regressions in
  `crates/propagate/src/forward_ad.rs`.
