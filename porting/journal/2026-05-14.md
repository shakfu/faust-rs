## Align Julia numeric casts with C++ backend

Commit date: 2026-05-14
Source commit: this commit (`Align Julia numeric casts with C++ backend`)

Adjusted the Rust Julia backend to better match the C++ Faust Julia generator
for impulse-test coverage.

Real-valued casts inside `mydsp{T}` methods now emit `T(...)` instead of fixed
`Float32(...)` or `Float64(...)` constructors. This matches C++ Julia output for
UI-derived values such as `T(dsp.fVslider0)` and keeps the generated DSP body
parametric over the internal Julia real type.

Integer casts now truncate through Julia helpers matching Faust/C++ semantics.
`Int64` casts use `trunc(Int64, ...)`; `Int32` casts use a generated
`faust_wrap_int32(...)` helper so values outside the signed 32-bit range wrap
instead of raising Julia `InexactError`. This fixes noise generator recurrence
casts such as `13622895711870` while preserving float-to-int truncation for
cases such as `int(0.6)`. Static top-level real tables keep concrete
`Float32`/`Float64` constructors because they are emitted outside the `where
{T}` scope.

The Julia prologue also defines missing Faust math aliases used by the impulse
corpus:

- `fmod(x, y) = rem(x, y)`
- `atan2(y, x) = atan(y, x)`

Float literals now use Rust's shortest round-trippable formatting instead of a
fixed 15-decimal format. This preserves small constants such as the Faust noise
LCG scale `4.656612875245797e-10`, fixing the `grain3.dsp` Julia impulse drift
caused by emitting it as `0.000000000465661`.

Validation:

- `cargo fmt --all`
- `cargo test -p codegen julia --lib`
- `cargo test -p compiler julia --test signal_fir_lane`
- `faust-rs -double -lang julia dsp/noise.dsp` checked for `T(dsp.fVslider0)`
- generated and compared Julia impulse output against references for:
  - `grain3.dsp`
  - `noise.dsp`
  - `noiseabs.dsp`
  - `pow.dsp`
  - `priority1.dsp`
  - `math.dsp`
