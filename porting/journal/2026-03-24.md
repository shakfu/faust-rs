# 2026-03-24 — Module splitting: boxes and eval crates

## boxes crate split

`crates/boxes/src/lib.rs` was 2169 lines. Extracted into five focused modules:

| File | Contents | Lines |
|------|----------|-------|
| `tags.rs` | 80+ `pub(crate) const BOX_*_TAG` string constants | ~110 |
| `internals.rs` | All `node_*` constructor functions, `intern_tag`, `list4`, `slider_params4`, layout helpers | ~500 |
| `builder.rs` | `pub struct BoxBuilder` — public construction API, 1:1 with C++ `box*` constructors | ~400 |
| `matcher.rs` | `pub enum BoxMatch`, `pub fn match_box` — structural pattern-match decoder | ~280 |
| `dump.rs` | `pub fn dump_box` — deterministic diagnostic dump | ~75 |

`lib.rs` is now a 40-line crate root with `pub use` re-exports only.

`cargo check -p boxes -p box-ffi` and `cargo test -p boxes` pass with zero errors, zero warnings.

## eval crate split

`crates/eval/src/lib.rs` was 6523 lines. Extracted into four focused modules:

| File | Contents | Lines |
|------|----------|-------|
| `source_context.rs` | `SamplePrecision`, `CachedLoadedSource`, `EvalSourceContext` | 209 |
| `error.rs` | `EvalStats`, `EvalError` (19 variants), `Display`/`IntoDiagnostic` impls | 634 |
| `environment.rs` | `EvalValue`, `ClosureValue`, `PatternMatcherValue`, `EnvFrameKey`, `EvalCacheKey`, `EnvLayer`, `EnvStore`, `Environment` | 536 |
| `loop_detector.rs` | `LoopFrame`, `LoopDetector` + per-pass caches (pm_store, closure_store, eval_cache, symbolic_box_cache) | 269 |

`lib.rs` reduced to 4917 lines (eval functions + public API only).

`cargo check -p eval`, `cargo test -p eval`, and `cargo clippy -p eval -- -D warnings` pass with zero errors, zero warnings.

## Design notes

- `EvalValue` / `ClosureValue` / `PatternMatcherValue` / `Environment` form a mutually-recursive cluster and were kept in one module (`environment.rs`) rather than split further.
- All internal types use `pub(crate)` for cross-module access within the crate.
- The public API (`Environment`, `LoopDetector`, `EvalError`, `EvalStats`, `EvalSourceContext`, `SamplePrecision`) is re-exported from `lib.rs` unchanged — zero breaking change.
