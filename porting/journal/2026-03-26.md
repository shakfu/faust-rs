## fix(eval): fold numeric case arguments before matcher dispatch

Fixed the `faust-rs -pn encoder3D_test tests/hoa_tests.dsp` stack overflow from
`faustlibraries`, and the related `optimBasic3D_test` failure on the same 3D HOA
path. The overflow reduced to the local recursive `p/pcalcul` helper inside
`hoa.lib`, where Rust repeatedly selected the catch-all recursive `case` branch
instead of the `0/1/2` base cases.

### Root cause

Rust already simplified `case` patterns before compiling the matcher automaton,
but `apply_pattern_matcher_value(...)` still matched against the raw evaluated
argument tree. In the HOA selector path, the first argument to `pcalcul(...)`
was a residual arithmetic box like `((l != 0) & ...) + ... * 2 + ... * 3`
instead of a folded numeric literal.

Because the matcher only saw that unfurled arithmetic tree, it could not match
the constant branches `0`, `1`, or `2`, so only the catch-all recursive rule
remained active. That sent `p(...)` back through `pcalcul(...)` indefinitely
until the Rust process overflowed its stack.

### Fix

- `apply_pattern_matcher_value(...)` now simplifies each consumed argument with
  `box_simplification(...)` before passing it to the pattern matcher.
- This restores the same effective behavior as the reference C++ compiler for
  numeric `case` dispatch: selector expressions are reduced to literal ints/reals
  before rule matching, so constant rules win before catch-all recursive clauses.

### Regression coverage

Added two eval tests:

- `eval_process_preserves_parser_case_rule_priority_for_recursive_local_cases` —
  self-contained regression covering the recursive `p/pcalcul` shape reduced
  from `hoa.lib`.
- `eval_process_parser_case_rule_priority_matches_cpp_reference` — file-backed
  differential test that compiles the same fixture with the C++ reference
  compiler when `FAUST_CPP_BIN` (or `/usr/local/bin/faust`) is available, then
  checks Rust eval on the same source.

### Validation

- `cargo test -p eval eval_process_preserves_parser_case_rule_priority_for_recursive_local_cases -- --nocapture`
- `cargo test -p eval eval_process_parser_case_rule_priority_matches_cpp_reference -- --nocapture`

## fix(eval): preserve captured environments for unapplied local case values

Fixed a second eval parity bug exposed by `faust-rs -pn u_transformer_test tests/wdmodels_tests.dsp`
and `u_transformerActive_test` against `faustlibraries`. The failing path passed a local
multi-clause function (`s`) through another higher-order function (`matrix`) before applying it.
Rust then raised `undefined symbol R0`, even though the reference C++ compiler kept `R0` and `R1`
captured correctly.

### Root cause

Rust already represented evaluated `case` values as `EvalValue::PatternMatcher`, but
`force_value_to_box(...)` still had one non-parity shortcut: for an unapplied matcher
(`state == 0`, no consumed arguments), it returned the original raw `case` tree instead of a
runtime `boxPatternMatcher(...)` node.

That shortcut dropped the captured matcher environments. As soon as a local `case` value escaped
its lexical site before first application, later calls re-entered a bare `case` tree with no
captured bindings, so free variables like `R0`/`R1` became undefined.

### Fix

- `force_value_to_box(...)` now always stores `EvalValue::PatternMatcher` in the side-table and
  reifies it as `boxPatternMatcher(key)`, even before the first application.
- This aligns the Rust representation with the reference compiler model where evaluated `case`
  values remain closure-like matcher runtime objects carrying their lexical environment.

### Regression coverage

Added two eval tests:

- `eval_process_preserves_unapplied_local_case_environment_through_higher_order_call` —
  in-memory regression for a local `transformer_scatter` case function passed through
  `apply_twice(...)`.
- `eval_process_unapplied_local_case_higher_order_matches_cpp_reference` — self-contained
  file-backed differential that compiles the same fixture with the C++ reference compiler when
  `FAUST_CPP_BIN` (or `/usr/local/bin/faust`) is available, then checks Rust eval on the same
  source.
