## fix(compiler): audit and harden WASM provenance tests against path-format drift

Commit date: 2026-04-02
Source commit: working tree after `c467580`

Audited the remaining compiler-side WASM provenance tests after the Windows CI
failure and converted the path-sensitive assertions to structured JSON checks.
The tests now deserialize `filename`, `library_list`, and
`include_pathnames` instead of depending on raw substring matches inside the
serialized JSON text.

### Scope

- updated
  [crates/compiler/src/lib.rs](/Users/letz/Developpements/RUST/faust-rs/crates/compiler/src/lib.rs)
  to add reusable test helpers for:
  - `filename`
  - `library_list`
  - `include_pathnames`
- updated WASM provenance-oriented tests to compare:
  - `filename` as a decoded string field
  - `library_list` entries as `PathBuf`
  - `include_pathnames` entries as `PathBuf`
- kept purely semantic JSON assertions (`ui`, `compile_options`, labels, etc.)
  unchanged because they are not path-format sensitive

### Validation

- `cargo test -p compiler --lib`
- `cargo fmt --all`
