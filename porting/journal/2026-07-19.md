## Serve cross-group symbolic back-edges with previous-sample state (X2b)

Commit date: 2026-07-19
Source commit: this commit

Completed X2b from
`porting/vector-plan-contexts-divergence-plan-2026-07-18-en.md` without changing
the prepared scalar signal tree. The P6.1 state producer now combines the
checked `Delayed { amount }` dependency with P4.4 placements: a recursive
projection whose alias and selected body have distinct owned loops gives the
body an effective history obligation of at least `amount`. The P6.1 checker
independently derives and verifies the same transition coverage and geometry.

The pure vector lowerer consumes that accepted body history for the matching
cross-loop `SIGPROJ(..., SYMREF(...))` instead of asking the route for a
current-sample value. The cross-loop placement and accepted transition are
both required; same-loop recursion retains its previous lowering. This guard
was necessary to avoid duplicating delay state and disabling the existing
lockstep register bundles for ordinary explicit `delay1` recursions.

The structural regression
`delayed_dependency_without_occurrence_delay_still_requires_history` pins the
P4.3b/P4.4-to-P6.1 mapping. The real library-level
`karplus_pass_through_backedge_is_certified_and_bit_exact` regression uses
`pm.ks(200, 0.5)` and passes certification plus scalar/vector interpreter
bit-exactness under both loop variants, all four scheduling strategies, and
vector sizes 32 and 24. The test runs on an explicitly enlarged test-thread
stack because expanding `stdfaust.lib` exceeds libtest's default thread stack
on some hosts.

Validation: `cargo fmt --all -- --check`, workspace clippy with warnings
denied, all-target workspace tests, the 385-test transform suite, the complete
33-test vector-mode integration suite, and `cargo run -p xtask -- golden-check`
are green. X3 remains the external/corpus qualification step.
