## Generate libfaust Box/Signal API matrices

Commit date: 2026-06-09
Source commit: this commit

Added an `xtask` workflow that generates parity matrices for the public
libfaust Box and Signal C APIs.

Implementation updates:

- added `cargo run -p xtask -- libfaust-api-matrix`;
- parsed `LIBFAUST_API` prototypes from `libfaust-box-c.h` and
  `libfaust-signal-c.h`;
- scanned Rust `*-ffi` crates for explicit and macro-generated C exports;
- generated `porting/generated/libfaust-box-c-api-matrix.md` and
  `porting/generated/libfaust-signal-c-api-matrix.md`;
- documented known adapted Box mappings for right shifts, `exp10`, soundfiles,
  and primitive predicate identity.

Matrix snapshot:

- Box C API: 178 exact candidates, 14 nearest Rust IR mappings, 0 missing.
- Signal C API: 7 exact candidates, 116 missing.

Validation:

## Plan libfaust Box/Signal API parity

Commit date: 2026-06-09
Source commit: this commit

Added `porting/libfaust-box-signal-api-parity-plan-2026-06-09-en.md` to define
a parity plan for the public libfaust Box and Signal C/C++ APIs.

Planning updates:

- documented the C++ reference headers and the expected Box/Signal API surface;
- summarized the current Rust crates involved in Box, Signal, and FFI exposure;
- proposed a shared FFI support layer for tree handles, strings, arrays, and
  `freeCMemory` ownership;
- split the work into Box API completion, Signal API introduction, headers,
  packaging, and export verification phases;
- listed open compatibility questions around shared context ownership, C++
  wrappers, AD carriers, primitive function identity, and signal interval
  metadata.

Validation:

- Documentation-only change; no code validation required.
