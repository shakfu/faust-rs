## Split Box logical and arithmetic right shift

Commit date: 2026-06-09
Source commit: this commit

Fixed the Box API representation gap where `CboxLRightShift*` and
`CboxARightShift*` both mapped through the same Rust `rsh` box primitive.

Implementation updates:

- added a distinct `BOXLRSH` box tag;
- added `BoxBuilder::lrsh` and `BoxMatch::LRsh`;
- preserved existing `BoxBuilder::rsh` / `BoxMatch::Rsh` as the arithmetic
  right-shift path;
- propagated `BoxMatch::LRsh` through flattening, complexity, eval, and signal
  propagation as `SigBuilder::lrsh`;
- changed `CboxLRightShift*` and `SOperator::kLRsh` to use the logical
  right-shift Box primitive;
- added a `box-ffi` structural non-regression test for distinct ARsh/LRsh tags;
- regenerated the Box/Signal API matrices.

Matrix snapshot:

- Box C API: 182 exact candidates, 10 nearest Rust IR mappings, 0 missing.
- Signal C API: 7 exact candidates, 116 missing.

- `cargo run -p xtask -- libfaust-api-matrix --cpp-root /Users/letz/Developpements/RUST/faust --out porting/generated`

## Extract shared tree FFI context support

Commit date: 2026-06-09
Source commit: this commit

Extracted the common Box/Signal FFI context mechanics into `crates/tree-ffi`.

Implementation updates:

- added `tree-ffi` as a workspace crate;
- moved shared opaque tree handle encoding/decoding into `FfiTreeContext`;
- moved context-owned C string pooling and null-terminated handle-array
  tracking into the shared crate;
- moved `SType`, `SOperator`, C string decoding, and null-safe out-pointer
  helpers into `tree-ffi`;
- migrated `box-ffi` to use the shared context support while keeping all public
  `Cbox*` and conversion symbols owned by `box-ffi`.

Architecture decision:

- chose the clean shared crate support layer over a new top-level `faust-ffi`
  context owner, so API crates can retain symbol ownership while sharing the
  low-level tree-handle contract.

Validation:

- `cargo fmt --all`
- `cargo check -p tree-ffi`
- `cargo check -p box-ffi`
- `cargo test -p tree-ffi --all-targets`
- `cargo clippy -p tree-ffi -p box-ffi --all-targets -- -D warnings`
- `cargo test -p box-ffi --all-targets`

## Plan libfaust Box/Signal API parity

Commit date: 2026-06-09
Source commit: this commit

Added `porting/libfaust-box-signal-api-parity-plan-2026-06-09-en.md` to define
a parity plan for the public libfaust Box and Signal C/C++ APIs.

Planning updates:

- documented the C++ reference headers and the expected Box/Signal API surface;
- summarized the current Rust crates involved in Box, Signal, and FFI exposure;

  metadata.
