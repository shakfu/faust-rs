## Implement RAD Phase B0 — `SigBlockReverseAD` carrier

Landed Phase B0 of
`porting/rad-block-reverse-ad-signal-ir-plan-2026-05-07-en.md`: the new
Signal-IR carrier exists, round-trips through the preparation pipeline,
and is structurally validated. No `rad(...)` user-visible behavior change
yet — phases B1+ (propagate dispatch, FIR lowering) follow.

Per-crate changes:

- `crates/signals`:
  - Added `SIG_BLOCK_REVERSE_AD_TAG = "SIGBLOCKREVERSEAD"`.
  - Added `BlockRevPolicy { TapeFull, Checkpointed, Recompute }` with
    `from_raw` / `to_raw` and Rustdoc explaining that only `TapeFull` is
    committed in B0.
  - Added `SigBuilder::block_reverse_ad(body, seeds, cotangents, policy)`
    with detailed Rustdoc on the projection contract, on-arena layout,
    block-local TBPTT semantics, and debug-mode assertions for body
    non-emptiness and body/cotangent length parity.
  - Added the `SigMatch::BlockReverseAD { body, primal_count, seeds,
    cotangents, policy }` variant and the corresponding decoder arm in
    `match_sig` (rejects malformed `primal_count` / `policy` ints).
  - New `core_api` round-trip tests: layout decoding, policy round-trip,
    distinct-policy interning.

- `crates/sigtype/src/rules.rs`:
  - New `SigMatch::BlockReverseAD` arm: returns a `Tuplet` of
    `body_components ++ seeds_components`, walking the cotangent list so
    constants stay typed for the future explicit-VJP successor.

- `crates/normalize/src/normalform.rs`:
  - New promotion arm: descends into `body`, `seeds`, `cotangents` lists
    via the existing list-traversal in `promote`, then rebuilds the
    carrier through `SigBuilder::block_reverse_ad` so subsequent passes
    observe a canonical layout.

- `crates/transform/src/signal_prepare.rs`:
  - Structural validation: `body_list` non-empty, `body.len() ==
    cotangents.len()`, `primal_count` matches body length, every child
    sub-tree is recursively verified.
  - Extended the `Proj` arm to recognise `Proj(slot, BlockReverseAD)`
    with arity = `primal_count + seeds.len()`, mirroring the
    `ReverseTimeRec` fast-path treatment but without re-routing through
    `match_sym_rec`.
  - New `signal_prepare/tests.rs` cases: end-to-end carrier preservation
    through `prepare_signals_for_fir`, and post-verification rejection
    of an out-of-range `Proj(7, BlockReverseAD)`.

- `crates/propagate/src/stateful_rad.rs`:
  - LTI/affine classifier returns `ExprClass::nonlinear()` for any
    `BlockReverseAD` carrier reached during classification, so the LTI
    bridge can never re-route a block fallback back into `ReverseTimeRec`.

in `crates/transform/src/signal_fir/`.

- §3 audit of the existing `ReverseADTransform` (three-pass postorder,
  FAD-aligned local rules), the LTI fast path
  (`recursive_projection_frontier` + `transpose_lti_de_bruijn_rec_with_cotangents`
  → `ReverseTimeRec`), and the `SigIIR` bridge for orders ≤ 2.
- §4 concrete signal IR shape `SIG_BLOCK_REVERSE_AD_TAG` with explicit
  children `[body_list, primal_count, seed_list, cotangent_list, policy]`,

- §5.1 static delay-bound check via the `interval` crate already populated

  (default 64), with explicit `delay-bound-unknown` and
  `delay-too-long-for-block` diagnostics. The cross-block continuity case is

  carry-over (Williams & Peng 1990), with a three-regime table and a survey
  of how PyTorch (`detach`, `torch.utils.checkpoint`, `torch.func.vjp`) and
  JAX (`lax.scan` + `stop_gradient`, `jax.checkpoint` / `remat`, Revolve in
  Diffrax/Equinox) let the user pick the truncation point.
- §11 file-by-file change list across `signals/`, `sigtype/`, `normalize/`,
  `transform/signal_prepare/`, `propagate/reverse_ad/`, and
  `transform/signal_fir/module.rs`. Documents the dispatch order
  symbolic → LTI fast path → block fallback.
- §11.4 backend lowering: tape allocation `BS × K` of `real_ty`, forward
  sweep using `sample_phases.sample_end`, reverse sweep reusing
  `simple_for_loop(reverse=true)` and `emit_reverse_time_rec_compute_resets`
  for adjoint resets, anti-causal rules for `Delay1` / `Delay(c)` / `Prefix`,
  recursive-carrier adjoint arrays.
- §11.5b post-B4 evolution paths: overlapping TBPTT(k1, k2) via a ring
  buffer driven by a new `truncation_depth` field, and inter-block adjoint
  carry that revives the `rad(expr, seeds, horizon)` / `--rad-horizon`
  reservation from the April plan.

## Implement RAD Phase B2 — reference BPTT executor + FD oracle

Landed Phase B2 of
`porting/rad-block-reverse-ad-signal-ir-plan-2026-05-07-en.md`: a
self-contained, tape-based TBPTT(BS, BS) reference executor that operates
directly on Signal-IR nodes, independent of the FIR lowering pipeline.  This
executor is the correctness oracle for Phase B3 (backend lowering).

New file: `crates/compiler/tests/block_reverse_ad.rs`

Public API:

- `collect_postorder(arena, root, visited, order)` — DFS postorder traversal
  covering all signal-node kinds with children (BinOp, unary math, Delay1,
  Select2, etc.).  Shared `visited` set allows multi-output bodies to reuse a
  single traversal.

- `eval_sig(arena, sig, vals, inputs, ui_vals, delay1_state) -> f64` — forward
  evaluation of one node given already-computed child values.  Handles: `Real`,
  `Int`, `Input`, `HSlider`/`VSlider`/`NumEntry`/`Button`/`Checkbox` (via
  `ui_vals`), all 17 `BinOp` variants, full trig/exp/log/sqrt/abs/floor/ceil
  suite, `IntCast`, `FloatCast`, `Select2`, and `Delay1` (reads from
  `delay1_state`, keyed by the `Delay1` node's own `SigId`).

- `propagate_adj(arena, sig, y_bar, vals, adj, adj_carry)` — one backward
  adjoint step.  Differentiable rules: Add/Sub/Mul/Div product-rule, Pow,
  Min/Max (subgradient), full trig/exp/log/sqrt/abs chain rule, FloatCast
  identity, Fmod/Remainder pass-through.  `Delay1` is anti-causal: deposits
  `y_bar` into `adj_carry[x]` instead of `adj[x]` so the contribution arrives
  at the backward step for the previous sample.

- `block_bptt_eval(arena, body_sigs, seed_sigs, cotangents, input_block,
  ui_vals, block_size) -> BpttResult` — full TBPTT(BS, BS) loop.  Forward
  pass: builds per-sample `tape[sample][sig] = f64`, updates `delay1_state`
  after each sample.  Backward pass (reverse sample order): starts each step
  with carry from sample+1, injects cotangents into primal adjoints, runs
  reverse-postorder backprop, records per-seed adjoints.  Returns
  `BpttResult { primals, grads, grads_total }`.

- Finite-difference helpers:
  - `forward_block_sum` — forward-only block evaluation for FD oracle.
  - `fd_const_seed` — central-difference estimate for constant (UI) seeds.
  - `fd_audio_seed` — forward-difference estimate for per-sample audio seeds
    (exact because block sum is linear in any single input sample).

Tests (7, all green):

- `block_ref_const_seed_linear` — `body=[2*x]`, x=0.5 constant; per-sample
  grad=2, total=16, matches FD.
- `block_ref_const_seed_sin` — `body=[sin(x)]`, x=0.5; grad=cos(0.5) per
  sample, total=BS*cos(0.5), matches FD.
- `block_ref_const_seed_delay1` — `body=[delay1(x)]`, x=0.5; forward gives
  0 at sample 0 then 0.5; BPTT total=BS-1 via anti-causal carry; FD matches.
- `block_ref_const_seed_square` — `body=[x*x]`, x=2.0; grad=4 per sample,
  matches FD.
- `block_ref_audio_seed_linear` — `body=[2*input(0)]`; per-sample grad=2,
  confirmed per sample via FD forward-difference.
- `block_ref_audio_seed_delay1` — `body=[delay1(input(0))]`; grad[j]=1 for
  j<BS-1, grad[BS-1]=0 (last sample's carry exits the block); confirmed by FD.
- `block_ref_carrier_round_trip` — builds a full `SigBlockReverseAD` carrier
  via `SigBuilder::block_reverse_ad`, extracts body/seed/cotangent lists with
  `match_sig` + `list_to_vec`, feeds to executor; guards B0 encoding contract.
