### `fir` / `codegen` / `transform` / `xtask` — Module API: `functions` rename + IO arity first

Commit date: 2026-03-03

Source commit: e7a1f5d

#### Context

The FIR `Module` shape was updated to:
- rename the module function section from `declarations` to `functions`,
- make `num_inputs` and `num_outputs` the first `Module` payload fields for
  both construction and matching/traversal.

After the core change in `crates/fir`, multiple downstream consumers still
used the old field name and/or kept non-canonical field ordering in
`FirMatch::Module` destructuring.

#### Changes

- `crates/fir/src/lib.rs`
  - `FirBuilder::module` now encodes `Module` children as:
    `(num_inputs, num_outputs, name, dsp_struct, globals, functions)`.
  - `FirMatch::Module` field renamed `declarations -> functions`.
  - `match_fir` decode and module-child traversal updated accordingly.

- Propagated `declarations -> functions` to consumers:
  - `crates/fir/src/checker.rs`
  - `crates/fir/src/inliner.rs`
  - `crates/codegen/src/backends/c/mod.rs`
  - `crates/codegen/src/backends/cpp/mod.rs`
  - `crates/codegen/src/backends/cranelift/mod.rs`
  - `crates/codegen/src/backends/interp/mod.rs`
  - `crates/transform/src/signal_fir/mod.rs`
  - `crates/xtask/src/main.rs`

- Canonicalized `FirMatch::Module` destructuring order where relevant so
  `num_inputs, num_outputs` appear first in call sites that consume full module
  metadata.

#### Validation

- `cargo check --workspace`
- `cargo test -p fir`
- `cargo test -p compiler --test signal_pipeline`
- `cargo test --workspace --all-targets`
