## Add a reader-oriented porting history

Commit date: 2026-07-24
Source commit: this commit

Added `docs/faust-cpp-to-rust-port-history-overview-en.md` as a compact,
English-language entry point to the complete technical history. The overview
keeps the full source document as the detailed reference while presenting the
compiler pipeline, four conceptual crate groups, five architectural decisions,
the validation evidence, current limits, and a condensed timeline.

The final overview expands the initially targeted 2,500-word format where the
technical context benefits from more precision. Dedicated sections now explain
the module-wide FIR verifier and FIR-owned execution model, the interpreter's
role as an immediate in-process runtime oracle, and the different contracts and
maturity levels of the C, C++, Interpreter, Cranelift, WebAssembly/WAT,
AssemblyScript, Julia, Rust, and FIR outputs.

The overview also gives the native embedding work appropriate weight. It
documents the Interpreter and Cranelift factory/instance APIs, lifecycle and
cache ownership, FBC and experimental Cranelift serialization status, UI and
metadata callbacks, auxiliary JSON/SVG generation, and the unified
`libfaust-rs` static/dynamic distribution. A separate Box/Signal API section
covers the shared `tree-ffi` context, construction and inspection families,
Box-to-Signal conversion, normal form and source generation, C memory
contracts, C++ wrappers, and the 269-symbol export check.

The root README now links to the concise and detailed histories separately,
and the detailed history links readers back to the overview.

Validation was documentation-scoped: all local links in the overview resolve,
no trailing whitespace was found, and `git diff --check` passed.
