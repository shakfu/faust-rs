### `fir` / `transform` / `codegen` / `compiler` / `cranelift-ffi` / `xtask` — FIR module `functions` harmonization + Cranelift API rename

Commit date: 2026-03-04

#### Context

After migrating FIR `Module` to use `functions` (instead of `declarations`) and
putting `num_inputs`/`num_outputs` first, remaining call sites still mixed old
naming in local variables, helper names, and diagnostics/documentation wording.

In parallel, the Cranelift entry-point rename was completed from
`compile_fir_to_cranelift_jit` to `generate_cranelift_module`, and all public
consumers had to be aligned.

#### Changes

- Cranelift backend API rename propagated:
  - `crates/codegen/src/backends/cranelift/mod.rs`
  - `crates/compiler/src/main.rs`
  - `crates/compiler/examples/corpus_scan_cranelift.rs`
  - `crates/cranelift-ffi/src/factory.rs`
  - `crates/cranelift-ffi/examples/compute_bench.rs`
  - `crates/xtask/src/main.rs`
  - `crates/codegen/README.md`

- FIR `Module.functions` terminology made consistent across builders/tests/helpers:
  - `crates/transform/src/signal_fir/module.rs`
  - `crates/codegen/src/fixtures.rs`
  - `crates/compiler/src/lib.rs`
  - `crates/codegen/src/backends/interp/mod.rs`

- Backend helper naming aligned with FIR field:
  - `crates/codegen/src/backends/c/mod.rs`
    (`collect_declared_functions` -> `collect_module_functions`)
  - `crates/codegen/src/backends/cpp/mod.rs`
    (`collect_declared_function_names` -> `collect_module_function_names`)
  - `crates/codegen/src/backends/cranelift/mod.rs`
    (`decls` -> `function_nodes` in module-function scanning path)

- FIR checker/inliner wording and enums aligned with `functions`:
  - `crates/fir/src/checker.rs`
    - `check_declarations` -> `check_functions`
    - diagnostics/doc text updated (`functions block`)
    - test helpers renamed (`make_full_functions`, local `functions` vars)
  - `crates/fir/src/inliner.rs`
    - `FirFunctionSection::Declarations` -> `FirFunctionSection::Functions`
    - section labels/messages updated to `"functions"`

#### Validation

- `cargo fmt --all --check`
- `cargo check -p fir -p transform -p codegen -p compiler -p cranelift-ffi -p xtask`
- `cargo test -p fir -p transform -p codegen -p compiler`
