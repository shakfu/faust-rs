# 2026-07-08

## Clock domains — lower held ondemand payloads like C++

Commit date: 2026-07-08
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

Bug found with `examples/ondemand/ondemand.dsp`:

```faust
process = (button("sound1") : ondemand(os.osc(500))),
          (button("sound2") : ondemand(os.square(300)));
```

faust-rs emitted too much stateful oscillator/delay code outside the guarded
`if`, so the held `ondemand` value was stable but internal state advanced at
the top sample rate. The C++ backend does not do this: it lowers the whole
held `Clocked(env, value)` payload inside the consuming guarded block, writes
the permanent hold field there, and only reads the hold field outside.

Root cause: signal-fir's clocked redirection rule was too broad. When clock
inference assigned a payload or child expression to an ancestor environment
(because it did not read a domain-internal input), `lower_signal` redirected
that stateful payload out of the open block. This was the same class of bug as
the earlier `Waveform` special case, but the correct rule is not
`Waveform`-specific.

Fix:

- `ensure_guarded_block` now recognizes held `Clocked(env, value)` stores and
  lowers the payload with ancestor-domain redirection temporarily suppressed.
- `TempVar` remains a boundary read: while lowering a `TempVar` source from a
  suppressed clocked payload, redirection is re-enabled so full-rate sources
  such as `serialize_in` / `interleave` stay outside the guarded block.
- Delay-line queries now account for the current guarded region during held
  payload lowering, so circular/IfWrapping delay maintenance moves inside the
  block when the line is really part of the payload.
- The old `Waveform`-only redirect exception was removed; the waveform
  regression is covered by the general held-payload rule.
- Global `fIOTA` finalization moved after output lowering, so delay lines
  reclassified into a domain during payload lowering no longer leave a dead
  top-rate `fIOTA` field and advance.

Generated C++ for `ondemand.dsp` now has `sound1` oscillator phase updates,
`sound2` delay writes/reads, and `fIOTA_d1 = fIOTA_d1 + 1` inside their
respective `if` blocks. There is no unused top-rate `fIOTA` left in that
output.

Verified:

- `cargo test -p compiler --test ondemand_pipeline`
- `cargo test -p compiler --test clocked_waveform_regression`
- `cargo test -p compiler --test clocked_emission_structure`
- Regenerated the C++ for `faustide-od/dist/examples/ondemand/ondemand.dsp`
  and compiled a probe that toggles `sound1` and `sound2` independently; each
  output holds while its button is off and advances only while its button
  fires.
