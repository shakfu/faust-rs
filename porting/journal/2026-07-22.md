# 2026-07-22

## Publish the compiler Wasm artifact and document `libfaust` C/C++ use

Commit date: 2026-07-22
Source commit: this commit

Added a verified `xtask` packaging step for the raw `wasm-ffi` compiler module.
Cargo still builds the crate-normalized `faust_wasm_ffi.wasm` internally, while
the distribution artifact is now consistently published as
`target/wasm32-unknown-unknown/release/libfaust-rs.wasm`. The packaging step
removes stale output, performs the rename, and checks the exported compiler ABI;
the Wasm crate, `xtask` guide, main README, and faustwasm planning note now use
the public filename consistently.

Documented the unified `faust-ffi` static and dynamic libraries in the main
README, including build and link commands plus executable C and C++ lifecycle
examples for the Interpreter and experimental Cranelift JIT backends. The C API
section maps the backend-specific factory, instance, initialization, compute,
and destruction functions and records the `freeCMemory()` ownership contract.
Added the missing C++ `override` annotations to the Interpreter factory and DSP
destructors and lifecycle methods; this is a source-level correctness annotation
with no ABI or behavioral change.

Validated with `cargo fmt --all`, `cargo clippy --workspace --all-targets -- -D
warnings`, `cargo test --workspace --all-targets`, `cargo build -p faust-ffi
--release`, `cargo run -p xtask -- build-faustwasm-compiler-module`, direct C11
and C++17 compilation/link/execution against `libfaust` for both backends, a C++
header check with missing-override warnings promoted to errors, and `git diff
--check`.

## Add project-local Faust libraries and validate the synthesis examples

Commit date: 2026-07-22
Source commit: this commit

Moved `interleave.lib` from the repository root to `libraries/interleave.lib`,
added the FAD optimizer helpers as `libraries/optimizers.lib`, and documented
the shared `-I libraries` import contract. Updated the spectral corpus comments
and porting references to the new location while preserving basename imports
such as `library("interleave.lib")` and `import("optimizers.lib")`.

Reworked the bilingual FAD/RAD synthesis examples so the five-coefficient
biquad is a complete executable program and the FxLMS controller uses canonical
Faust recursion. The FxLMS differentiated expression now contains only the
controller sensitivity over the filtered reference; the physical secondary
path and measured-error term remain outside FAD. The examples were compiled and
executed, with measured error reduction for FxLMS, resonant-filter
identification, and the adaptive biquad.

Aligned `impulse-runner` with the compiler CLI's 64 MiB stack contract. This
prevents deep standard-library expansion, including the Newton example using
`stdfaust.lib` and `ma.tanh`, from exhausting the platform main-thread stack.

Validated with `cargo fmt --all -- --check`, `cargo clippy --workspace
--all-targets -- -D warnings`, `cargo test --workspace --all-targets`,
`cargo run -p xtask -- golden-check`, direct imports of both project libraries,
a real `interleave.lib` spectral corpus check, example execution, convergence
measurements, and `git diff --check`.

## Refresh FAD/RAD user documentation and add the bilingual synthesis

Commit date: 2026-07-22
Source commit: this commit

Refreshed the FAD and RAD user notes against the current compiler contracts:
FAD output ordering is per primal output and seed lane; RAD returns primal
outputs followed by gradients of their implicit all-ones sum; temporal RAD uses
the current `compute(count)` block as its reverse horizon with zero terminal
adjoint state. The notes now distinguish exact symbolic rules from the
documented finite-difference table-index rule and zero-tangent or explicit
diagnostic boundaries.

Added the English `docs/fad-rad-synthesis-en.md` counterpart to the French
synthesis, cross-linked both documents, and replaced references to local,
unversioned sketches with tracked corpus coverage or explicit illustrative
labels. The main README now introduces clock-domain primitives (`ondemand`,
`upsampling`, `downsampling`) and FAD/RAD, with links to the user-facing notes.

The FAD note records FAD's clock-domain payload handling and RAD's current
clock-boundary rejection. The RAD notes also document block-local BRA behavior,
Signal-IR seed identity, unsupported-family behavior, and the current `abs(0)`
formula limitation. Removed stale FAD/synthesis PDF copies rather than leaving
them inconsistent with their Markdown sources.

Validated with `cargo test -p compiler --test fad_recursive_runtime --test
ondemand_pipeline --test rad_runtime` (96 passed; one pre-existing
`abs(0)`-related RAD test remains ignored), `cargo check -p compiler --examples`,
Markdown link/fence checks, and `git diff --check`.
