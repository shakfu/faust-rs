# 2026-07-10 — Vector mode S-D: first real loop split (pure tail)

Branch: `ondemand-vec-fad-synthesis`.

## S-D — recursive-slice split, the vector-mode payoff

The vector substrate (P6 V1–V5b) chunked only state-free slices; a recursive
slice stayed one plain serial loop, so SIMD never touched a stateful DSP. S-D
delivers the first real per-statement split (vector doc §5.4, "pure tail").

### What lands

A recursive sample-loop slice reaches the FIR as a fused loop-carried chain, e.g.
for `process = (_ : + ~ _) * 0.5`:

```c
float fRecCur = (float)input0[i0] + fRec;   // serial: reads state
output0[i0]   = 0.5f * fRecCur;             // state-free tail: reads fRecCur
fRec          = fRecCur;                     // serial: writes state
```

`-vec` now emits:

```c
float vbuf0[32];
for (int vindex = 0; vindex < count; vindex += 32) {
    int vend = ((vindex + 32 < count) ? vindex + 32 : count);
    for (int i0 = vindex; i0 < vend; i0++) {        // serial core
        float fRecCur = (float)input0[i0] + fRec;
        fRec = fRecCur;
        vbuf0[i0 - vindex] = fRecCur;                // buffer the carrier
    }
    for (int i0 = vindex; i0 < vend; i0++) {        // vectorizable tail
        output0[i0] = 0.5f * vbuf0[i0 - vindex];
    }
}
```

The output scaling is now a self-contained inner loop with no loop-carried
dependency, a contiguous buffer read and a contiguous output write — the C
compiler can auto-vectorize it. The recursion stays serial.

### How

Three pure pieces (S-A/B/C) assembled into the emission:

- **`partition_recursive_body`** (`loop_graph.rs`) — splits a flat body into a
  **serial core** (statements touching `Struct` state, plus, by fixpoint, every
  producer of a temp a serial statement reads) and a **vectorizable tail** (the
  state-free rest). Boundary temps = serial-produced temps read by the tail. The
  fixpoint guarantees no serial statement reads a tail-produced temp, so hoisting
  the tail past the whole serial core is order-safe. Returns `None` (→ single-loop
  fallback) for unsupported statement shapes (nested `If`/loops — clocked islands)
  or when nothing is state-free to hoist.
- **`ChunkBuffer`** (S-C) — `vbufN[i0 - vindex]` chunk-local store/load.
- **`rewrite_var_loads`** — rebuilds the tail statements, redirecting boundary-temp
  `LoadVar`s to the buffer load, preserving interned identity elsewhere.
- **`emit_split_sample_loop` / `build_split_chunk_driver`** (`module/build.rs`) —
  emit `[buf decls…, chunk driver{ serial inner loop, tail inner loop }]`.
  `emit_sample_loop` now returns `Vec<FirId>` (buffers + driver) and only takes
  this path for a forward `Recursive` slice in vector mode.

### Why bit-exact

The serial core runs the whole chunk first — state evolves exactly as in the
fused loop, buffering each per-sample carrier — then the tail reads those values
back at the same global `i0`. Only storage/loop structure changed, not the
arithmetic.

### Validation

- **V6 oracle** (`crates/compiler/tests/vector_mode.rs`) — added
  `recursive_split_pure_tail_is_bit_exact` (`(_ : + ~ _) * 0.5` at `-vs 32` and a
  non-dividing `-vs 24`), scalar vs `-vec` byte-identical via interp. 5/5.
- **Partition/rewrite units** — `partition_hoists_state_free_tail`,
  `partition_declines_fully_recursive_body`,
  `partition_declines_unsupported_statement`,
  `rewrite_var_loads_redirects_boundary_temp_to_buffer` (24 `loop_graph` tests).
- **Scalar untouched** — 190 goldens byte-identical (the split is guarded by
  `ComputeMode::Vector`; scalar never reaches it).
- **Valid C** — the split compute compiles clean at `-O3 -Wall -Wextra` (two
  sibling `for (int i0 …)` are per-loop-scoped; `float vbuf0[N]` stack array OK).
- `cargo clippy -p transform -p compiler --all-targets -D warnings` clean.
