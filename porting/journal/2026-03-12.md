### `ui` / `propagate` / `transform` / `codegen` — Implement C++-style UI label metadata extraction and `declare(...)` emission

Commit date: 2026-03-12

Source commit: worktree-authored-2026-03-12-ui-label-metadata-parity

#### Context

The grouped-UI rewrite had already introduced canonical `UiProgram`
ownership, but one visible parity gap remained versus Faust C++:

- labels such as `gain [style:knob]` were still kept as raw display labels in
  Rust
- `ControlSpec.metadata` stayed empty
- generated C/C++ `buildUserInterface` output therefore missed
  `declare(..., "style", "knob")`

Faust C++ handles this through `extractMetadata(...)` in
`compiler/generator/description.cpp`, then emits simplified labels plus
`declare(...)` calls in `buildUserInterface`.

#### Changes

- [ui/src/lib.rs](../../crates/ui/src/lib.rs)
  - added canonical `split_label_metadata(...)` with Rustdoc provenance to
    match C++ `extractMetadata(...)`

    `UiMatch`
  - introduced `UiMetadata` as the canonical metadata carrier

  - now splits widget and group labels during grouped-UI construction
  - stores simplified labels plus extracted metadata in `ControlSpec`
  - preserves group metadata when rewriting synthesized root labels

- [transform/src/signal_fir/module.rs](../../crates/transform/src/signal_fir/module.rs)
  - lowers UI/group metadata to FIR `AddMetaDeclare`
  - emits root-level group metadata on fictive zone `0`
  - emits control metadata before the corresponding UI `add*` instruction
  - treats soundfile `url` metadata like C++ by routing it into
