## docs(draw): write SVG draw module porting plan

Commit date: 2026-05-02

Source: `porting/draw-svg-port-plan-2026-05-02-en.md`

Full porting plan for the Faust block-diagram drawing module (`compiler/draw/`)
targeting a `-svg` CLI flag identical to the reference C++ compiler.

### Analysis scope

Reviewed all 46 C++ files (~5 400 lines) in `faust/compiler/draw/`:

- **Entry points**: `drawschema.cpp` (orchestration, folding, box‚Üíschema mapping),
  `sigToGraph.cpp` (signal tree ‚Üí Graphviz dot).
- **Device layer**: abstract `device` class (11 pure-virtual primitives),
  concrete `SVGDev` (XML output via `FILE*`), `PSDev` (PostScript, deferred).
- **Schema tree**: abstract `schema` class with `point`, `trait`, `collector` types;
  18 concrete schema types (BlockSchema, CableSchema, SeqSchema, ParSchema,
  MergeSchema, SplitSchema, RecSchema, TopSchema, DecorateSchema, EnlargedSchema,
  RouteSchema, OnDemandSchema, DownSamplingSchema, UpSamplingSchema, and leaf variants).
- **Layout algorithm**: bottom-up sizing in constructors, top-down placement in
  `place(x, y, orientation)`, then `draw(device)` + `collectTraits` for wires.
- **Folding logic**: `boxComplexity(bd) > gFoldThreshold` ‚Üí one `.svg` per named
  sub-diagram above `gFoldComplexity`.

### Plan decisions

- `DrawDevice` trait (11 methods) + `SvgDevice` writing raw XML via `write!()`
  with manual entity escaping ‚Äî no external crate.
- `Schema` trait with `&mut self` for `place()`, `&self` for `draw()`.
- `Box<dyn Schema>` for composite schemas (mirrors C++ pointer model).
- Color constants match the active C++ set:
  `linkcolor=#003366`, `normalcolor=#4B71A1`, `uicolor=#477881`,
  `slotcolor=#47945E`, `numcolor=#f44800`, `invcolor=#ffffff`.
- CLI: `-svg` ‚Üí `--svg` via `normalize_legacy_args`; output dir `<name>-svg/`.
- PostScript deferred ‚Üí `DrawError::NotSupported`.
