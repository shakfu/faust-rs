## docs(draw): write SVG draw module porting plan

Commit date: 2026-05-02

Source: `porting/draw-svg-port-plan-2026-05-02-en.md`

Full porting plan for the Faust block-diagram drawing module (`compiler/draw/`)
targeting a `-svg` CLI flag identical to the reference C++ compiler.

### Analysis scope

Reviewed all 46 C++ files (~5 400 lines) in `faust/compiler/draw/`:

- **Entry points**: `drawschema.cpp` (orchestration, folding, box→schema mapping),
  `sigToGraph.cpp` (signal tree → Graphviz dot).
- **Device layer**: abstract `device` class (11 pure-virtual primitives),
  concrete `SVGDev` (XML output via `FILE*`), `PSDev` (PostScript, deferred).
- **Schema tree**: abstract `schema` class with `point`, `trait`, `collector` types;
  18 concrete schema types (BlockSchema, CableSchema, SeqSchema, ParSchema,
  MergeSchema, SplitSchema, RecSchema, TopSchema, DecorateSchema, EnlargedSchema,
  RouteSchema, OnDemandSchema, DownSamplingSchema, UpSamplingSchema, and leaf variants).
- **Layout algorithm**: bottom-up sizing in constructors, top-down placement in
  `place(x, y, orientation)`, then `draw(device)` + `collectTraits` for wires.
- **Folding logic**: `boxComplexity(bd) > gFoldThreshold` → one `.svg` per named
  sub-diagram above `gFoldComplexity`.

### Plan decisions

- `DrawDevice` trait (11 methods) + `SvgDevice` writing raw XML via `write!()`
  with manual entity escaping — no external crate.
- `Schema` trait with `&mut self` for `place()`, `&self` for `draw()`.
- `Box<dyn Schema>` for composite schemas (mirrors C++ pointer model).
- Color constants match the active C++ set:
  `linkcolor=#003366`, `normalcolor=#4B71A1`, `uicolor=#477881`,
  `slotcolor=#47945E`, `numcolor=#f44800`, `invcolor=#ffffff`.
- CLI: `-svg` → `--svg` via `normalize_legacy_args`; output dir `<name>-svg/`.
- PostScript deferred → `DrawError::NotSupported`.

## feat(draw): implement phases A–E — all schema types (24 tests pass)

Implemented the full schema + device infrastructure for the SVG draw module.
All 24 unit tests pass; no unsafe code used.

### Phase A — Core infrastructure

**`crates/draw/src/error.rs`** — `DrawError` enum (`Io`, `Layout`, `NotSupported`).

**`crates/draw/src/schema.rs`** — base types and `Schema` trait:
- Layout constants: `D_WIRE=8`, `D_LETTER=4.3`, `D_HORZ=4`, `D_VERT=4`
- Color palette matching C++ active set
- `Orientation` (`LeftRight` / `RightLeft`) with `sign() -> f64`
- `Point` with `Ord` (lex on x then y)
- `Trait` with `has_real_input/has_real_output` reachability flags
- `TraitCollector`: collects traits + computes visible wires
- `Placement` value type
- `Schema` object-safe trait (11 methods)

**`crates/draw/src/device.rs`** — `DrawDevice` trait + `SvgDevice<W: Write>`:
- 11 drawing primitives matching C++ `device` API
- `SvgDevice` stores `Option<BufWriter<W>>` so `finish()` can extract the inner
  writer safely without unsafe code; `Drop` writes `</svg>` if not yet finished
- `xml_escape()` handles `< > & ' "`
- Key lesson: raw strings `r#"..."#` are terminated by `"#`, so CSS color strings
  like `"#FFFFFF"` require `r##"..."##`

### Phase B — Leaf schemas (`crates/draw/src/schemas/block.rs`, `cable.rs`)

- `BlockSchema`: text label, colors, optional hyperlink; `quantize()` rounds text
  width to next multiple of 3×D_LETTER for grid alignment
- `InverterSchema`: fixed-size triangle symbol (wraps BlockSchema)
- `CableSchema`: n parallel passthrough wires (n inputs, n outputs, zero width)
- `CutSchema`: 1 input, 0 outputs, zero width — absorbs a wire
- `ConnectorSchema`: invisible 1→1 D_WIRE square

### Phase C — Composition schemas (`seq.rs`, `par.rs`, `merge.rs`, `split.rs`, `rec.rs`)

- `SeqSchema` (`make_seq`): sequential composition with auto-balancing cables;
  `compute_horz_gap` analyzes wire directions (Up/Down/Hor) to compute spacing
- `ParSchema` (`make_par`): parallel stacking with `make_enlarged` to equalize widths
- `MergeSchema` (`make_merge`): many-to-one fan-in; hgap = (a.height+b.height)/4
- `SplitSchema` (`make_split`): one-to-many fan-out; same gap formula
- `RecSchema` (`make_rec`): recursive feedback `s1~s2`; delay-sign rendering +
  `collect_feedback` / `collect_feedfront` for proper wire routing

### Phase D — Decorator schemas (`composed.rs`)

- `EnlargedSchema` (`make_enlarged`): pads width with equal left/right margins;
  no-op passthrough if already wide enough
- `DecorateSchema` (`make_decorate`): dashed-border group box with label
- `TopSchema` (`make_top`): root wrapper — white background, title, output arrows;
  inputs=0, outputs=0 (hides all ports at the top level)

### Phase E — Specialized schemas (`route.rs`, `multirate.rs`)

- `RouteSchema` (`make_route`): explicit 1-based (src, dst) routing with validation
- `MultiRateSchema` shared body for `make_ondemand`, `make_downsampling`,
  `make_upsampling`: solid-border box, extra clock/rate input port, labels

### Key Rust issues encountered

1. **Rust 2024 format strings**: cannot mix named captures `{x}` with positional
   `{}` in the same `writeln!` — pre-compute all values into named locals.
2. **`r#"..."#` raw strings**: the sequence `"#` terminates the literal; CSS color
   strings like `"#FFFFFF"` require `r##"..."##`.
3. **`Drop` prevents moving fields**: `SvgDevice` originally stored `BufWriter<W>`
   directly; tests tried to `into_inner()` which moves a field out of a `Drop` type.
   Fixed by wrapping in `Option<BufWriter<W>>` — `finish()` calls `take()` to
   safely extract; `Drop` checks `if let Some(ref mut bw)`.
