## docs(rad): plan disabling ReverseTimeRec RAD fast path

Commit date: 2026-05-10
Source commit: this commit (`Plan disabling ReverseTimeRec RAD fast path`)

Added `porting/rad-disable-reverse-time-rec-fast-path-plan-2026-05-10-en.md`.
The plan records the decision to stop routing recursive, temporal, and IIR RAD
cases through the legacy `ReverseTimeRec` LTI/IIR fast path and to make
`SigBlockReverseAD` the sole fallback carrier for cases that the symbolic
feed-forward sweep cannot resolve locally.

The preserved `ReverseTimeRec` signal carrier, type/normalization support, and
FIR lowering are documented as dormant infrastructure: they remain compilable
and tested for a future optimized fast-path revival, but RAD propagation should
not produce them after the follow-up implementation patch.

Key implementation notes captured by the plan:

- Collapse recursive projection and `SigIIR` handling in
  `crates/propagate/src/reverse_ad.rs` into `RadUnsupportedNode` paths consumed
  by `build_block_reverse_ad`.
- Add `iir-state-space` to the `generate_rad_signals` fallback whitelist so
  first-order, second-order, and higher-order IIR cases all reach BRA.
- Update propagation-level tests that assert `ReverseTimeRec` carrier shape to
  assert `BlockReverseAD`, while keeping direct `ReverseTimeRec` infrastructure
  tests enabled.
- Add Rustdoc notes on the dormant carrier/lowering entry points so future LTI
  optimization work can be re-enabled deliberately.
