## Specify signal-level vector planning and general `-ss` scheduling

Commit date: 2026-07-11
Source commit: this commit

Added
[`vector-mode-signal-level-analysis-cpp-port-plan-2026-07-10-en.md`](../vector-mode-signal-level-analysis-cpp-port-plan-2026-07-10-en.md)
after re-analyzing the pinned Faust C++ compiler at
`master-dev-ocpp-od-fir-2-FIR19` (`8eebea429`).

The analysis establishes the target architecture:

- vector loop boundaries, ownership, dependencies, effects, epochs, and
  transports are planned over prepared `SigId` values before FIR fusion;
- the resulting `VectorPlan` is immutable and independent of scheduling;
- one public `-ss` option and one generic scheduler implementation are used in
  scalar and vector modes, with no separate Rust `-dfs` option;
- scalar mode schedules hierarchical signal DAGs, while vector mode schedules
  each completed per-epoch `LoopGraph` subgraph;
- `-ss 0` subsumes C++ vector `-dfs`, while `-ss 3` is the closest match for
  default C++ `CodeLoop::sortGraph` levelization. The earlier `-ss 1` mapping was
  incorrect because it measures depth from dependency leaves instead of output
  roots.

The plan also adds a formal specification layer:

- typed analysis judgments over signal type, variability, clock domain, and
  effects;
- labelled dependency and causality rules;
- mathematical definitions and soundness/completeness obligations for all four
  scheduling strategies;
- ordered loop-separation, ownership, region-aware lowering, and typed transport
  rewrite rules;
- explicit effect commutation, epoch barriers, region visibility, and
  `VecSafe`/`StaticFissionSafe` conditions;
- scalar/vector simulation obligations for delays, recursion groups, clock
  domains, and forward/reverse AD;
- independently checkable schedule, vector-plan, and routed-FIR certificates;
- formal acceptance gates and bounded/exhaustive validation tasks for P0-P7.
