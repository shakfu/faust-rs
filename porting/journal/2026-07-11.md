## Fix adequacy and proof gaps in the Lean scheduling specification

Commit date: 2026-07-11
Source commit: this commit

Reviewed
[`vector-mode-scheduling-formal-spec.lean`](../vector-mode-scheduling-formal-spec.lean)
as a formalization and corrected the places where the mechanization was weaker
than, or contradicted, the properties claimed by the companion port plan. The
file still compiles with `lean porting/vector-mode-scheduling-formal-spec.lean`
(bundled Std, no `sorry`, no axioms).

Fixed:

- **Typing was not functional.** `intLit`/`realLit`/`input` took a free `clock`
  parameter, so one expression admitted many judgments and the plan's
  `Totality` ("exactly one judgment exists") was false. Base signals now live in
  a fixed `rootClock`; `clocked` is the sole clock-changer. Added
  `hasType_functional`, proving the judgment assigns at most one decoration per
  expression.
- **`Effect.conflicts` symmetry was assumed, not proved.** The commutation test
  (`effectsCommute`, `Commute`) relies on it. Switched resource comparison to
  `decide (a = b)` on `DecidableEq Resource` and added `decide_eq_symm` +
  `conflicts_symm`.
- **Duplicability and vec-safety were vacuous.** `VectorPlan.duplicable` and
  `vectorizationSafe` were opaque `Prop` fields satisfiable by `fun _ => True`.
  Replaced them with per-signal `effects`/`vectorability` data and *defined*
  `duplicableEffectsB`, `sampleReorderableB`, `VectorPlan.Duplicable`,
  `VectorPlan.VecSafe`, so `placementDuplicate` and the certificate's
  `vectorizableSafe` now carry real, conservative content.
- **`verifySchedule` had no independent semantic anchor.**
  `verifySchedule_eq_true_iff` was a `rfl` (the proposition *was* the checker).
  Added relational `CoversRel`/`RespectsDepsRel`/`ValidScheduleRel` (stated
  without the Boolean checker) and proved `validScheduleB_iff` plus
  `verifySchedule_sound`/`verifySchedule_complete` against them, with bridging
  lemma `noDuplicatesB_iff_nodup`.
- **Reference checker was weaker than the Rust R1 checker.** `coversB` did not
  require `graph.nodes` to be duplicate-free, so a graph with duplicate nodes was
  "covered" by its deduplication. Added `noDuplicatesB graph.nodes`.
- **`VSimulation` was too strong to be true.** It demanded scalar/vector
  agreement for every `epochSchedules`, including non-topological ones. Added a
  `scheduleValid` premise so the definition states the theorem that actually
  holds.

Also added a note that `delay` over-approximates state effects on purpose (the
sound direction; `n = 0` discharge is left to concrete delay analysis).

Not changed: `DecidableEq ValueTy` was left underived. It does not auto-derive
through `List ValueTy`, and the plan-certificate layer stays at the `Prop` level
in this spec, so decidable type equality belongs to the Rust checker rather than
a fragile hand-written instance here.

Already present in the file from earlier work and kept: the exhaustive
`separateLoop_complete` characterization and the `#guard` regression assertions.

The plan defines:

[`vector-verification-certificate-v1.schema.json`](../schemas/vector-verification-certificate-v1.schema.json).

## Specify signal-level vector planning and general `-ss` scheduling

Commit date: 2026-07-11
Source commit: this commit

Added
[`vector-mode-signal-level-analysis-cpp-port-plan-2026-07-10-en.md`](../vector-mode-signal-level-analysis-cpp-port-plan-2026-07-10-en.md)
after re-analyzing the pinned Faust C++ compiler at
`master-dev-ocpp-od-fir-2-FIR19` (`8eebea429`).

The analysis establishes the target architecture:

- vector loop boundaries, ownership, dependencies, effects, epochs, and
  transports are planned over prepared `SigId` values before FIR fusion;
- the resulting `VectorPlan` is immutable and independent of scheduling;
- one public `-ss` option and one generic scheduler implementation are used in
  scalar and vector modes, with no separate Rust `-dfs` option;
- scalar mode schedules hierarchical signal DAGs, while vector mode schedules
  each completed per-epoch `LoopGraph` subgraph;
- `-ss 0` subsumes C++ vector `-dfs`, while `-ss 3` is the closest match for
  default C++ `CodeLoop::sortGraph` levelization. The earlier `-ss 1` mapping was
  incorrect because it measures depth from dependency leaves instead of output
  roots.

The plan also adds a formal specification layer:

- typed analysis judgments over signal type, variability, clock domain, and
  effects;
- labelled dependency and causality rules;
- mathematical definitions and soundness/completeness obligations for all four
  scheduling strategies;
- ordered loop-separation, ownership, region-aware lowering, and typed transport
  rewrite rules;
- explicit effect commutation, epoch barriers, region visibility, and
  `VecSafe`/`StaticFissionSafe` conditions;
- scalar/vector simulation obligations for delays, recursion groups, clock
  domains, and forward/reverse AD;
- independently checkable schedule, vector-plan, and routed-FIR certificates;
- formal acceptance gates and bounded/exhaustive validation tasks for P0-P7.
