## Add Faust directory-info compatibility flags

Commit date: 2026-06-16
Source commit: this commit

Added C++ Faust-compatible directory-info queries to the Rust compiler CLI and
faustwasm helper surface:

- `-libdir` / `--libdir`;
- `-includedir` / `--includedir`;
- `-archdir` / `--archdir`;
- `-dspdir` / `--dspdir`;
- `-pathslist` / `--pathslist`.

The CLI now treats these as early-exit information flags, matching the C++
`global::printDirectories()` behavior: no DSP input file is required and, when
several are present, the C++ precedence order is used. The rendered output keeps
the C++ newline and section format conventions.

The implementation adds a `FaustInstallPaths` helper for install-root-derived
paths and path-list rendering, wires the same outputs into
`Compiler::get_faustwasm_info`, and updates the wasm-ffi documentation to mark
the keys as supported rather than stubbed. The impulse benchmark wrapper still
answers these queries from the configured C++ Faust installation so local,
uninstalled `target/release/faust-rs` builds use the same header and
architecture tree during `faustbench`.

Validation:

- `cargo test -p compiler cli::tests::normalize_legacy_args_maps_directory_info_flags`
- `cargo test -p compiler compiler_get_faustwasm_info_supports_cpp_directory_keys`
- `cargo run -p compiler -- -includedir`
- `cargo run -p compiler -- -pathslist`
- `make -C tests/impulse-tests -f Make.bench compile-bench dspfiles=dsp/APF.dsp COMPILE_BENCH_CSV=build/bench/doc-check-compile-summary.csv`

## Add impulse faustbench and compile-time comparisons

Commit date: 2026-06-16
Source commit: this commit

Added performance comparison workflows to the impulse-test harness.

The new `Make.bench` target runs the impulse DSP corpus twice through
`faustbench -single`: once with the reference C++ Faust compiler (`FAUST_CPP`)
and once with `faust-rs` (`FAUST_RS`). Since `faustbench` resolves a binary
named `faust` from `PATH`, the makefile creates temporary compiler wrappers
under `build/bench/` and writes:

- `build/bench/summary.csv` with C++ Faust throughput, faust-rs throughput, and
  relative delta per DSP;
- `build/bench/logs/*.log` with full `faustbench` output for each compiler run.

The benchmark defaults to `BENCH_OPTIONS=-double`, reuses the impulse-test
library include paths, and exposes `FAUSTBENCH`, `BENCH_WARN_MIN`, and
`BENCH_CSV` as overridable variables. On macOS, a local C++ wrapper patches the
temporary `faustbench.cpp` copy to disable Apple-only `-clang` duplicate cases,
so the reported comparison stays on the common scalar `-single` case.

The same makefile also adds `compile-bench`, which measures direct
`-lang cpp -double` code-generation wall time for both compilers on the same
DSP corpus. It writes `build/bench/compile-summary.csv`, generated C++ sources
under `build/bench/compile/`, and per-compiler logs under `build/bench/logs/`.
The compile timer uses `Time::HiRes` so small DSPs do not collapse to `0.00s`
under `/usr/bin/time -p`, and its parser normalizes comma decimal separators
from localized environments.

Validation:

- `make help`
- `make -n bench`
- `make -f Make.bench bench dspfiles=dsp/osc.dsp BENCH_CSV=build/bench/test-summary.csv`
- `make -C tests/impulse-tests compile-bench dspfiles="dsp/APF.dsp dsp/BPF.dsp dsp/HPF.dsp dsp/LPF.dsp" COMPILE_BENCH_CSV=build/bench/test-compile-summary.csv`
