Branch: `ondemand-vec-fad-synthesis`

Added ondemand-FFT corpus fixtures, all consuming the real `interleave.lib`
(S2 primitive, commit c088ced) rather than re-inlining `frame_clock` /

Analysis-only (2N held bin reals, no resynthesis):
`ondemand_fft_framed_{004,008,016,032,064,128,256}.dsp` —
`il.serialize_in(N) : (il.frame_clock(N), si.bus(N)) : ondemand(fftFX(N))`.
Same construction locked by `crates/compiler/tests/interleave_fft.rs`.

Complete round-trip (time → spectrum → effect → time) via `il.interleave(N, FX)`
with an N-in/N-out frame operator `real→complex : an.fft : <effect> : an.ifft :
real-part`:
- `ondemand_fft_roundtrip_id_016.dsp` — identity FX, correctness anchor: wet ==
  dry `@(N-1)` (verified with impulse-runner: dry impulse at frame 0, wet at 15).
- `ondemand_fft_lowpass_{016,032}.dsp` — brickwall LP by Hermitian-symmetric bin
  masking (keep |m| ≤ kc). Impulse response is exactly `(1+2·Σcos)/N`

- `ondemand_fft_robot_016.dsp` — robotization: replace each bin by (|X|, 0)

`FRS-PROP-0002 left outputs (2N) != right inputs (N)`. This is the
"analysis-only mode (no serialize_out)" of
`porting/interleave-spectral-primitive-2026-07-07-en.md` §6. The round-trip
fixtures do use `il.interleave` because their FX round-trips back to `N`.

  butterfly codegen, is the wall (`an.fftb`/`c_bit_reverse_shuffle` recursion
  crosses it at N≥512). Matches the §6 compile-time risk.
- Round-trip is far heavier (fft + ifft + serialize_out): N=64 lowpass ~12s,

## Corpus — phase-vocoder with inter-frame phase accumulation

Branch: `ondemand-vec-fad-synthesis`

Closed the "inter-frame" gap flagged above: a per-bin phase memory and synthesis
phase accumulator that update **once per frame inside the `ondemand` block** —
the state shape the earlier spectral fixtures could not express.

Enabler first: verified a recursive accumulator `+ ~ _` placed inside an
`il.interleave_hop` frame operator advances on the per-domain (frame) clock (a
constant-1 source yields a staircase that steps every `hop` samples). This is
the recursion analogue of the already-tested per-domain `@(1)` delay.

Per bin k, each frame: `mag=|X_k|`, `ph=atan2(Im,Re)`,
`incr = expected_k + princarg(ph - ph' - expected_k)` (true unwrapped per-hop
advance, `expected_k = 2*pi*k*hop/N`, `princarg` via `floor`), synthesis phase
`psi = <increment> : (+ ~ _)`, bin `= mag*(cos psi, sin psi)`; then IFFT, window,
overlap-add. `ph'` is a one-frame phase memory.

Fixtures (N=16, hop=N/2, impulse-runner verified):
- `ondemand_stft_pv_identity_016.dsp` — phase-propagation identity: accumulate
  the true measured advance. Self-generated 2 kHz sine → wet reconstructs it
  exactly: `max|wet[t] - dry[t-(N-1)]| = 1.5e-5`. Proves the machinery.
- `ondemand_stft_pv_freqshift_016.dsp` — add a constant per-sample phase rate to
  every bin (single-sideband frequency shift). DC input → a pure tone at exactly
  the shift frequency (period fs/16, amplitude 1.0). A *correct* constant-rate
  effect on the same accumulator.

`--dump-cpp` confirms the accumulator (`fRecCur = fRec + atan2(…) + (pi - (fVec[1]
+ 2*pi*floor(…)))`) and the phase memory (`fVec[1]` = ph') are emitted **inside**
the frame-clock guard `if ((iRecCur… == 0))` — i.e. lowered as frame-rate state.

Honest scope note: true **time-stretch** (analysis hop ≠ synthesis hop → duration
change) is *not* expressible in a synchronous 1-in/1-out Faust process at constant
rate; it needs an external rate-decoupling buffer. What was the actual gap vs
SC/Csound — the inter-frame phase machinery — is now expressible, and its correct
constant-rate products (identity reconstruction, frequency shifting) are shipped
and verified. A duration-changing stretch / clean pitch shift (stretch + resample,
or bin reassignment) remains future work.

Full sweep: 19/19 ondemand FFT/STFT fixtures compile through `--lang interp`.

## FFT scalability — root cause: CSE skips ondemand block bodies

Branch: `ondemand-vec-fad-synthesis`

Diagnosed why the ondemand FFT does not scale. The generated in-block FFT is
**O(N^2.6)** (arithmetic ops grow ~6×/doubling; `ops/(N log N)` diverges 7.8→262
for N=8→128), not O(N log N) — worse than a naive O(N²) DFT. For N=32 the code
holds only ~65 temporaries (≈2N output bins) and zero intermediate-stage temps:
the butterfly DAG is flattened to 2N trees.

Not intrinsic to Faust: the **plain** `an.fft(N)` (no ondemand) is O(N log N)
(`ops/(N log N)` flat 8.0→9.0, temps O(N log N)). Same core, same compiler —
the only difference is being inside an `ondemand` guarded block. Signals
(`SigId=TreeId`, `TreeArena::intern`) and FIR (`fir::encoding::intern_tag`) are
both hash-consed; the shared DAG exists. The FIR CSE pass (`signal_fir/cse.rs`)
ref-counts fan-out and materializes shared nodes — but `build.rs` runs it only
on the flat buckets (constants/control/sample_loop), and `value_children_of`
**deliberately does not descend into block/loop bodies** (`If => vec![cond]`,
per its own comment "separate execution scopes"). The `ondemand` block is a
`BoolIf` body (`module/clocked.rs`), so CSE never visits the transform inside
it → no `DeclareVar` temps → the hash-consed DAG is re-emitted as inlined trees.

So the wall is a one-line scope-coverage gap, not unrolling, not the eval budget,
not the FFT formulation — and it hits *every* nontrivial computation inside any
P3 clock-domain block. Fix = run CSE per nested scope (each block/loop body its
own bucket, temps local to the scope), which improves compile time and runtime
together (O(N^2.6)→O(N log N)).

Wrote the full diagnosis + phased correction plan:
`porting/fft-scalability-cse-in-clocked-blocks-2026-07-09-en.md` (Phase A =
per-scope CSE; B = iterative evaluator / propagate memoization to pass N=512;
C = rfft + twiddle folding + -vec). Added §6 to
`docs/ondemand-fft-spectral-comparison-en.md`. No code change yet — analysis only.

## FFT scalability — Phase A: per-scope CSE inside guarded blocks (implemented)

Branch: `ondemand-vec-fad-synthesis`

Implemented Phase A of the scalability plan
(`porting/fft-scalability-cse-in-clocked-blocks-2026-07-09-en.md`): the FIR CSE
pass now runs **per execution scope** instead of only on the flat tier buckets,
so shared sub-expressions inside `ondemand`/US/DS guarded blocks (and loop
bodies) are materialized into temporaries.

`crates/transform/src/signal_fir/cse.rs`:
- New `materialize_scope(store, statements, counters)` computes per-scope
  ref-counts and, for each statement carrying a nested body (`If`/`Control`/
  `SimpleForLoop`/`ForLoop`/`IteratorForLoop`/`WhileLoop`/`Block`), recurses via
  `rewrite_scope_body` — each body is its own materialization scope with temps
  declared **inside** the body (never hoisted across the guard).
- `TypedCounters` is threaded as `&mut` through the whole scope tree so temp
  names stay unique; `RewriteState` no longer owns the counters.
- `materialize_shared_values` lost its precomputed `ref_counts` argument (each
  scope recomputes its own); `module/build.rs` call sites updated.

Guard conditions and loop headers are intentionally left untouched (they belong
to the enclosing scope and may be re-evaluated), so correctness is preserved: a
temp can only be shared within the scope that declares it.

Result — the framed FFT is now **O(N log N)**, not O(N^2.6):
- arithmetic ops N=8→128: 188/1080/6488/39048/234632 → 104/301/834/2179/5428
  (**43× fewer at N=128**, `ops/(N log N)` flat 4.3→6.1, ×/doubling ≈ 2.5);
- N=256 interp codegen 9.2 M → 60 k lines (~152×), compile 8.6 s → 1.8 s;
  N=128 1.5 M → 26 k lines, 1.6 s → 0.9 s.
- N=512 still fails at `FRS-EVAL-0099` (front-end eval budget = Phase B).

Numerics unchanged: `interleave_fft` (bins == DFT), impulse-runner effect checks
(identity wet==dry@(N-1); lowpass 0.375/0.302/0.125/…; PV identity err 1.5e-5),
and `cpp_clocked_differential` (C++ parity) all pass.

Golden housekeeping: the spectral fixtures import the repo-root `interleave.lib`,
which is outside the golden import search path (`tests/corpus` +
`/usr/local/share/faust`), so they can't be golden-checked. Added

interleave.lib).

## FFT runtime cost measured — the O(N)/sample OLA is the real bottleneck

Branch: `ondemand-vec-fad-synthesis`

Benchmarked the N=1024 Wiener denoiser as generated C++ (`clang -O3`, mono
@48 kHz) vs a hand-written classical STFT denoiser (iterative radix-2 + O(1)
ring-buffer OLA) on the same machine:

- Faust ondemand N=1024: **2136 ns/sample, 10.3 % of one core (~10 voices/core)**.
- Classical radix-2 STFT: 93.6 ns/sample, 0.45 % core → **Faust ~23× slower**
  (~50–100× vs FFTW).

Isolation (identity frame op, no FFT): `il.interleave_hop(1024,512,id)` alone
costs **1751 ns/sample (82 % of the total)**. So the FFT+IFFT+gate amortized is
only ~385 ns; the dominant cost is `serialize_out`
(`par(j,N, up0(H):@(j)) :> _`), which sums N delayed clock-masked lanes **every
sample** — O(N)/sample. A classical resynthesis is O(1)/sample (a single output
accumulator ring buffer: N-wide scatter-add once per hop, read-and-clear one cell
per sample).

Recorded as **Phase C** (top runtime priority, above Phase D) in
`porting/fft-scalability-cse-in-clocked-blocks-2026-07-09-en.md` §5, with §6
holding the measured table. Key point: it cannot be a pure-Faust library fix —
`par:>_` is inherently O(N)/sample and `rwtable` has one write per activation so
the frame-wide scatter-add is inexpressible; it needs a dedicated `serialize_out`
/ OLA primitive (ring-buffer accumulator) or a pattern-matching lowering of the
masked-delay-sum idiom. Expected O(N)→O(1) win ~5–6×, larger than rfft+SIMD.

Phase A stays the enabler (made N=1024 *compile*); this note reframes the
*runtime* priority. No code change — measurement + plan only.

## Phase C generality — gather/scatter asymmetry, blocking vs rate-change

Branch: `ondemand-vec-fad-synthesis`

Measured the gather side in isolation: `serialize_in(1024) + ondemand(sum)`
(no serialize_out, 1 output) = **4.2 ns/sample** (112-line codegen) vs the
`serialize_out` scatter at **1751 ns/sample**. The two sides are NOT symmetric:
gather is already O(1)/sample + O(N)/hop (one delay line + fire-time reads);
only the scatter-add has the O(N)/sample problem.

Recorded in the plan (§5, under Phase C) as "Generality: these are the blocking
operators of multirate, not FFT sugar":
- `serialize_out` primitive = the fundamental fix, general to all frame-based
  multirate (filter banks, block conv, feature/neural frames, LPC) — worth doing
  independent of FFT; priority `serialize_out` ≫ `serialize_in`.
- `serialize_in` primitive = optional for the contiguous case; earns its keep for
  polyphase/strided gather, guaranteeing the fire-time lowering, and symmetry.
- Sufficiency: yes for the *blocking* axis, with three caveats — they optimize
  the glue not the payload (Phase A/C), they do not change rate (that is the
  orthogonal `upsampling`/`downsampling` axis), and they are contiguous-only
  until a strided `serialize_in` lands. Framing: US/DS = rate-change axis,
  serialize_in/out = blocking axis; together with `ondemand` they complete an
  efficient multirate primitive set (polyphase framework).

No code — analysis + plan only.

## Vector mode P6 / V5 — chunk driver emission (bit-exact vs scalar)

Branch: `ondemand-vec-fad-synthesis`

Vector mode now emits the `-vec` chunk driver in `module/build.rs`
(`emit_sample_loop`), in **vector mode only** — scalar codegen is byte-untouched
(golden-check 190 OK). A forward sample loop under `ComputeMode::Vector` becomes:

```c
for (int vindex = 0; vindex < count; vindex += vec_size) {
    int vend = (vindex + vec_size < count) ? vindex + vec_size : count;
    for (int i0 = vindex; i0 < vend; i0++) { <body> }
}
```

Bit-exact by construction: the inner loop keeps the **global** sample index `i0`
(no I/O pointer rebasing), so the body is identical to scalar and only the loop
bounds are chunked. Reverse-time loops force scalar (chunking would change the
implicit TBPTT window — §5 caveat). FIR-verifier-driven fixes: a general
`ForLoop` init must be the loop var's `DeclareVar`; comparisons are Int32
(not Bool).

V6 oracle test `crates/compiler/tests/vector_mode.rs`: compiles a DSP scalar and
`Vector { vec_size: 32 }`, runs both through the interpreter over a 64-sample
block, asserts **byte-identical** output. 4 cases — stateless gain, integrator +
delay crossing the chunk boundary, `vec_size` not dividing the block (tail chunk),
a two-pole recurrence. Also confirmed C++ bit-exact via a scratch harness.

The un-partitioned inner loop is already valid `-vec` (the C compiler vectorizes
it when the body has no loop-carried dependency); loop *separation* (splitting
recursive groups into serial loops, so the vectorizer sees the non-recursive
parts) is the next slice.

Verified: golden-check 190 OK, vector_mode 4/4, `cargo test --workspace` 77/77,
clippy -D warnings clean.

## Vector mode P6 / V5b — loop-carried classification (separation foundation)

Branch: `ondemand-vec-fad-synthesis`

Architectural finding first: true per-statement loop *separation* (splitting a
recursive core from its vectorizable pre/post parts, with cross-loop chunk
buffers) cannot be a post-pass. By the time a slice reaches the FIR it is a fused
loop-carried chain (`fRecCur = fRec + …; out = f(fRecCur); fRec = fRecCur`), so
the split has to happen during lowering (a current-loop stack over the signal
graph, C++ `VectorCompiler`-style) — a larger rewire of `core_lowering`, its own
focused effort.

What *is* recoverable from the FIR, and the useful first step, is the
**classification**: `loop_graph::slice_has_persistent_state` walks a slice's
statements (into guarded-block / loop bodies) for a `Struct`-access
`StoreVar`/`StoreTable` — a recursion carrier `fRec*`, a delay line, an `fIOTA`.
`build.rs` now marks each loop node `Recursive` (has state) or `Vectorizable`
(state-free), and vector mode **chunks only `Vectorizable` slices**: a
fully-recursive body cannot auto-vectorize as one block, so wrapping it in the
chunk driver is pure overhead — it stays one plain serial loop. Verified with
`--lang cpp`: `_ * 0.5` → chunk driver; `_ : + ~ _` → plain loop.

Still bit-exact (both kinds compute the same, scalar ignores the kind):
golden-check 190 OK, vector_mode oracle 4/4 (the recursive cases now emit a plain
loop and remain byte-identical to scalar), `cargo test --workspace` 77/77, clippy
-D warnings clean. New unit test `persistent_state_detection`.
