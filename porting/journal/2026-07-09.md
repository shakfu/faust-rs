## Clock domains ‚Äî lock unparenthesized OD precedence rejection

Commit date: 2026-07-09
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

Rechecked the original unparenthesized independent-OD-domain fixture against the
C++ reference compiler. C++ rejects it as a sequential-composition arity error,
because `PAR` binds tighter than `SEQ` in Faust's block-diagram grammar. Current
faust-rs rejects the same source for the same class of error.

Added a regression test that runs both sides:

- the C++ reference codegen must fail and mention sequential composition;
- faust-rs `compile_source_to_signals` must also fail and mention sequential
  composition.

This guards against accidentally accepting the invalid unparenthesized form in
the future. The valid independent-domain differential remains the named
`od2`/`od3` form.

Verified:

- `cargo fmt --all`
- `cargo test -p compiler --test cpp_clocked_differential unparenthesized_independent_ondemand_domains_reject_like_cpp`
- `cargo test -p compiler --test cpp_clocked_differential`

## Clock domains ‚Äî expand held-payload parity guardrails

Commit date: 2026-07-09
Source commit: this commit
Branch: `ondemand-vec-fad-synthesis`

Follow-up to the held-payload clock-domain fix from 2026-07-08. The original
fix made `PermVar(Clocked(env, value))` lower the held payload inside the
guarded block even when clock inference gives the payload an ancestor
environment. This session tightened the regression surface so future changes
cannot silently reintroduce top-rate state advancement.

Changes:

- Reworded `signal_fir/module/clocked.rs` to document the general held-payload
  rule instead of describing it as a waveform-specific exception.
- Added a structural C++ emission test proving that a domain-free recursive
  held payload updates its `iRec*` state inside the OD `if` before the held
  output is read.
- Expanded `cpp_clocked_differential` from 4 to 10 clock-domain cases, covering
  boolean OD, integer OD, upsampling, downsampling, long circular recursion, and
  independent OD domains in one DSP.

## Corpus ‚Äî ondemand FFT compile-scaling + spectral-effect examples

Branch: `ondemand-vec-fad-synthesis`

Added ondemand-FFT corpus fixtures, all consuming the real `interleave.lib`
(S2 primitive, commit c088ced) rather than re-inlining `frame_clock` /
`serialize_in`. They need `-I <faustlibraries>` (analyzers.lib, signals.lib)
**and** `-I .` (interleave.lib at repo root).

Analysis-only (2N held bin reals, no resynthesis):
`ondemand_fft_framed_{004,008,016,032,064,128,256}.dsp` ‚Äî
`il.serialize_in(N) : (il.frame_clock(N), si.bus(N)) : ondemand(fftFX(N))`.
Same construction locked by `crates/compiler/tests/interleave_fft.rs`.

Complete round-trip (time ‚Üí spectrum ‚Üí effect ‚Üí time) via `il.interleave(N, FX)`
with an N-in/N-out frame operator `real‚Üícomplex : an.fft : <effect> : an.ifft :
real-part`:
- `ondemand_fft_roundtrip_id_016.dsp` ‚Äî identity FX, correctness anchor: wet ==
  dry `@(N-1)` (verified with impulse-runner: dry impulse at frame 0, wet at 15).
- `ondemand_fft_lowpass_{016,032}.dsp` ‚Äî brickwall LP by Hermitian-symmetric bin
  masking (keep |m| ‚â§ kc). Impulse response is exactly `(1+2¬∑Œ£cos)/N`
  (N=8,kc=1 ‚Üí 0.375, 0.302, 0.125, ‚àí0.052‚Ä¶).
- `ondemand_fft_robot_016.dsp` ‚Äî robotization: replace each bin by (|X|, 0)
  (zero phase). Magnitude preserved (impulse ‚Üí impulse, as expected).

Why the analysis fixtures do **not** use `il.interleave`: that primitive appends
`serialize_out` (OLA back to one time stream) and requires `FX : N‚ÜíN`. The
FFT frame operator is `N‚Üí2N` (2N bin reals) and analysis must stop at frame rate
to expose the spectrum ‚Äî `il.interleave(N, fftFX)` fails propagation with
`FRS-PROP-0002 left outputs (2N) != right inputs (N)`. This is the
"analysis-only mode (no serialize_out)" of
`porting/interleave-spectral-primitive-2026-07-07-en.md` ¬ß6. The round-trip
fixtures do use `il.interleave` because their FX round-trips back to `N`.

Measured (`faust-rs --lang interp`, release):

- Analysis N=4‚Üí256 all compile; `.fbc` size/time grow ~4√ó per doubling
  (N=64 250k lines/0.4s, N=128 1.5M/1.6s, N=256 9.2M lines/8.6s).
  **Ceiling: N=512 fails** ‚Äî `FRS-EVAL-0099 stack overflow in eval (depth
  budget 1024)`: the box evaluator's guarded recursion budget, not FFT
  butterfly codegen, is the wall (`an.fftb`/`c_bit_reverse_shuffle` recursion
  crosses it at N‚â•512). Matches the ¬ß6 compile-time risk.
- Round-trip is far heavier (fft + ifft + serialize_out): N=64 lowpass ~12s,
  N=128 > 2 min ‚Äî the **practical** ceiling is ~N‚â§64, well before the N=512
  eval-budget wall. Kept the round-trip effect fixtures at N ‚â§ 32.

More spectral effects (all `il.interleave`, impulse-runner verified):
- `ondemand_fft_highpass_016.dsp`, `ondemand_fft_bandpass_016.dsp` ‚Äî symmetric
  bin masks (`min(m,N-m)` band predicate), same round-trip skeleton as lowpass.
- `ondemand_fft_fastconv_{016,032}.dsp` ‚Äî fast convolution: per-bin product with
  a fixed kernel spectrum `H(m)=DFT{h}` folded at compile time (`sum(k,L,‚Ä¶)`),
  `h=[0.5,0.5]`. Rectangular framing ‚áí *circular* convolution per frame; impulse
  response is exactly 0.5 @(N-1) + 0.5 @N.

Overlap-add STFT (phase-vocoder structure) on `il.interleave_hop(N, hop, FX)`:
- `ondemand_stft_cola_016.dsp` ‚Äî COLA reconstruction probe. Periodic Hann
  analysis window, hop=N/2, identity spectral stage, constant source. wet ‚Üí 1.0
  exactly in steady state (latency N-1): `wa[n]+wa[n+N/2]==1`. Negative control
  (rectangular window, same hop) double-counts the overlap ‚Üí gain 2.0, proving
  the window is the COLA agent, not the harness.
- `ondemand_stft_robot_ola_016.dsp` ‚Äî phase-vocoder robotization (magnitude kept,
  phase zeroed) with the windowed overlap-add path.

Full sweep: 17/17 ondemand FFT/STFT fixtures compile through `--lang interp`.
Every fixture needs `-I <faustlibraries> -I .` (interleave.lib at repo root).

## Docs ‚Äî ondemand FFT vs existing spectral environments

Branch: `ondemand-vec-fad-synthesis`

Added `docs/ondemand-fft-spectral-comparison-en.md`: an English synthesis note
comparing the pure-Faust frame-rate FFT (built on `ondemand` / `interleave.lib`)
against Max `pfft~`, Pd, SuperCollider `PV_*`, Csound `pvs`, and C++ FFTW/pffft,
along three axes ‚Äî compile time, runtime cost of the generated code, and
spectral-algorithm expressiveness. Grounded entirely in this session's
measurements (the `--dump-cpp` evidence that butterflies are frame-clock-guarded
and twiddles are folded to literals; the N=4‚Üí256 compile ladder and the N=512
eval-budget wall; the impulse-runner-verified effect/COLA fixtures).

Key framing: Faust *unrolls everything*, which explains both ends ‚Äî competitive
small-N runtime (amortized to frame rate, immediate twiddles, no loop overhead)
and the compile-time wall / large-N I-cache cost. The decisive win is a
composable, dependency-free, multi-backend, **differentiable** STFT in one graph;
the open gap versus SC/Csound is inter-frame phase accumulation (phase-vocoder
time-stretch), which lands on the S4 differentiable-STFT milestone.
