### fbc_to_cpp codegen: bug-fix pass on karplus.fbc output (4 commits)
Commit date: 2026-02-28

Context:

- First real-world validation of the AOT generator using `karplus.fbc`
  (a polyphonic Karplus-Strong DSP from the WAC 2017 Faust archive).
- The generated `.cpp` file did not compile; four distinct bugs identified
  and fixed across four commits.

Bugs fixed:

**1. Invalid C++ float literals without decimal point** (`f023dcf`)

- `fmt_real_lit` used Rust's `Display` for `f32`/`f64`, which omits the
  trailing `.0` for whole-number values (`0.0 → "0"`, `1.0 → "1"`).
- In C++, `0f` is lexed as an invalid octal constant; `1f` and `440f` as
  invalid decimal constants (the `f` suffix is only valid on floating-point
  literals, which require a `.` or exponent in many compiler/standard modes).
- Fix: after formatting, if the string contains no `.` or exponent character,
  insert `.0` before the type suffix. E.g. `0 → 0.0f`, `440 → 440.0f`.

**2. Variable name collision (`fR0` redefined)** (`f023dcf`)

- `compile_block_body` created a fresh `BlockComp` (counters `rc=0, ic=0`)
  for every block call. `instanceInit()` compiled 4 sub-blocks in sequence
  into the same C++ function body; `compute()` compiled 2. Each fresh
  `BlockComp` restarted at `fR0`/`iI0`, causing duplicate local variable
  declarations (C++ error: redefinition of `fR0`).
- Fix: removed `compile_block_body`; each method now creates one `BlockComp`
  and calls `comp.compile_block(...)` for all its sub-blocks sequentially,
  keeping counters monotone across the whole function scope.

**3. Abstract class — three missing pure-virtual overrides** (`de3dcad`)

- The Faust `dsp.h` base class declares `instanceConstants(int)`,
  `instanceResetUserInterface()`, and `instanceClear()` as pure virtual.
  The generated class only implemented `instanceInit()`, leaving the class
  abstract and non-instantiable.
- Fix: added explicit `override` implementations for all three methods, each
  emitting the corresponding FBC block (`init_block`, `reset_ui_block`,
  `clear_block`). Added `override` to `instanceInit` as well.

**4. `declare(&fVec[-1], ...)` — out-of-bounds array access** (`de3dcad`)

- A `Declare` UI instruction with `offset == -1` means a group-level
  annotation (no associated heap slot). The generator emitted `&fVec[-1]`,
  which is undefined behaviour and triggers a `-Warray-bounds` warning
  (and `-Werror` failure).
- Fix: when `offset < 0`, emit `nullptr` instead of `&fVec[offset]`.

**5. Lifecycle refactor: add `classInit`, make `instanceInit` a pure orchestrator** (`3da4f28` + `53579a2`)

- `static_init_block` (sample-rate-independent table init) was inlined
  directly in `instanceInit()` with a comment. Moved into a dedicated
  `void classInit(int sample_rate)` method, matching Faust's own lifecycle
  naming.
- `instanceInit` is now a pure call chain: `classInit → instanceConstants
  → instanceResetUserInterface → instanceClear`, with no inline emitted code.
- `classInit` is not declared `virtual` in `dsp.h`, so `override` would be
  a compile error; corrected in a follow-up commit.

Validation:

- `cargo test -p codegen` ✅ (182 tests, 9 fbc_to_cpp pass after each fix)
- `karplus.cpp` compiles without errors after the four commits.

### FBC → native C++ AOT code generator (`codegen::backends::fbc_to_cpp`)
Commit date: 2026-02-28

Context:

- The previous `crates/compiler/src/fbc_cpp.rs` module emitted a C++ *wrapper*
  that embedded raw `.fbc` text and delegated all DSP work to the Faust
  interpreter runtime (`readInterpreterDSPFactoryFromBitcode`, `interpreter_dsp`).
  That approach required the Faust C++ runtime at link time and produced no
  standalone, portable C++ code.
- Goal: replace it with a true ahead-of-time compiler that reads the structured
  `FbcDspFactory<R>` IR and emits self-contained native C++ — no interpreter
  runtime dependency at the output side.

Changes:

- Added `crates/codegen/src/backends/fbc_to_cpp/mod.rs` (~1 400 lines):
  - Public API: `FbcCppOptions` (class name, `#pragma once`, namespace),
    `FbcCppError`, `generate_cpp_from_fbc<R: FbcReal>(&factory, &opts) -> Result<String, FbcCppError>`.
  - `CppGen<'_, R>`: class-level emitter — heap declarations (`iVec`, `fVec`),
    all lifecycle methods (`classInit`, `instanceInit`, `buildUserInterface`,
    `metadata`, `compute`), static block-constant tables.
  - `BlockComp`: block-level stack-machine compiler with independent real/int
    virtual stacks (named C++ temporaries `fRN` / `iIN`).
  - All 294 FBC opcodes covered: Numbers, Load/Store/Move/Shift/BlockStore,
    Cast/Bitcast, four-variant arithmetic forms (Stack×Stack, Heap×Heap,
    Heap×Stack, Value×Stack, Value×Heap, inverted), unary/binary extended
    math, Loop, CondBranch, If, SelectReal, SelectInt, UI, Nop.
  - Control-flow translations:
    - `Loop(init, body)` → `{ <init>; while (true) { <body> } }`
    - `CondBranch` → `if (!<cond>) { break; }`
    - `If(t, f)` → virtual-stack save/restore around each branch
    - `SelectReal/Int` → pre-declared merge variable captured from both branches
  - Real literal formatting: `std::numeric_limits<T>::quiet_NaN()` / `infinity()`
    for special values; `f`-suffix for `float`, no suffix for `double`.
  - 9 unit tests (empty factory, nop, arithmetic, cast, loop, select, blockstore,
    ui/meta, full-round-trip), all passing.
- Updated `crates/codegen/src/backends/mod.rs`: added `pub mod fbc_to_cpp;`.
- Updated `crates/compiler/src/main.rs`:
  - Removed deleted `mod fbc_cpp;` (file no longer exists).
  - Added `use codegen::backends::fbc_to_cpp::{FbcCppOptions, generate_cpp_from_fbc};`
    and `read_fbc` to the existing interp import.
  - Replaced `fbc_cpp::render_cpp_wrapper_from_fbc_file(...)` with the native
    pipeline: `read_fbc::<f32>` → `generate_cpp_from_fbc` → `emit_output`.

Validation:

- `cargo check -p compiler` ✅ (fixed E0583 + E0277)
- `cargo test --workspace` ✅ (0 failures)

Usage:

```sh
# 1. compile DSP to .fbc
cargo run -p compiler -- --lang fir mysynth.dsp -o mysynth.fbc
# 2. emit native C++ from .fbc (no runtime dependency)
cargo run -p compiler -- --dump-cpp-from-fbc mysynth.fbc -o mysynth.h
```
