## Plan: scalar emission determinism (delay-line ordering)

Commit date: 2026-07-20
Source commit: this commit

Wrote `porting/scalar-emission-determinism-plan-2026-07-20-en.md`, the first
follow-up the cleanup plan deferred: fixing the pre-existing run-to-run
nondeterminism of scalar emission recorded at R0/R3 (77 of 396 cases frozen
in `nondeterministic-frozen.txt`; reproducer `zita_rev1.dsp`).

The plan upgrades the HANDOFF "suspect" to a fully traced root cause, read
in code at this baseline: `DelayPlan.lines: HashMap<(SigId, Option<u32>),
i32>` (`delay/plan.rs:61`) is iterated directly by `prepare_delay_lines`
(`module/setup.rs:221`), which fixes both the `fVec*` struct-field order
(declaration push, `delay/manager.rs:163`) and the `lDelayN` numbering
(clear-loop registration order feeding `fresh_loop_var`,
`delay/context.rs:122`); `DelayManager.delay_lines` is additionally iterated
in emission-visible positions (`lines()` via `module/clocked.rs:371`,
`emit_sample_end_updates` values order).

Key decisions recorded in the plan:

- canonical `BTreeMap`/`BTreeSet` order on `(SigId, Option<u32>)` rather
  than insertion order (no C++ byte-alignment gate exists; no new
  dependency); the already-deterministic 319 cases can only be stable with
  <= 1 entry per randomized iteration, so any stable order keeps them
  byte-identical — expected diff surface is exactly the 77 frozen cases
  ("77-only rule");
- gate before fix: new `cargo xtask emission-determinism` (double emission,
  byte-compare, sorted findings) landed in D1 with an allowlist initialized
  to the frozen list, shrunk to empty in D2; the gate, not a manual audit
  of hash-using files, is the completeness argument;
- D3 dissolves the exclusion list (arbiter goes 319 -> 396 refereed cases)
  under the full semantic battery; D4 records a rejecting mutation
  (reintroduced `HashMap` must turn the gate red) and notes that
  stable-but-different orders are owned by the byte-identity/golden gates
  instead.

Status: proposed; no production code touched by this commit.
