# 2026-05-11

## Disable ReverseTimeRec RAD dispatcher fast path

Commit date: 2026-05-11
Source commit: this commit (`Disable ReverseTimeRec RAD dispatcher fast path`)

Implemented `porting/rad-disable-reverse-time-rec-fast-path-plan-2026-05-10-en.md`.
The public RAD propagation path no longer emits `SigReverseTimeRec` for
recursive, temporal, or `SigIIR` cases. The symbolic reverse sweep remains
feed-forward only; recursive projections now raise `recursive-projection`,
structured IIR carriers raise `iir-state-space`, and both kinds are consumed by
the `SigBlockReverseAD` fallback in `generate_rad_signals`.

Per-crate changes:

- `crates/propagate/src/reverse_ad.rs`:
  - Removed the `ReverseADTransform` recursive projection frontier and the
    dispatcher calls that built `ReverseTimeRec` from public `rad(...)`.
  - Routed all `SigIIR` carriers through the `iir-state-space` fallback path.
  - Added `iir-state-space` to the block fallback whitelist.
  - Kept the lower-level LTI transpose helpers and direct unit tests as dormant
    infrastructure.
  - Updated RAD Rustdoc to describe the symbolic sweep plus `SigBlockReverseAD`
    fallback and mark the legacy LTI/IIR path dormant.

- `crates/propagate/tests/core_api.rs` and in-source reverse AD tests:
  - Replaced public-dispatch `ReverseTimeRec` shape expectations with
    `BlockReverseAD` carrier assertions for strict LTI recursion and IIR cases.
  - Kept direct `ReverseTimeRec` helper tests enabled.

- `crates/signals/src/lib.rs`,
  `crates/transform/src/signal_fir/module.rs`, and
  `crates/transform/src/signal_fir/recursion.rs`:
  - Added dormant-infrastructure notes to the preserved carrier and lowering
    entry points.

- Clippy cleanups:
  - Fixed a `signals::filter_algebra` range-loop warning.
  - Adjusted current-toolchain style/doc warnings in `signal_fir` and test
    support code so `cargo clippy --workspace --all-targets -- -D warnings`
    is green.

Validation:

- `cargo fmt --all`
- `cargo check -p propagate -p signals -p transform`
- `cargo test -p propagate`
- `cargo test -p signals -p sigtype -p normalize -p transform`
- `cargo test -p compiler --test diagnostic_errors`
- `cargo test -p compiler --test block_reverse_ad`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo run -q -p compiler -- --dump-sig tests/corpus/rad_lti_recursive_one_pole.dsp`
  showed `SIGBLOCKREVERSEAD` and no `SIGREVERSETIMEREC`.
- `cargo run -q -p compiler -- --dump-sig tests/corpus/rad_lti_recursive_state_space.dsp`
  showed `SIGBLOCKREVERSEAD` and no `SIGREVERSETIMEREC`.

Known remaining issue:

- `cargo test -p compiler --test rad_runtime` still fails
  `debug_softclip_gradient_values` with an FBC `stack_underflow`; the other
  45 tests passed and one softclip TBPTT test remains ignored. This appears to
  be the existing softclip/debug runtime issue rather than a `ReverseTimeRec`
  dispatcher regression.
