### Fixed Linux/Windows duplicate-symbol failures in unified `faust-ffi` test builds
Commit date: 2026-02-27

Addressed CI linker failures caused by duplicated global C symbols when running
workspace-wide tests after introducing the unified `faust-ffi` facade.

Root cause:

- `box-ffi` and `cranelift-ffi` exposed `standalone-capi-globals` as default
  features.
- with Cargo feature unification in workspace test builds, those defaults were
  enabled transitively, re-exporting global symbols also defined by `interp-ffi`
  (`getCLibFaustVersion`, `startMTDSPFactories`, `stopMTDSPFactories`,
  `freeCMemory`).
- Linux/Windows linkers rejected these duplicate global definitions.

Fix:

- changed default features to empty in:
  - `crates/box-ffi/Cargo.toml`
  - `crates/cranelift-ffi/Cargo.toml`
- kept `standalone-capi-globals` opt-in only for explicit standalone backend
  distributions.
- added crate-level `#![allow(non_snake_case)]` in:
  - `crates/box-ffi/src/lib.rs`
  - `crates/cranelift-ffi/src/lib.rs`
  to preserve C API naming parity without breaking `clippy -D warnings`.

Validation:

- `cargo check --workspace --all-targets` ✅
- `cargo clippy --workspace --all-targets -- -D warnings` ✅

### Interp refactor executed: inline ownership for `BlockStore*` payloads (6-step plan completed)
Commit date: 2026-02-27

Executed the planned interpreter hardening refactor end-to-end:

1. Introduced instruction-owned payload model:
   - `FbcInstruction<R>` now has `block_store: Option<BlockStoreData<R>>`.
2. Migrated serializer:
   - writer emits `BlockStore*` from inline instruction payload.
   - reader attaches parsed table payload directly to instruction.
3. Migrated executor:
   - `BlockStoreReal/Int` execution reads payload from current instruction
     instead of block-level side-table lookup.
4. Migrated optimizer:
   - rewrite/copy paths now preserve payload by plain instruction clone.
   - removed rewrite-time side-table remap logic.
5. Removed side-table representation:
   - `FbcBlock.block_store_data` removed.
   - `push_block_store` now sets inline payload on instruction.
6. Updated tests and validated:
   - bytecode/serial/optimizer tests adapted to inline payload checks.
   - `cargo test -p codegen -- --nocapture` ✅ (all passing).

Compatibility note:
- `.fbc` external text format remains unchanged; refactor is internal memory
  representation hardening only.

### Documented interpreter hardening refactor plan: inline `BlockStore*` payload ownership
Commit date: 2026-02-27

Added a dedicated refactor plan in:

- `porting/phases/faust-rust-interpreter-porting-en.md`

Goal:

- replace the current `FbcBlock` side-table model
  (`block_store_data: Vec<(instruction_index, payload)>`)
  with instruction-owned payload for `BlockStoreReal/BlockStoreInt`, closer to
  C++ object semantics and structurally safer during optimizer rewrites.

Why:

- side-table indexing is rewrite-sensitive and previously allowed payload-loss
  regressions when blocks were rebuilt.
- instruction-owned payload removes this class of bug by construction.

Plan content added:

- target representation sketch (`FbcInstr` style enum)
- 6-step incremental migration sequence (serializer -> executor -> optimizer ->
  side-table removal)
- explicit validation gates:
  - unit tests across serialization/optimizer/executor
  - optimization invariants per level (`1..=6`)
  - runtime differential checks (`opt_level=0` vs `opt_level=6`) on table-heavy `.fbc` cases
- acceptance criteria for completion.

### Fix Interp regression: preserve `BlockStore*` payloads during optimizer rewrites
Commit date: 2026-02-27

Fixed a runtime regression in the interpreter backend where optimized `.fbc`
programs could lose static table initialization data.

Root cause:

- `optimizer::apply_rewriter` rebuilt `FbcBlock` instructions but dropped
  `block_store_data` metadata attached to `BlockStoreReal`/`BlockStoreInt`.
- this made table writes in `static_init_block` no-op at runtime, yielding
  incorrect all-zero audio on some programs (for example waveform tables).

Changes:

- `crates/codegen/src/backends/interp/optimizer.rs`
  - preserve `block_store_data` on copied instructions during rewrite passes
  - preserve payload on safe 1:1 emits that keep `BlockStore*` opcode
- added non-regression test:
  - `test_block_store_payload_preserved_across_optimization`
  - asserts `BlockStoreReal` payload survives optimization passes

Validation:

- `cargo test -p codegen test_block_store_payload_preserved_across_optimization -- --nocapture` ✅
- manual runtime repro with external `waveform.fbc`:
  - before fix (`opt_level >= 1`): output all zeros
  - after fix (`opt_level = 6` from file): output matches expected sequence
    (`0.25, 0.30, 0.35, 0.40, 0.45, ...`)
