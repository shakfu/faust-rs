### Box FFI pass: new `box-ffi` crate + shared tree helpers + FIR/source export wiring
Commit date: 2026-02-27

Implemented the full `boxes` FFI pass as a dedicated crate and aligned related
infrastructure so the C/C++ API can be exercised with consistent memory and
conversion behavior.

Main deliverables:

- Added new crate `crates/box-ffi` (workspace member) with:
  - C export surface (`libfaust-box-c.h` style) and C++ convenience wrapper
    (`libfaust-box.h`)
  - global context lifecycle (`createLibContext` / `destroyLibContext`)
  - large `Cbox*` constructor/matcher surface
  - conversion pipeline exports:
    - `CDSPToBoxes`
    - `CboxesToSignals`
    - `CboxesToSignals2` (symbolic path via `de_bruijn_to_sym`)
    - `CcreateSourceFromBoxes` for `c|cpp|fir|interp`
  - `freeCMemory` ownership handling for strings and tracked pointer arrays
- Added `box-ffi` dependencies and workspace wiring:
  - `Cargo.toml`
  - `Cargo.lock`
- Introduced safe shared helpers in `tlib`:
  - `tree_to_string`
  - `tree_to_int`
  - `tree_to_double`
- Reused the new safe helpers in existing crates to reduce duplicated
  node-decoding logic:
  - `crates/compiler/src/lib.rs`
  - `crates/fir/src/lib.rs`
  - `crates/propagate/src/lib.rs`
  - `crates/tlib/src/recursion.rs`
  - `crates/transform/src/signal_fir/module.rs`

Documentation:

- Added Rustdoc and per-symbol `# Safety` sections across the `box-ffi` exports.
- Added crate-level `box-ffi/README.md`.
- Added header smoke tests under
  `crates/box-ffi/tests/header-smoke/`.

Validation:

- `cargo fmt --all` ✅
- targeted compile/clippy checks run during the pass on touched crates ✅
