## Split propagate crate facade into focused modules

Commit date: 2026-06-08
Source commit: this commit

Split the large `crates/propagate/src/lib.rs` implementation into focused
internal modules while preserving the crate's public propagation API.

Implementation updates:

- kept `lib.rs` as the crate facade and public re-export point;
- moved typed entry points into `api.rs`;
- moved arity inference into `arity.rs`;
- moved flat-box validation and FAD-recursion classification into `flat.rs`;
- moved propagation diagnostics into `error.rs`;
- moved grouped UI collection into `ui_build.rs`;
- moved the propagation dispatcher, route helpers, clock wrappers, and
  De Bruijn helpers into `engine.rs`;
- moved former inline unit tests into `tests.rs`;
- added module-level rustdoc to every extracted file to document its boundary
  responsibility and propagation invariants;
- adjusted intra-crate visibility to keep implementation details internal;
- fixed public rustdoc links that pointed at private modules after the split.

Validation:

- `cargo fmt --all`
- `cargo check -p propagate`
- `cargo test -p propagate --all-targets`
- `RUSTDOCFLAGS='-D warnings' cargo doc -p propagate --no-deps`
