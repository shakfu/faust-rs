## Split high-priority large modules

Commit date: 2026-06-08
Source commit: this commit

Split the remaining high-priority large Rust modules into focused internal
modules while preserving their public crate/backend facades.

Implementation updates:

- kept `crates/xtask/src/main.rs` as the command dispatcher and moved workflow
  implementation into `shared`, `code_graphs`, `wasm`, `backend_align`,
  `fir_dump`, `golden`, `runtime_trace`, `reports`, and `tests` modules;
- kept `crates/codegen/src/backends/cranelift/mod.rs` as the public backend
  entry point and moved implementation details into `api`, `core`, `host`,
  `jit_data`, `lowering`, and `subset` modules;
- kept `crates/fir/src/lib.rs` as the public FIR facade and moved types,
  store, builder, matcher, dump, and encoding helpers into dedicated modules;
- added module-level rustdoc to every newly extracted file so the split
  documents each module's responsibility and invariants at the file boundary;
- adjusted intra-crate visibility for sibling modules and moved tests without
  broadening public APIs.
- fixed a stale public rustdoc link in the Julia backend that pointed at a
  private helper.

Validation:

- `cargo fmt --all`
- `cargo check -p xtask`
- `cargo check -p codegen`
- `cargo check -p fir`
- `cargo test -p xtask --all-targets`
- `cargo test -p codegen --all-targets`
- `cargo test -p fir --all-targets`
- `RUSTDOCFLAGS='-D warnings' cargo doc -p xtask -p codegen -p fir --no-deps`

## Split propagate crate facade into focused modules

Commit date: 2026-06-08
Source commit: this commit

Split the large `crates/propagate/src/lib.rs` implementation into focused
internal modules while preserving the crate's public propagation API.

Implementation updates:

- kept `lib.rs` as the crate facade and public re-export point;
- moved typed entry points into `api.rs`;
- moved arity inference into `arity.rs`;
- moved flat-box validation and FAD-recursion classification into `flat.rs`;
- moved propagation diagnostics into `error.rs`;
- moved grouped UI collection into `ui_build.rs`;
- moved the propagation dispatcher, route helpers, clock wrappers, and
  De Bruijn helpers into `engine.rs`;
- moved former inline unit tests into `tests.rs`;
- added module-level rustdoc to every extracted file to document its boundary
  responsibility and propagation invariants;
- adjusted intra-crate visibility to keep implementation details internal;
- fixed public rustdoc links that pointed at private modules after the split.

Validation:

- `cargo fmt --all`
- `cargo check -p propagate`
- `cargo test -p propagate --all-targets`
- `RUSTDOCFLAGS='-D warnings' cargo doc -p propagate --no-deps`
