### `eval` — Port C++ `evalLabel(...)` placeholder semantics into Rust evaluator

Commit date: 2026-03-07

Source commit: pending (pre-commit eval implementation step in this session)

#### Context

The Rust evaluator only supported literal labels for modulation target matching.
The C++ evaluator supports `%...` substitutions through `evalLabel(...)` and
uses that logic across modulation and UI/group label-bearing nodes.

Without this step, Rust still diverged from C++ on:

- interpolated modulation target labels,
- interpolated widget/group labels,
- width-formatted placeholder rendering,
- malformed placeholder fallback behavior.

#### Changes

- `crates/eval/src/lib.rs`
  - added an internal `eval_label` helper with Rustdoc provenance mapped to the
    C++ `evalLabel(...)` / `writeIdentValue(...)` pair
  - ported the `%ident`, `%<width>ident`, and `%{ident}` placeholder parser
  - resolved placeholder values through scalar constant evaluation using
    `eval -> a2sb -> propagate`
  - integrated evaluated labels into modulation target handling and dedicated
    UI/group/widget evaluator branches

- `crates/eval/Cargo.toml`
  - added `propagate` and `signals` dependencies needed for scalar constant
    evaluation during label interpolation

- `crates/eval/tests/core_eval.rs`
  - added regression coverage for:
    - simple placeholder substitution,
    - width clamping behavior,
    - malformed placeholder fallback,
    - undefined placeholder failure,
    - interpolated modulation target matching

#### Validation

- `cargo fmt --all`
- `cargo test -p eval --test core_eval`
