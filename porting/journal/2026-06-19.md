## Plan signal graph input/output latency analysis

Commit date: 2026-06-19
Source commit: this commit

Added `porting/signal-io-latency-analysis-plan-2026-06-19-en.md`, a planning
document for computing `[min, max]` audio input/output latency intervals over
the signal graph.

The plan explains the expected uses of the analysis, defines an explicit
latency domain, recommends a `transform`-level analysis module, covers
propagation rules for sources, transparent nodes, delays, variable delay
intervals, bargraphs, recursive groups, and
`ondemand`/`upsampling`/`downsampling` clock-domain boundaries, and lists the
parity decisions that must be resolved before implementation. It also defines
the intended user-facing access path: an explicit signal-dump companion first,
then a stable `--latency-report=text|json` option once the semantics are ready
for tooling.
