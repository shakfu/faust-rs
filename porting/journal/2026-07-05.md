# 2026-07-05

## Shared C-family emitter core: c/cpp backends deduplicated, 4 drifts closed (Phases 1–5)

Executed Phases 1–5 of `porting/c-family-emitter-core-plan-2026-07-04-en.md`
on branch `main-dev`, addressing improvement #5 of the port quality review:
the `c` and `cpp` backends were near-parallel implementations (~1 700 lines,
42–44 functions each) tracking each other function for function, with seven
confirmed silent drifts (same function, different behavior) between them.

New module `crates/codegen/src/backends/c_family.rs` (1 032 lines, 9 unit
tests) owns the shared emission core, parameterized by plain data — no traits:

- `CFamilySyntax` — fixed token leaves (`bool`/`int`, `nullptr`/`NULL`,
  UI glue threading, `FAUSTFLOAT` cast spelling, switch `default` style).
- `CFamilyValueCtx` / `CFamilyStmtCtx` — per-language seams (fn pointers for
  capture-free rules, `&dyn Fn` for `Options`-aware rendering, `EmitNodeFn`
  recursion seams) consumed by `emit_value_common` / `emit_stmt_common`.
- Shared leaf functions: `emit_binop(_expr)`, `emit_type`, `trim_float`,
  `format_float32`, `string_literal`, `emit_static_tables`.
- The core owns only the **intersection** of both backends (`None`-contract:
  unmatched arms fall back to each backend's own language-only arms and
  error type), so unification cannot silently grant one backend arms the
  other had.

20 statement arms and all common value arms are now shared; per-language
remainders: `AddMetaDeclare`, `Label`, cpp-only `DeclareFun`, plus cpp's
`Quad`/array literals, `NewDsp`, `Bitcast`.

Drift outcomes (numbering from the plan §2):

| Drift | Status | How |
|---|---|---|
| 1 — cpp dropped `DeclareTable` initializer values | **closed** | shared arm, c as reference; regression test |
| 2 — c lacks `Bitcast`; cpp spelling matches no oracle | deferred | standalone fix against upstream `*reinterpret_cast<T*>(&v)`; unifying now would launder the wrong spelling |
| 3 — `-0.0` normalization missing in cpp | **closed** | shared `trim_float`; regression test |
| 4 — `\r`/`\t` escaping missing in cpp | **closed** | shared `string_literal`; regression test |
| 5 — `FAUSTFLOAT` cast missing on cpp UI widgets | **closed** | `CFamilySyntax` cast leaves; oracle = in-repo upstream golden `tests/golden/cpp/rep_09_ui_slider/` (`FAUSTFLOAT(0.5f)`) |
| 6 — cpp lacks `StructInit`/`TableInit` reset replay | open | Phase 6 follow-up |
| 7 — c lacked `Control`/`WhileLoop` | **closed** | shared arms; regression test |

Single-owner decision: the former cpp-only `DeclareStructType` /
`DeclareBufferIterators` / `ShiftArrayVar` / `IteratorForLoop` arms were
silent comment stubs (`IteratorForLoop` even unrolled its body once — wrong
code instead of an error); both backends now hard-error, with a rejection
regression test. These nodes only occur on interp/cranelift/asc paths today.

Guardrails green after every phase: `cargo fmt`, workspace clippy
`-D warnings`, `cargo test -p codegen` (273) and `-p compiler` (333),
`golden-check` 190/190, `cpp-backend-diff-report` 8/8 with 0 DIFF. The only
rust golden artifacts are stdout fingerprints, so the intentional DRIFT 5
output change is locked by the updated `cpp_fir_sine_phasor.rs` integration
test rather than a golden regeneration.

Result: `cpp/mod.rs` 1 756 → 1 430, `c/mod.rs` 1 676 → 1 326 — ~680 lines of
duplicated surface removed while *gaining* four bug fixes. The drift class
"fixed in one backend, forgotten in the other" is structurally impossible for
everything the core owns. Upstream C++ precedent: `Text.hh/.cpp` is already
shared across its ~12 backends.

Method: analysis, plan, and Phases 1–2 plus most of 3–5 by a **Sonnet 5
sub-agent** (twice interrupted by session limits); completion, stabilization,
