Status: active parity workstream (as of February 19, 2026).

Context:
- Current shell-signature parity is green on representative cases (`OK=8` in this report).
- Full C++ text parity (`cargo run -p xtask -- golden-check-cpp`) still reports a high diff volume (34 cases at last run).
- Most remaining differences are envelope/emission-shape differences rather than pure type/signature mismatches.

Goal:
- Align Rust C++ backend output envelope with C++ reference emitter output so `golden-check-cpp` diffs become actionable semantic deltas instead of bulk formatting/ordering noise.

Scope (in):
- File prologue envelope:
  - optional comment/banner handling policy,
  - macro blocks and include ordering,
  - platform guards (`__APPLE__`, `RESTRICT`) ordering/placement.
- Class envelope:
  - `class mydsp : public dsp` layout style,
  - section ordering (`private`/`public`),
  - method block ordering (`metadata`, `getNumInputs/Outputs`, lifecycle methods, `compute`).
- Deterministic emission rules:
  - stable statement ordering where C++ reference is deterministic,
  - newline/blank-line policy and trailing-space normalization.

Scope (out, separate tracks):
- Missing semantic support in parse/eval/propagate/transform pipeline (`case`, `abstr`, `modulation`, stream wrappers, long-delay lowering).
- Full behavioral parity of unsupported corpus cases.

Execution plan:
1. Baseline classification:
   - split `golden-check-cpp` failures into:
     - envelope-only diffs,
     - semantic/unsupported diffs.
2. Prologue normalization:
   - align macro/include/platform guard emission order in `crates/codegen/src/backends/cpp/mod.rs`.
3. Class/method envelope normalization:
   - align class field/function declaration order and block formatting.
4. Determinism hardening:
   - ensure stable traversal and emission ordering for all generated sections.
5. Re-measure:
   - rerun `cargo run -p xtask -- golden-check-cpp`,
   - update this report and `phase-6-backend-full-corpus-diff-report-en.md` with reduced diff counts and residual root causes.

Acceptance criteria:
- `golden-check-cpp` no longer dominated by envelope-level bulk diffs.
- Residual diffs are predominantly semantic/feature gaps and can be triaged to eval/propagate/transform tasks.
